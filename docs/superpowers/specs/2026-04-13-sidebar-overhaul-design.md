# サイドバー大幅改修 設計書

## 概要

サイドバーのコンテンツ選択UXを抜本的に改善する。現在の4階層構造（レベルフィルタ→カテゴリフィルタ→シリーズアコーディオン→コンテンツ）を、3+1タブのフラット構造に置き換える。合わせてアーカイブ機能・右クリックコンテキストメニュー・Shift+クリック複数選択を追加する。

## 背景と課題

- レベルフィルタ→カテゴリフィルタの2段階操作が冗長
- 零式は実質最新ティアしか使わないのに、過去ティアが同列に並ぶ
- 絶は全部で6つ程度なのに、レベル→カテゴリで絞り込む必要がある
- SeriesAccordion/CategoryAccordionの階層が深すぎる

## 設計

### 1. タブ構成

レベルフィルタ（SegmentButton）とカテゴリフィルタ（SegmentButton）を完全廃止し、4タブに置き換える。

| タブ | 表示内容 | 備考 |
|------|---------|------|
| ⚔ 零式 | 現在の拡張パッケージの零式（全ティア） | 黄金: LHW + CW + HW |
| 👑 絶 | 全絶コンテンツ（レベル不問） | UCoB〜FRU、フラット一覧 |
| 📁 その他 | ダンジョン・レイド・フリープラン | カテゴリ dungeon/raid/custom |
| 📦 アーカイブ | 過去拡張のアーカイブ済みプラン | 遅延ロード+解凍 |

- タブ状態はローカル保持（リロードで零式タブに戻る）
- データ構造（ContentLevel, ContentCategory, seriesId等）は一切変更しない
- 表示のフィルタリングロジックのみ変更

### 2. 各タブ内の表示

#### 零式タブ
- セクションラベル（例:「至天の座アルカディア零式：ヘビー級」）でティアを区切る
- 各ティア内のコンテンツはフラットに並ぶ（SeriesAccordionは廃止）
- 「最新拡張」の判定: `getContentDefinitions()` から最大levelを持つ `category === 'savage'` を抽出

#### 絶タブ
- 全絶コンテンツをフラットに一覧
- レベルや拡張による区切り・アコーディオンなし

#### その他タブ
- ダンジョン・レイド・フリープランを表示
- 現在のFreePlanSectionと同等の表示

#### アーカイブタブ
- `archived: true` のプランを表示
- レベル順（70→100）、カテゴリ順（零式→絶→その他）でソート
- プランをクリック → ローディングアニメーション中に解凍 → 通常通り開く
- 一括削除オプションあり

### 3. コンテンツ行のUI変更

#### ホバー「+」ボタン追加
- コンテンツ行（ContentTreeItem）の右端に、ホバー時に出現する「+」ボタンを追加
- クリック → 既存の `onSelect(content, true)` と同じハンドラを呼ぶ（プラン名ダイアログ→createPlanDirectly）
- コンテンツを展開しなくても新規プラン追加が可能に

#### プラン行のアクションアイコン
- 現状維持（常時表示）。ホバーで隠す方式には変更しない

### 4. 右クリックコンテキストメニュー

プラン行を右クリック（スマホ: ロングプレス）でコンテキストメニューを表示する。

**メニュー項目:**
- シェア（URL生成）
- 複製
- 名前変更
- ─（区切り線）
- 削除（赤色）

既存のインラインアクションアイコン（複製・編集・削除）もそのまま残す。コンテキストメニューは追加の操作手段として提供。

### 5. Shift+クリック複数選択

- プラン行をShift+クリックで複数選択に入る
- 選択中はフローティングアクションバーが表示される（シェア・削除）
- 既存の複数選択モードボタン（Multi-Select / Select Delete）もそのまま残す
- Shift+クリックはモードレスな追加手段として併存

### 6. アーカイブ機能

#### データモデル
`SavedPlan` に以下のフィールドを追加:
```typescript
interface SavedPlan {
  // ...既存フィールド
  archived?: boolean;          // アーカイブ済みフラグ
  compressedData?: string;     // 圧縮済みデータ（base64）
}
```

#### アーカイブ時の処理
1. `archived = true` に設定
2. `data`（PlanData）を JSON.stringify → gzip圧縮 → base64エンコード → `compressedData` に保存
3. `data` フィールドを削除（Firestoreストレージ節約）
4. 圧縮率: 70〜85%削減（9KB → 1.5〜3KB）

#### アーカイブタブで開くとき
1. `compressedData` を base64デコード → gzip解凍 → JSON.parse → `data` に復元
2. 既存のローディングアニメーション中に実行（追加遅延: 1〜2ms、体感ゼロ）
3. `data` は一時的にメモリに展開されるのみ

#### 再圧縮タイミング
- **A) プラン切替時**: アーカイブプランから別プランに切り替えたとき、現在のプランを保存する既存処理（Sidebar.tsx:297-300）の後に再圧縮（+2ms、トランジションアニメーション1.1秒に完全に隠れる）
- **C) 起動時フォールバック**: アプリ起動時に `archived === true && data !== undefined` のプランを検知して再圧縮（ブラウザ突然終了時の回収。Firestore読み込みと並行で追加遅延ゼロ）

#### 自動アーカイブのトリガー
- 管理画面で新レベル（例: 110）のコンテンツ追加時、`dataVersion` がインクリメントされる
- クライアントがバージョン変更を検知し、新レベルのコンテンツが存在する場合:
  - 初回アクセス時に確認ダイアログ「過去拡張のプランをアーカイブしますか？」
  - 了承 → 前拡張の零式プランを一括アーカイブ（圧縮含む）
  - 拒否 → そのまま（零式タブに表示され続ける。次回起動時に再度確認）
- 同じ拡張内のティア移行（例: LHW→CW→HW）ではアーカイブしない。全ティアを零式タブに表示

#### パフォーマンス影響

| 処理 | 対象ユーザー | 頻度 | 追加遅延 |
|------|------------|------|---------|
| プラン切替時の再圧縮 | アーカイブを開いた人のみ | 切替のたび | 2ms（アニメに隠れる） |
| 起動時の未圧縮回収 | ブラウザ突然終了した人のみ | 次回起動時のみ | 0ms（並行処理） |
| 通常利用 | 全ユーザー | 常時 | 影響なし |

### 7. チュートリアル変更

#### mainチュートリアル
- ステップ1 `main-1-content`: `[data-tutorial-first-item]` の付与先を変更。タブ化後は零式タブ内の最初のコンテンツに付与する
- チュートリアル開始時の初期化（Sidebar.tsx:772-776）: `setActiveLevel` / `setActiveCategory` → タブを「零式」に切り替える処理に置き換え
- その他のステップ（2〜14）: サイドバー構造に依存しないため変更不要

#### create-planチュートリアル
- 変更不要。NewPlanModal内で完結しており、サイドバーのタブ構造に依存しない

#### shareチュートリアル
- 変更不要。サイドバー構造に依存しない

### 8. NewPlanModal

- 変更なし。レベル→カテゴリ→コンテンツ→プラン名のフローはそのまま
- close時のサイドバー同期（Sidebar.tsx:1469-1483）: `setActiveLevel` / `setActiveCategory` → 作成されたプランが所属するタブに切り替える処理に置き換え

### 9. contentRegistry

- 既存関数は変更なし（NewPlanModalや他の箇所がそのまま使える）
- 新規関数を追加:
  - `getCurrentExpansionLevel()`: 最新拡張のレベルを返す（例: 100）
  - `getSavageForCurrentExpansion()`: 現在の拡張の零式コンテンツ一覧を返す
  - `getAllUltimates()`: 全絶コンテンツ一覧を返す
  - `getOtherContents()`: dungeon/raid/customコンテンツ一覧を返す

### 11. スマホ対応

- 4タブは276px幅に収まる（1タブ約69px）
- 現在のボトムシート形式は維持
- コンテキストメニューはスマホではロングプレスで発火
- Shift+クリックはスマホでは使えないため、既存の複数選択モードボタンがモバイルの主要手段
