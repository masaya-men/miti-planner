# 野良主流 OGP 表示ポリシー整理 設計書

**作成日**: 2026-04-28
**対象**: `MitigationSheet`（野良主流ボトムシート）／`ShareModal`／OGP 生成

---

## 1. 背景

野良主流ページ・ボトムシートに、共有されたプランの **OGP 画像とプラン名** が表示されている。
規約上、共有プランは匿名集計の対象として野良主流に掲載される合意を取得済み。
ただし以下が懸念として残っている。

- **プラン名の意図せぬ拡散**：ユーザーが私的に付けたプラン名（メモ的・内輪的な命名）が公開ランキングに晒される
- **OGP 表示時の身元情報**：チームロゴ等が、本人が意図しない文脈（ランキング等）で見られる
- **同意取得の不透明さ**：「規約に書いてある」だけで、実際に共有する瞬間に「この行為が何を意味するか」がユーザーに明示されていない

## 2. 設計方針

### 2.1 プラン名は「載せない」一択にする

プラン名は野良主流に掲載しない。OGP 画像にも焼き込まない。
理由：

- プラン名は通常、内輪向けの私的命名で、他者にとっては情報価値が低い
- OGP にはコンテンツ名が大きく描画されており、識別には十分
- 「プラン名を載せたい人」のための機能を残すと、UI／同意設計が複雑化する

### 2.2 ロゴ ON/OFF で「身元公開の可否」を決める

野良主流への掲載自体はデフォルト OK（規約合意済み）。
**ロゴトグル（既存 `team_logo.show_on_ogp`）が、身元公開の意思表示を兼ねる。**

| ロゴトグル | 共有 OGP | 野良主流での見え方 |
|-----------|----------|----------------|
| ON | ロゴ付き | ロゴ付き OGP（チーム識別可） |
| OFF | ロゴなし | ロゴなし OGP（匿名） |

ロゴを持たないユーザーは自動的にロゴ OFF と同等。匿名で野良主流に掲載される。

### 2.3 同意の明示化

「共有 = 野良主流掲載候補になる」「ロゴ ON = ロゴが野良主流に出る」を、
**初回共有時のワンタイムダイアログ + 共有モーダル内の常駐キャプション** の二段で伝える。

## 3. 機能変更詳細

### 3.1 削除対象

#### a. プラン名 OGP 焼き込みトグル
- 共有モーダルに存在する「プラン名を共有画像に表示」トグル
- 現状の OGP API パラメータ `&showTitle=false` の制御スイッチ

**処理:**
- ShareModal の該当 UI 削除
- OGP 生成側（`ogpHelpers.ts` `buildOgImageUrl`）の `showTitle` を **常に `false` 固定** に変更
  - 既存呼び出し側で `showTitle: true` を渡している箇所は、引数自体を削除
  - `&showTitle=false` を常に付ける（または OGP API 側のデフォルトを `showTitle=false` に変更）
- `ogpImageHash.ts` の hash 計算からも `showTitle` フィールドを削除（型変更）
- 関連テスト（`ogpImageHash.test.ts`、`ogpHelpers.test.ts`）の更新
- i18n キー `share.show_title_on_ogp`（または現キー名）の削除

> 注：OGP 画像生成 API（`api/og` 側）が `showTitle=false` をどう処理しているかは、実装フェーズで確認。デフォルトを反転できるならそちらが綺麗。

#### b. 野良主流シートカードのプラン名
- `MitigationSheet.tsx` 内 `miti-card-bottom` の `miti-card-title`（`{entry.title}` 表示部）を削除
- 削除後のカード構成：OGP 画像 + ジョブアイコン + コピー数バッジ + コピーボタン

#### c. 野良主流ページカードのプラン名（`PopularPage.tsx`）
- `entry.title` を表示している箇所を削除
- 代替表示は不要（コンテンツ名・コピー数・ジョブで十分）
- X 共有テキスト生成（`handleShareX`）も `entry.title` 依存を解除：
  - 現状 `${contentName} - ${entry.title}` → `contentName` のみに変更
  - `contentName` が空の場合のフォールバックは検討要（プロジェクト名等）

### 3.2 維持対象

- ロゴトグル `team_logo.show_on_ogp`（変更なし）
- 共有 URL での個別共有（プラン名は共有 URL の `<title>` メタタグや SharePage 上では引き続き表示。野良主流への掲載とは別）

> SharePage の挙動は変更しない方針。プラン名は「共有 URL を直接見た人」には見えるが、「野良主流ランキング経由で出会った人」には見えない、という分離。

### 3.3 新規追加

#### a. ワンタイムダイアログ（初回共有時）

トリガー：
- `shareId` を Firestore に新規登録（＝野良主流の集計対象になる瞬間）する直前に表示
- ユーザー単位（ログインユーザーは Firestore のユーザードキュメント、未ログインは localStorage）で「同意済みフラグ」を持ち、初回のみ表示
- 既に共有済みユーザー（同意フラグなしで shareId が既存）に対しては、次の共有時に1度だけ表示

文言（日本語版、他言語は実装時に i18n 同等翻訳）:

> **共有について**
>
> 共有したプランは、個人が特定できない形で集計され、人気になると **野良主流ページ** に表示されることがあります。
>
> **ロゴをONにしている場合**、野良主流ページにもロゴ付きで表示されます（チーム宣伝向け）。
> 匿名のままにしたい場合はロゴをOFFにしてから共有してください。
>
> [キャンセル] [理解して共有する]

UI:
- 既存のモーダル系（glass-tier3）と同じスタイル
- スマホ: 下からスライドイン

データ:
- ログインユーザー: `users/{uid}.consentedToPopularDisplay: true`
- 未ログイン: `localStorage.lopo.popularDisplayConsent = '1'`

#### b. 共有モーダル内の常駐キャプション

ロゴトグルのすぐ下（または同セクション内）に、小さく以下を常時表示。
**ⓘ アイコン + `text-app-xs text-app-text-muted`** 程度の控えめさ。

> ⓘ 共有したプランは匿名で集計され、野良主流ページに掲載されることがあります。
> ロゴ ON の場合は、ロゴも野良主流に表示されます。

i18n キー: `share.popular_display_notice`（またはネスト下に新設）

### 3.4 既存データの扱い

既存の野良主流エントリーはユーザー本人の自作分のみ（他者なし）。マイグレーション不要。

## 7. 設計上の意思決定の記録

- **Q. なぜプラン名を「常に非表示」にしたか？**
  - 内輪命名が多く、他者にとっての情報価値が低い
  - OGP API・UI・同意フローを単純化できる
  - 載せたい場合は OGP 画像の content name が代替として機能する

- **Q. なぜロゴトグルに同意の意味を兼ねさせるか？**
  - ロゴを付ける = チーム宣伝意図あり、と素直に解釈できる
  - トグル数を増やさず UX を簡潔に保てる
  - ロゴなしユーザーも匿名で野良主流に載るので、取りこぼしは生じない（規約合意済みの前提）

- **Q. なぜ初回ダイアログ + 常駐キャプションの二段構えか？**
  - 初回ダイアログ: 強い印象で1度だけ確実に伝える
  - 常駐キャプション: 認知が薄れた頃に再認識できる
  - どちらか一方だけだと、伝達漏れか UI ノイズかでバランスが取りにくい
