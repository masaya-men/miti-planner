## 1. 背景・課題

FFLogs インポートは現状**完全上書きのみ**で、取り込むたびに既存のタイムラインイベントも配置済み軽減も全消去される。これにより 2 つの不便がある。

1. **ワイプログで前半を作り込んだ後、クリアログで後半を足したい** のに、追記ができず全部やり直しになる（⑤）。
2. **同じ戦闘をより正確なログで取り直したい** だけなのに、配置済みの軽減まで失われる。

### 根拠（実コード）

- 上書き処理は全イベント置換＋軽減クリア: `src/store/useMitigationStore.ts:923` `importTimelineEvents()`、`:944` で `timelineEvents` を全置換、`:945` で `timelineMitigations: []`。
- 軽減は**時刻ベース**で、イベント ID への参照を持たない: `src/types/index.ts:92` `AppliedMitigation { id, mitigationId, time, duration, ownerId, targetId?, ... }`。→ イベントを差し替えても軽減は時刻に独立して存在でき、保持は技術的に容易。
- イベントは FFLogs の ability GUID を保持済み: `src/types/index.ts:107` `guid?: number`、変換経路 `src/utils/fflogsMapper.ts`（mapper が GUID を保存）。本 Phase では未使用だが、Phase 1.5 の再アンカーで利用する土台。
- インポート入口はユーザー側では FFLogs 1 つ（`src/components/Timeline.tsx:3844` `<FFLogsImportModal>`）。CSV インポートは管理画面専用（`src/components/admin/AdminTemplates.tsx:530`）。ルートの `src/components/CsvImportModal.tsx` は未参照（デッドコードの疑い、本 Phase では触らない）。

## 2. 取り込みモード（3 種）

プレビュー画面でモードを選択し、「インポート」ボタンで実行する。

| モード | i18n キー（仮） | イベント | 軽減 | 用途 |
|---|---|---|---|---|
| 置き換え（軽減も削除） | `fflogs.import_mode.replace_all` | 全置換 | 全消去 | 別の戦闘を入れる／最初からやり直す |
| **置き換え（軽減は残す）** ※既定 | `fflogs.import_mode.replace_keep` | 全置換 | 時刻のまま保持 | 同じ戦闘をより良いログで取り直す |
| 追記（既存より後のイベントのみ） | `fflogs.import_mode.append` | 既存の最終時刻より後だけ追加 | 全保持 | ワイプログ→クリアログで後半を足す |

- **既定 = 「置き換え（軽減は残す）」**。データ消失に厳しい本プロジェクト方針に合わせ、非破壊な選択肢を既定にする。軽減が消えるのは「軽減も削除」を明示選択した時だけ。
- 文言は淡々とした説明のみ（「まっさら」等の砕けた表現は使わない）。

### 軽減保持の意味（Phase 1 では絶対時刻）

「置き換え（軽減は残す）」では軽減を**配置済みの絶対時刻のまま**残す（再アンカーしない）。新ログで同じ技の着弾時刻がズレている場合、軽減が新イベントから多少ズレ得るが、全消去より大幅に良く、ユーザーが微調整すればよい。**技に合わせて軽減もずらす「再アンカー」は Phase 1.5（§6）に分離**（ユーザー合意済）。

## 3. UI

### 配置

プレビュー画面（`status.phase === 'preview'`）のフッター直前。現在そこにある「上書き警告」バナー（`src/components/FFLogsImportModal.tsx:317-321` `fflogs.warning_overwrite`）を**モード選択 UI に置き換える**。「インポート」ボタン（`:553` `handleImport`）は選択中モードを適用する。

### 表示条件

- **既存タイムラインが空（`timelineEvents.length === 0`）のときはモード選択を出さない**。どのモードでも結果が同じなため、従来どおり「インポート」一発。
- 既存タイムラインがある時のみ 3 モードのラジオを表示。

### モックアップ（PC・モバイル共通の論理構造）

```
┌─ プレビュー: ○○（ボス名）  4m 32s ────────────┐
│  取り込み: イベント 48 / フェーズ 5 ...           │
│                                                   │
│  取り込み方法:                                     │
│   ○ 置き換え（軽減も削除）   ⚠軽減も削除されます    │
│   ● 置き換え（軽減は残す）                         │
│   ○ 追記（既存より後のイベントのみ）                │
│                                                   │
│             [ キャンセル ]   [ インポート ▶ ]      │
└───────────────────────────────────────────────────┘
```

- 白黒ベース。選択中は白枠/背景で強調（既存トグルの作法に合わせる）。
- 「軽減も削除」を選んだ時のみ ⚠ 黄色注記（`text-app-amber` 系、既存の警告色トークン）。
- 「インポート」ボタンは青（`bg-app-blue`、=実行）。デザインルール（白黒＋機能色）準拠。
- i18n: ラベル・注記は全て i18n キー経由（ja/en/ko/zh）。en で崩れないこと。

## 4. 追記モードの挙動

- **取り込み範囲**: 取り込むイベントのうち `time > （既存タイムラインの最終イベント時刻）` のものだけを追加。
- **境界の重複**: `time ===（既存最終時刻）` ちょうどのイベントは取り込まない（既存優先）。
- **フェーズ**: 後半に存在する新規フェーズは追加。既存フェーズは変更しない。フェーズの境界時刻が既存最終時刻より後のもののみ追加。
- **軽減・パーティ編成・ラベル**: 一切変更しない。
- **時間軸**: 同一戦闘の延長を前提とするため、取り込むイベントの時刻はそのまま使う（オフセット補正なし）。別戦闘の連結は本 Phase の対象外。

## 5. データ変更とロジック

`importTimelineEvents()`（`src/store/useMitigationStore.ts:923`）を、**モードを必須引数**として取る形に拡張する。呼び出し元は FFLogsImportModal の 1 箇所のみなので、必ずモードを明示して渡す（暗黙デフォルトは設けない＝意図しない挙動を防ぐ）。UI 上の既定選択（プリセレクト）が「置き換え（軽減は残す）」であることと、関数引数の必須性は別レイヤーである点に注意。

- `'replace_all'`: 現状の挙動（events 全置換 ＋ `timelineMitigations: []`）。
- `'replace_keep'`: events 全置換、`timelineMitigations` は**触らない**。
- `'append'`: 既存 events に `time > lastTime` の新 events を結合してソート、`timelineMitigations` 等は触らない。

### 共同編集（collab）との整合

タイムラインは collab 時 Yjs にも同期される（`src/lib/collab/collabProvider.ts`）。本変更は「軽減を消さない/イベントを部分追加する」だけで、同期経路自体は既存の import フローを通すため新たな破壊リスクは低い。ただし **collab ON 時のインポート挙動は実機 2 タブで回帰確認**（軽減が他参加者側でも消えない/追記が正しく伝播する）を必須とする。

## 6. スコープ外（明示）

- **再アンカー（技に合わせて軽減もずらす）** = Phase 1.5。`AppliedMitigation` に技参照が無いため、取り込み時に「軽減開始直後の被ダメージイベントをアンカーとみなす」等の推測ルール＋ GUID＋出現順マッチが必要で、曖昧さ（複数occurrence/複数技カバー/近接イベント無し）の設計を独立して行う。プレビュー＋やり直し可能にして誤爆を抑える。
- **スプレッドシート取込（⑥）** = Phase 2。取込元が 2 つ以上になるため、その時に「汎用インポート → 取込元チューザー」へ導線再設計。
- **多言語翻訳の仕上げ（⑧）** = 既に GUID ベースで概ね実装済み（`src/components/admin/FflogsTranslationModal.tsx:155` 他）。本 Phase では扱わない。
- **CSV インポートの一般ユーザー開放** = 本 Phase 対象外。

## 7. テスト方針

- **回帰（最重要）**: `replace_keep` で `timelineMitigations` が保持されること（消えないこと）を store の単体テストで永久ロック。
- `replace_all` が従来どおり軽減を消すこと。
- `append`: 既存最終時刻より後のイベントのみ追加・同時刻は除外・既存イベント/軽減が不変であること。
- 空タイムライン時はモードに関わらず全件取り込み（モード UI 非表示）。
- i18n: 4 言語キーの存在と en 表示崩れなし。
- collab ON: 2 タブ実機で軽減非消去・追記伝播を確認（自動テスト外の手動検証）。
