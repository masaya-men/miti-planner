# みんなの軽減表 — ボトムシートUI 設計書

## 概要

「みんなの軽減表」は、共有プランのコピー数に基づく野良向け基準軽減表をアプリ内ボトムシートで提供する機能。別ページ（`/popular`）ではなく、メインアプリ内で完結する体験を目指す。

### ビジョン

- **野良の軽減スタンダードをLoPoが作る** — 「軽減はLoPoのこれでいい」と思ってもらえる状態
- 零式も絶も野良需要がある。自分で組みたい人以外は基準があれば使いたい
- 信頼の源泉: 有名攻略者が共有 → コピー数で自然に浮上 + 管理者ピックアップ

### 設計原則

- ユーザー情報は一切保持しない（匿名のまま）
- 追加コストゼロ（既存API + Vercelキャッシュで完結）
- 作業フローを壊さない（別ページに飛ばない）
- モダンなアニメーション体験

- コントロールバーのボタン（既存のPopularボタンを置き換え）をクリック

## コンポーネント詳細

### 1. カプセル通知

- **位置**: 画面上部中央（シート外、`position: fixed`）
- **表示タイミング**: ボトムシート登場後 0.8秒遅延
- **アニメーション**: スプリングバウンス（上から降りてきて、少しオーバーシュート → 戻る）
- **内容**: `現在 {コンテンツ名} を編集中`（i18nキー）
- **パルスドット**: 青い点が呼吸アニメーション
- **閉じ**: シート閉じ時にフェードアウト

### 2. ボトムシート本体

- **高さ**: 80vh
- **背景**: glassmorphism tier3（`rgba(22,22,24,0.78)` + `blur(16px)`）
- **border-top**: `1px solid rgba(255,255,255,0.12)`
- **shadow**: `0 -12px 48px rgba(0,0,0,0.5), inset 0 1px 0 rgba(255,255,255,0.08)`
- **角丸**: `20px 20px 0 0`
- **ドラッグハンドル**: 上部中央（40px × 4px）

#### 開くアニメーション

- `translateY(100%)` → `translateY(0)` 
- `cubic-bezier(0.34, 1.56, 0.64, 1)` — スプリング系
- duration: 550ms

#### 閉じるアニメーション

- `translateY(0)` → `translateY(100%)`
- `cubic-bezier(0.4, 0, 1, 1)` — 素早く退場
- duration: 350ms
- トリガー: ×ボタン、ESCキー、オーバーレイクリック

### 3. ヘッダー

#### 1行目: ×ボタン（右寄せ）

- 32px × 32px 円形ボタン
- ESCキーでも同じ閉じ処理を実行

#### 2行目: タブ + コピーアクション

**タブ（左寄せ）:**
- `零式` / `絶`
- アクティブ: `bg rgba(255,255,255,0.1)` + 白文字
- 非アクティブ: `#71717a`

**コピーアクション（右寄せ、横並び）:**

| ボタン | スタイル | 挙動 |
|--------|---------|------|
| 選択コピー | ゴースト | チェックモード切替。全カードにチェックボックスが出現。チェックした件数を「N件をコピー」で表示 |
| 零式まとめてコピー / 絶まとめてコピー | ゴースト | アクティブタブの全コンテンツを一括コピー。データがないコンテンツはスキップ。タブ切替でラベルも切替 |
| コピーして使う | primary、一回り大きい（padding 7px 20px, font-size 12px） | 左で選択中のプランを1件コピー |

### 4. 左カラム: OGPカードリスト（280px）

各カードの構成:

```
┌─────────────────────────┐
│ 1層                      │  ← 層ラベル（i18nキー、多言語対応）
├─────────────────────────┤
│ ┌─────────────────────┐ │
│ │ OGP画像 (1.91:1)    │ │  ← /api/og?id=xxx で取得、<img>で表示
│ │                     │ │
│ │           [T][T][H]…│ │  ← ジョブアイコン（右下オーバーレイ）
│ └─────────────────────┘ │
│ 142 copies               │
└─────────────────────────┘
```

- **OGP画像**: `aspect-ratio: 1200/630`（1.91:1）。`/api/og?id={shareId}` を `<img>` で直接表示。Vercelキャッシュ済みなのでコストゼロ
- **層ラベル**: `getFloorLabel()` で取得（既存関数）。i18nで多言語対応
- **ジョブアイコン**: OGP画像上の右下にオーバーレイ。実際のジョブアイコン画像を使用
- **コピー数**: カード下部に小さく表示
- **選択状態**: `border: 2px solid rgba(255,255,255,0.35)` + `scale(1.02)`
- **チェックモード**: 右上にチェックボックスが出現。クリックでトグル

#### ドラムロールアニメーション

ボトムシート開くときの演出:

1. シート登場後 0.6秒でドラムロール開始
2. カードを仮想的に複製して2回転分の高速スクロール
3. 現在編集中のコンテンツのカードがリスト中央に正確に停止
4. 停止後グロウエフェクト（`box-shadow` アニメ）で選択状態に
5. 同時にプレビューのfetchが完了していればテーブル表示

**通常選択時も同様**: どのカードをクリックしても `scrollIntoView({ block: 'center' })` + グロウエフェクト

**イージング**: `easeOutExpo` — 高速スタート → 滑らかに減速

### 5. 右カラム: プレビューエリア

#### 案内パネル（2段）

| 種別 | アイコン | テキスト |
|------|---------|---------|
| 青（互換配置） | ↔ | ジョブ構成が違っても大丈夫 — コピー後にパーティメンバーを変更するとスキル互換配置で自動調整することもできます |
| グレー（新規プラン） | + | 新しいプランとして追加されます（編集中のプランは上書きされません）· 1コンテンツ最大5件 |

#### 軽減表ミニプレビュー

- 読み取り専用。編集不可、ハイライトなし
- 実際のplanDataをテーブル形式で描画（縮小版）
- 上下スクロール可能
- アプリ準拠スクロールバー（8px、`rgba(128,128,128,0.3)`、角丸4px）

**データ取得**: 左カード選択時に `/api/share?id={shareId}` でplanDataを取得。Vercelキャッシュ済み

**テーブル構成**:
- ヘッダー（sticky）: フェーズ | 時間 | 技名 | ジョブ1〜ジョブN
- 各セル: スキルアイコン（ミニサイズ、ロール色分け）
- フォントサイズ: 7-8px（コンパクト表示）

**ローディング**: 初回ドラムロール中はスピナー表示。データ到着でフェードイン

#### エラーハンドリング

- **5件制限到達時**: コピーボタン押下 → トースト「このコンテンツのプラン上限（5件）に達しています。不要なプランを削除してからコピーしてください」
- **データ未取得時**: カードはスケルトン表示
- **API失敗時**: トースト「データの取得に失敗しました」

## データフロー

### 一覧データ取得

```
ボタン押下
  → /api/popular?contentIds=m1s,m2s,...（既存API）
  → 各コンテンツの copyCount 降順 top1 を取得
  → OGP画像URL: /api/og?id={shareId}
  → レスポンスは Cache-Control: s-maxage=900 でキャッシュ済み
```

### プレビューデータ取得

```
カード選択（またはドラムロール停止時）
  → /api/share?id={shareId}（既存API）
  → planData からテーブル描画
  → 先行fetch: ドラムロール中にターゲットコンテンツのデータを先読み
```

### コピー処理

```
「コピーして使う」押下
  → planDataを取得（キャッシュ済みなら即座）
  → 新しいSavedPlanとしてusePlanStoreに追加
  → copyCount +1（既存ロジック: localStorage重複チェック + /api/popular POST）
  → トースト「コピーしました」

「まとめてコピー」押下
  → アクティブタブの全コンテンツのtop1プランを取得
  → データがないコンテンツは自動スキップ（エラーにしない）
  → 5件制限に達しているコンテンツもスキップし、トーストで通知
  → 各プランを順次コピー（Promise.allSettled）
  → トースト「{N}件コピーしました」（スキップがあれば「{M}件はスキップ」も併記）

「選択コピー」→ チェック → 「N件をコピー」押下
  → チェック済みカードのプランを取得
  → 各プランを順次コピー
  → トースト「{N}件コピーしました」
```

### Firestoreコスト

- `/api/popular`: CDNキャッシュ15分。1日最大 960読み取り（無料枠50,000の2%未満）
- `/api/share`: 個別プラン取得。1回1読み取り。CDNキャッシュあり
- `/api/og`: Vercel Edge Functionで画像生成。CDNキャッシュ済み
- **追加コスト: 実質ゼロ**

## 利用規約・プライバシーポリシーについて

当初の懸念: 「軽減データの集約・公開」が現規約でカバーされていないのではないか。

**結論: 追加の規約更新は不要。**

理由:
- この機能は既存の共有プラン（`shared_plans`）のコピー数ランキングを表示するだけ
- 新たなデータ収集や新しい種類のデータ利用は発生しない
- 既存の利用規約 第8条（アップロードコンテンツ）とプライバシーポリシー 第1条（作るデータ）で既にカバー済み
- 共有プランを作成した時点で公開に同意している（既存フロー）

## アニメーション仕様

| アニメーション | トリガー | duration | easing | 詳細 |
|-------------|---------|----------|--------|------|
| シート登場 | ボタンクリック | 550ms | `cubic-bezier(0.34, 1.56, 0.64, 1)` | 下から出現、スプリングオーバーシュート |
| カプセル登場 | シート登場後800ms | 700ms | 同上 | 上から降りてバウンス |
| ドラムロール | シート登場後600ms | 2200ms | `easeOutExpo` | 高速→減速、ターゲットで正確に停止 |
| グロウエフェクト | ドラムロール停止 / カード選択 | 600ms | `ease-out` | box-shadow 0→20px→0 |
| プレビューフェード | データ取得完了 | 400ms | `ease-out` | `opacity: 0, translateY(6px)` → 表示 |
| シート退場 | ×/ESC/オーバーレイ | 350ms | `cubic-bezier(0.4, 0, 1, 1)` | 下にスライドアウト |
| カプセルパルス | 常時 | 2000ms | `ease-in-out` | 青ドットの呼吸（opacity 1→0.4→1） |

## スマホ対応

- ボトムシートは**フルスクリーン**（100vh）に拡大
- 左右分割ではなく**縦積み**: OGPカードリスト（横スクロール）→ プレビュー
- ドラッグハンドルで閉じる操作対応
- タブとコピーボタンはスクロール可能な横並び
