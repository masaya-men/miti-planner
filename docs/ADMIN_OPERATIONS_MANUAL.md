## 1. 管理画面へのアクセス

### URL
**https://lopoly.app/admin**

### ログイン方法
1. LoPo本体（https://lopoly.app/）で **Googleアカウント** でログインする
2. ブラウザのアドレスバーに `https://lopoly.app/admin` と入力してアクセス
3. ダッシュボードが表示されれば成功

### アクセスできない場合
- トップページにリダイレクトされる → **ログアウト → 再ログイン** を試す
- それでもダメな場合 → 管理者ロールが設定されていない可能性あり（→ [トラブルシューティング](#10-トラブルシューティング) 参照）

## 2. 管理画面の全体像

管理画面の左側にメニューがあり、以下のページに移動できる:

| メニュー | URL | 用途 |
|---------|-----|------|
| ダッシュボード | /admin | コンテンツ数・テンプレート数の確認 |
| コンテンツ | /admin/contents | 零式・絶などのコンテンツの追加・編集・削除 |
| テンプレート | /admin/templates | タイムラインテンプレートの管理 |
| スキル | /admin/skills | ジョブ・軽減スキルの編集、アイコンのアップロード |
| ステータス | /admin/stats | レベル補正値・パッチ別ステータスの管理 |
| サーバー | /admin/servers | DC・サーバー・ハウジングエリアの管理 |
| 設定 | /admin/config | テンプレート昇格の閾値など全体設定 |
| ← アプリに戻る | /miti | LoPo本体に戻る |

## 3. 日常運用: 新コンテンツの追加

**いつやるか**: FF14に新しい零式・絶が実装されたとき

### 手順

1. メニューから **「コンテンツ」** を開く
2. 画面上部のフォームに以下を入力する:

| 項目 | 説明 | 例 |
|------|------|-----|
| ID | 英数字の短い識別子（後から変更不可） | `m5s` |
| 日本語名 | コンテンツの正式名称（日本語） | 至天の座アルカディア零式：ライトヘビー級5 |
| 英語名 | コンテンツの正式名称（英語） | AAC Light-heavyweight M5 (Savage) |
| 日本語略称 | サイドバーに表示される短い名前 | M5S |
| 英語略称 | 同上（英語） | M5S |
| カテゴリ | savage（零式）/ ultimate（絶）/ dungeon / raid / custom | savage |
| レベル | 100, 90, 80, 70 のいずれか | 100 |
| パッチ | ゲームのパッチ番号 | 7.45 |
| シリーズID | 同じシリーズのコンテンツをグループ化するID | aac_lhw |
| 表示順 | シリーズ内での並び順（1, 2, 3...） | 5 |
| FFLogs Encounter ID | FFLogsのエンカウンター番号（分かれば） | 97 |

3. **「追加」ボタン** を押す
4. 一覧に追加されたことを確認する

### シリーズの追加

新しいシリーズ（例: 新しい零式シリーズ）が来た場合:
- コンテンツ管理画面の中で、シリーズの追加もできる
- シリーズID・日本語名・英語名・カテゴリ・レベル・表示順を入力して追加

### 注意点
- **IDは後から変更できない** — 入力ミスに注意
- **日本語名と英語名の両方が必須** — 片方だけだとエラーになる
- 追加後、ユーザーのアプリに反映されるのは次回アプリ起動時（キャッシュの仕組み上）

## 4. 日常運用: テンプレートの管理

**いつやるか**: 新コンテンツにテンプレート（タイムライン）を設定するとき

### テンプレートのアップロード

1. メニューから **「テンプレート」** を開く
2. テンプレート一覧が表示される
3. **「テンプレートをアップロード」** エリアで:
   - 対象コンテンツIDを入力（例: `m5s`）
   - JSONファイルを選択してアップロード
4. アップロード完了後、一覧に表示される

### テンプレートのロック/アンロック

- **ロック**: テンプレートがFFLogsインポートから自動更新されなくなる（内容を確定したいとき）
- **アンロック**: 自動更新を再び許可する

テンプレート一覧の各行にロック/アンロックボタンがある。

### 人気プラン昇格候補の管理

ユーザーの共有プランが一定回数以上コピーされると「昇格候補」として表示される。
- **承認**: そのプランのタイムラインをテンプレートとして採用する
- **却下**: 候補から外す（プラン自体は残る）

### テンプレートの削除

テンプレート一覧の各行にある削除ボタンから削除できる。削除前に自動でバックアップが作られる。

## 5. 日常運用: スキルの編集

**いつやるか**: パッチでスキル効果が変更されたとき、新ジョブが追加されたとき

### スキルの値を変更する

1. メニューから **「スキル」** を開く
2. 左側の **ジョブ一覧** から編集したいジョブを選ぶ
3. 右側にそのジョブの **スキル一覧** が表示される
4. 変更したいスキルの値を直接編集する（軽減率、リキャスト、効果時間など）
5. **「保存」** ボタンを押す

### 変更できる値の意味

| フィールド | 意味 | 例 |
|-----------|------|-----|
| name（ja/en） | スキル名 | ホーリーシェルトロン / Holy Sheltron |
| recast | リキャストタイム（秒） | 5 |
| duration | 効果時間（秒） | 8 |
| value | 軽減率（%） | 15 |
| burstValue | バースト時の追加軽減率（%） | 5 |
| burstDuration | バースト効果の時間（秒） | 4 |
| type | 軽減タイプ（all / physical / magic） | all |
| scope | 対象範囲（self / party） | self |
| isShield | シールドかどうか | true/false |
| shieldPotency | シールドの強さ（ポーテンシー） | 1000 |

### アイコンのアップロード

1. スキル編集画面で、アイコン欄の横にある **「アップロード」** ボタンを押す
2. PNG画像を選択する
3. Firebase Storageに自動アップロードされ、スキルに紐づく

### 注意点
- 変更は保存ボタンを押すまで反映されない
- 変更前のデータは自動バックアップされる（間違えても復元可能）

## 6. 日常運用: ステータスの更新

**いつやるか**: 新パッチでステータスの上限値やデフォルト値が変わったとき

### パッチ別ステータスの追加

1. メニューから **「ステータス」** を開く
2. 「パッチ別ステータス」セクションで新しいパッチを追加:
   - パッチ番号（例: `7.50`）
   - タンクの値（HP, メインステータス, DET, WD）
   - その他ロールの値（HP, メインステータス, DET, WD）
3. 保存する

### レベル補正値の編集

新しいレベルキャップが来たとき（拡張パッケージごと）:
1. 「レベル補正値」セクションで編集
2. main, sub, div, hp の各値を更新
3. 保存する

### デフォルトレベル紐付け

どのレベルがどのパッチのステータスをデフォルトで使うかの設定:
1. 「デフォルトステータス by レベル」セクション
2. レベルとパッチの紐付けを変更
3. 保存する

## 7. 日常運用: サーバー/DC管理

**いつやるか**: 新DC追加、サーバー名変更、ハウジングエリア追加時（めったにない）

### 管理画面の構成

メニューから **「サーバー」** を開くと、4つのタブがある:

| タブ | 内容 |
|------|------|
| データセンター | DC・サーバー・エイリアス（表記揺れ） |
| ハウジングエリア | ミスト、ラベンダーベッドなど |
| ハウジングサイズ | S, M, L |
| タグ | テイスト（モダン、和風等）、季節（春、夏等） |

### エイリアス（表記揺れ）とは

同じサーバーの別の呼び方をカンマ区切りで登録する:
- 例: Aegis → `イージス, Aegis, Aeg`
- ハウジングツアー機能で検索にヒットさせるために使う

## 8. 日常運用: 設定の変更

メニューから **「設定」** を開く。

| 設定項目 | 意味 | デフォルト |
|---------|------|-----------|
| 昇格閾値（promotionThreshold） | 共有プランが何回コピーされたらテンプレート昇格候補にするか | 20 |
| 昇格倍率（promotionMultiplier） | 既存テンプレートの何倍のコピー数で昇格候補にするか | 2 |

通常は変更不要。

## 9. Firebase Consoleの操作

Firebase Console: https://console.firebase.google.com/project/lopo-7793e

### よく使う操作

#### ユーザーの確認・BAN

1. Firebase Console → **Authentication** → **Users**
2. ユーザーのメールアドレスやUIDで検索
3. ユーザー行の右端 **「…」メニュー** → **「アカウントを無効化」** でBAN
   - BAN後はログイン・データアクセス・API呼び出しが全てブロックされる
   - 元に戻したい場合は同じメニューから「アカウントを有効化」

#### Firestoreのデータ確認

1. Firebase Console → **Firestore Database**
2. コレクション一覧が左側に表示される:
   - `master` → コンテンツ・スキル・ステータスなどのマスターデータ
   - `templates` → タイムラインテンプレート
   - `plans` → ユーザーのプラン
   - `shared_plans` → 共有プラン
   - `users` → ユーザー情報
   - `admin_logs` → 管理操作の履歴
3. ドキュメントをクリックすると中身が見える

> **注意**: Firebase Consoleから直接データを編集しないこと。必ず管理画面から操作する（セキュリティルールで直接書き込みはブロックされている）

#### Storageの確認

1. Firebase Console → **Storage**
2. `icons/` フォルダにスキルアイコンが保存されている
3. 管理画面からアップロードすればここに保存される

#### 予算アラートの確認

1. Firebase Console → **使用量と請求**（左下メニュー）
2. 現在の使用量と請求額を確認できる
3. 予算アラートは500円に設定済み — 超えそうなときはメールが届く

## 10. トラブルシューティング

### 管理画面にアクセスできない

| 症状 | 対処 |
|------|------|
| トップページにリダイレクトされる | ログアウト → 再ログインする |
| それでもダメ | 1時間待ってから再度ログイン（トークン更新待ち） |
| それでもダメ | 管理者ロールが外れている可能性 → ADMIN_REFERENCE.md の手順でロール再設定 |

### データを変更したのにユーザーに反映されない

管理画面での変更はすぐにFirestoreに保存されるが、ユーザーのアプリは **キャッシュ** を使っている。
- ユーザーが **アプリを再起動**（ページリロード）すると反映される
- 自動的にバージョン番号が上がるので、次回起動時に新データを取得する

### テンプレートが自動更新されない

- テンプレートが **ロック** されていないか確認する（テンプレート管理画面で確認）
- ロック中は自動更新されない仕様

### スキルの変更を元に戻したい

管理画面でスキルを保存するたびに、変更前のデータが自動バックアップされている。
- Firebase Console → Firestore → `master_backups` コレクションで確認可能
- 現時点では管理画面からのワンクリック復元は未実装 — 開発者に連絡すること

### アプリ全体が重い・エラーが出る

1. ブラウザのキャッシュをクリア（Ctrl+Shift+Delete）
2. ページをリロード
3. 治らない場合 → ブラウザのDevToolsでConsoleのエラーを確認し、開発者に連絡

### Firebase の請求が増えている

1. Firebase Console → 使用量と請求 で確認
2. 読み取り回数が異常に多い場合 → 不正アクセスの可能性
3. 対処: 不正ユーザーをBAN（Authentication → アカウント無効化）

## 11. 定期メンテナンス

### パッチごと（約3ヶ月周期）にやること

- [ ] スキルの効果値が変わっていないか確認 → 変わっていたら管理画面で更新
- [ ] ステータスのデフォルト値を最新パッチに更新
- [ ] 新しいコンテンツ（零式・絶）が追加されていたら管理画面で追加

### 拡張パッケージごと（約2年周期）にやること

- [ ] 新ジョブが追加されていたらスキル管理画面で追加
- [ ] レベルキャップが上がっていたらレベル補正値を更新
- [ ] 新レベルのデフォルトステータスを追加

### 随時

- [ ] テンプレート昇格候補が来たら確認・承認/却下
- [ ] 不正利用の報告があればユーザーをBAN
- [ ] Firebase の請求額を月1回確認
