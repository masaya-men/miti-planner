# 多言語対応（i18n）アーキテクチャ設計方針

## 1. 目的
本アプリケーションにおける多言語対応（現在は日本語・英語を想定）を、スケーラブルかつ保守性の高い構造で実装する。

## 2. 現状の課題と解決策
現状、ソースコード内に直接記述された日本語（ハードコード）と、`react-i18next` を使用した辞書参照が混在している。これを完全に分離し、以下の2層構造で管理する。

## 3. 実装方針

### A. UIテキスト層（コンポーネント内の文字列）
ボタン名、ポップアップのメッセージ、モーダルのタイトルなど、静的なUIテキストはすべて `react-i18next` の辞書ファイル（JSON）に切り出す。

* **辞書ファイルの分離**: `src/locales/ja.json`, `src/locales/en.json` のように言語ごとにファイルを作成。
* **キーの命名規則**: `category.element_name` の形式とする（例: `ui.button_save`, `timeline.delete_confirm`）。
* **コンポーネントでの使用**: ハードコードを禁止し、必ず `useTranslation` フックの `t('key')` 関数を通して呼び出す。

### B. データ層（データベース内の文字列）
ギミック名、スキル名、ジョブ名など、マスタデータとして管理される文字列は、JSON辞書ファイルではなく、**データベース（オブジェクト）自身に多言語のプロパティを持たせる**。
（詳細はデータベース設計書の通り）

## 4. 今後のタスク
1. 既存のコンポーネント（特に最近追加したモバイル用UIなど）から、ハードコードされている日本語を抽出し、辞書ファイルへ移行する。
2. 言語切り替えトグル（UI）の実装（必要に応じて）。
