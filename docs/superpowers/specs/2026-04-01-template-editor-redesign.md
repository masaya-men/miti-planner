# テンプレート管理画面リデザイン設計書

> 承認日: 2026-04-01
> スコープ: 管理画面 /admin/templates のテンプレート編集機能強化

---

## 背景・目的

βテスターから「攻撃名の日英混在」「ダメージ不正確」「スプシと時間が合わない」等のフィードバックが多数寄せられている。根本原因はFFLogsからの自動生成テンプレートの品質問題。

現状の管理画面ではJSONファイルを丸ごとアップロードするしか編集手段がなく、個別の修正が困難。

**目的:** 記憶がなくても間違わずにテンプレートを管理できるUIを作る。

---

## スコープ

### 含む
1. プランをテンプレートにする機能
2. スプシ（CSV/TSV）からの読み込み機能
3. スプレッドシート型インライン編集UI
4. 翻訳欠落の検出・自動マッチング・手動修正

### 含まない（別設計書で対応）
- コンテンツウィザードの短縮
- サーバー管理のタグ入力UI
- 設定項目の説明追加
- 監査ログの詳細表示
- その他管理画面の改善

---

## 設計方針

- **「赤ん坊でもわかる」UI**: 技術用語を使わない。すべて平易な日本語で案内
- **PC前提**: モバイル対応不要
- **マウスホバー時のポインター変更**: 編集可能なセルには `cursor: pointer` を適用
- **将来の列追加に耐える設計**: ギミックグルーピング列等の追加を見据える
- **将来の多言語対応に耐える設計**: `LocalizedString` をオプショナルフィールドで拡張可能にする（ko?, zh?）。今回は日英2列のみ実装

### UI用語ルール（全画面共通）

| 技術用語 | UIに表示する言葉 |
|---------|---------------|
| 昇格/プロモート | テンプレートにする |
| インポート | 読み込む |
| バリデーション | 入力チェック |
| フィルター | 絞り込み |
| エクスポート | 書き出す |
| マッチング | 自動で埋める |

## 1. 画面構成

### 全体レイアウト

```
/admin/templates
├── コンテンツ選択ドロップダウン + 概要（イベント数・フェーズ数）
├── ツールバー
│   ├── 「プランをテンプレートにする」ボタン（青）
│   ├── 「スプシから読み込む」ボタン（緑）
│   ├── 「FFLogsから英語名を取得」ボタン（紫）
│   ├── 「未翻訳: N件」カウンター（黄色）
│   └── 「未翻訳だけ表示」トグル
├── スプレッドシート型テーブル
│   ├── ヘッダー: 時間 | フェーズ | 技名(JA) | 技名(EN) | ダメージ | 種別 | 対象 | 削除
│   ├── フェーズ区切り行（青背景）
│   ├── データ行（セルクリックで編集）
│   └── [将来] グループ列を追加可能
└── フッター
    ├── 「変更を元に戻す」ボタン
    └── 「保存する」ボタン（青）
```

### テーブル列の詳細

| 列 | 幅(目安) | 表示 | 編集方法 |
|----|---------|------|---------|
| 時間 | 60px | `M:SS` 形式 | クリック → テキスト入力（`0:13`, `1:30` 形式） |
| フェーズ | 80px | フェーズ名 | クリック → ドロップダウン |
| 技名(JA) | 可変 | 日本語技名 | クリック → テキスト入力 |
| 技名(EN) | 可変 | 英語技名 or 「未翻訳」表示 | クリック → テキスト入力 |
| ダメージ | 80px | 数値（カンマ区切り表示） | クリック → 数値入力 |
| 種別 | 70px | 魔法/物理/回避不可/時間切れ | クリック → ドロップダウン |
| 対象 | 60px | 全体/MT/ST | クリック → ドロップダウン |
| 削除 | 40px | × アイコン | クリック → 確認なしで削除（保存前なら元に戻せる） |

### セルのハイライトルール

| 状態 | 背景色 | テキスト色 |
|------|-------|-----------|
| 通常 | 透明 | 白 |
| 手動変更済み | `rgba(245,158,11,0.06)` 黄色薄 | 黄色 `#f59e0b` |
| 自動入力済み（FFLogs） | `rgba(96,165,250,0.06)` 青薄 | 青 `#60a5fa` + 「自動」ラベル |
| 未翻訳 | 透明 | 破線枠 + 「未翻訳 — クリックで入力」 |
| ホバー | `rgba(255,255,255,0.03)` | - |

### 行の操作

- ホバー: `cursor: pointer` + 行背景のわずかなハイライト
- フェーズ区切り行: `rgba(96,165,250,0.08)` 背景、折りたたみ不要
- 削除: 行右端の × クリックで即削除（未保存状態）。「変更を元に戻す」で復元可能

## 2. プランをテンプレートにする

### フロー

1. ツールバーの「プランをテンプレートにする」ボタンをクリック
2. モーダルが開く:
   - 「プランの共有URLを貼り付けてください」テキスト欄
   - URL入力 → プランデータをAPI経由で取得
   - プレビュー表示: プラン名・作成者・イベント数・フェーズ数
   - 対象コンテンツが自動判定・表示される
3. 「テンプレートにする」ボタンで確定
4. 既にテンプレートがある場合:
   - 「今のテンプレートを置き換えます。よろしいですか？」確認モーダル
   - 確認後、自動バックアップ → 置き換え

### データ変換（プラン → テンプレート）

```
PlanData → TemplateData 変換:

- timelineEvents: そのままコピー
- phases: PlanData.phases（id: string, name: string, endTime: number）→ TemplateData.phases（id: number, startTimeSec: number, name?: string）に逆変換
  - Phase 1 の startTimeSec = 0
  - Phase N (N>=2) の startTimeSec = Phase N-1 の endTime
  - id: "phase_2" → 2（数値に変換）
  - name: "Phase 1\nP1" → "P1"（改行以降を取得、なければそのまま）
- timelineMitigations: 除外
- partyMembers: 除外
- aaSettings: 除外
- schAetherflowPatterns: 除外
- generatedAt: 現在時刻
- source: "plan_promote"
- 将来追加されるフィールド（groupなど）: そのまま含める
```

### API

既存の `/api/admin?resource=templates` の POST メソッドを使用。追加パラメータとして `source: "plan_promote"` を送信。

プランデータの取得は、共有URLからプランIDを抽出し、Firestoreから直接読み取る（管理者APIとして）。

## 3. スプシから読み込む

### フロー

1. ツールバーの「スプシから読み込む」ボタンをクリック
2. モーダルが開く:
   - 大きなテキストエリア: 「スプレッドシートからコピーしたデータを貼り付けてください」
   - スプシからセル範囲をコピー → Ctrl+V で貼り付け
   - タブ区切り（TSV）として自動パース
3. 列の対応付け画面:
   - パースされたデータの最初の5行をプレビュー表示
   - 各列のヘッダーにドロップダウン: 「この列は何ですか？」
     - 選択肢: 時間 / 技名 / ダメージ / 種別 / 対象 / フェーズ / 使わない
   - ヘッダー行の自動推測: 「時間」「技名」「ダメージ」等のキーワードを検出して自動選択
4. 「読み込む」ボタンで確定 → テーブルにデータが入る
   - **この時点ではまだ保存されない**（テーブルで確認・修正してから保存）

### 時間のパース

| 入力 | 変換結果 |
|------|---------|
| `0:13` | 13秒 |
| `1:30` | 90秒 |
| `13` | 13秒 |
| `1:30.5` | 90秒（小数点以下切り捨て） |

### 言語の扱い

- スプシのデータは日本語として `name.ja` に入る
- `name.en` は空のまま → 翻訳欠落として後から埋める

### データ変換

- 種別: 「魔法」→ `magical`、「物理」→ `physical` に自動変換。未指定時は `magical` をデフォルト
- 対象: 「全体」→ `AoE`、「MT」→ `MT`、「ST」→ `ST` に変換。未指定時は `AoE` をデフォルト
- フェーズ: フェーズ列があれば `phases` 配列を自動生成（フェーズ名が変わるタイミングで新フェーズ）
- ID: 各イベントに `tpl_{index}_{ランダム6文字}` 形式で自動生成

---

## 4. 翻訳の欠落を埋める

### 自動マッチング（FFLogsから取得）

1. ツールバーの「FFLogsから英語名を取得」ボタンをクリック
2. モーダルが開く: 「FFLogsレポートのURLを貼り付けてください」
3. URL入力 → 既存の `fetchFightEvents` + `fflogsMapper` を利用して日英ペアを取得
   - `translate: true`（英語）と `translate: false`（日本語）の2回APIリクエスト
   - `ability.guid` をキーにして日英を紐づけ
4. テーブル内の `name.ja` と照合し、一致する英語名を自動入力
5. 自動入力されたセルは青ハイライト + 「自動」ラベル
6. すべて編集可能（間違っていれば上書き修正）

### 手動修正の補助

- 「未翻訳だけ表示」トグル: 未翻訳イベントだけに絞り込み
- ツールバーに「未翻訳: N件」カウンター（黄色表示）
- 全て埋まるとカウンターが「翻訳完了」に変わる（緑表示）

### 将来の多言語対応

- `LocalizedString` 型に `ko?: string`, `zh?: string` をオプショナルで追加
- FFLogs APIは `translate` パラメータで韓国語・中国語にも対応（FF14は4言語対応）
- テーブルに「技名(KO)」「技名(ZH)」列を追加するだけで拡張可能
- 今回は日英2列のみ実装

## 5. データ構造

### テンプレートデータ（既存・変更なし）

```typescript
interface TemplateData {
  contentId: string;
  generatedAt: string;
  sourceLogsCount: number;
  timelineEvents: TimelineEvent[];
  phases: { id: number; startTimeSec: number; name?: string }[];
  source?: string; // "fflogs_auto" | "plan_promote" | "csv_import" | "admin_upload"
}
```

### TimelineEvent（既存・変更なし）

```typescript
interface TimelineEvent {
  id: string;
  time: number;
  name: LocalizedString;
  damageType: 'magical' | 'physical' | 'unavoidable' | 'enrage';
  damageAmount?: number;
  target?: 'AoE' | 'MT' | 'ST';
  warning?: boolean;
  // [将来] group?: LocalizedString; — ギミックグルーピング用
}
```

### LocalizedString（既存・将来拡張予定）

```typescript
// 現在
type LocalizedString = { ja: string; en: string };

// 将来（後方互換あり）
type LocalizedString = { ja: string; en: string; ko?: string; zh?: string };
```

### 編集状態の管理（フロントエンドのみ）

```typescript
interface EditState {
  original: TimelineEvent[];        // 元データ（変更を元に戻す用）
  current: TimelineEvent[];         // 現在の編集データ
  modified: Set<string>;            // 変更されたセルのキー（"eventId:fieldName"）
  autoFilled: Set<string>;          // 自動入力されたセルのキー
  deleted: Set<string>;             // 削除されたイベントID
}
```

## 8. エラー処理

| 操作 | エラーケース | 対応 |
|------|------------|------|
| プランURL貼り付け | URLが無効/プランが存在しない | 「このURLからプランを読み込めませんでした」メッセージ |
| スプシ貼り付け | データがパースできない | 「貼り付けたデータを読み取れませんでした。スプレッドシートからコピーしたデータか確認してください」 |
| FFLogs英語取得 | FFLogs APIエラー | 「FFLogsからデータを取得できませんでした。URLを確認してもう一度試してください」 |
| 保存 | API通信エラー | 「保存に失敗しました。もう一度試してください」 |
| 保存 | 未翻訳が残っている | 保存は許可するが「未翻訳が N 件あります。このまま保存しますか？」確認 |

## 9. 操作フロー全体像

### フローA: スプシから新しいテンプレートを作る

```
1. コンテンツを選ぶ
2. 「スプシから読み込む」をクリック
3. スプシからコピペ → 列を対応付け → 読み込む
4. テーブルにデータが入る（日本語のみ）
5. 「FFLogsから英語名を取得」をクリック
6. FFLogsレポートURL入力 → 英語名が自動入力される
7. 間違いがあれば手動修正、残りの未翻訳を手入力
8. ダメージ・種別・対象も必要に応じて修正
9. 「保存する」で完了
```

### フローB: 自分のプランをテンプレートにする

```
1. コンテンツを選ぶ
2. 「プランをテンプレートにする」をクリック
3. 共有URL入力 → プレビュー確認 → 「テンプレートにする」
4. テーブルにプランのタイムラインが入る
5. 必要に応じて修正 → 「保存する」で完了
```

### フローC: 既存テンプレートの修正

```
1. コンテンツを選ぶ → テンプレートがテーブルに表示される
2. 修正したいセルをクリック → 直接編集
3. 変更箇所が黄色ハイライトで確認できる
4. 「保存する」で完了（自動バックアップあり）
```
