# PS5 ストラテジーボード 貼り付けアシスト — 設計書

作成: 2026-06-29 / 状態: ユーザー承認済み（実装可）
事実調査メモ: `docs/.private/2026-06-26-ps5-remote-play-paste-assist.md`

## 1. 目的・背景

FF14 のゲーム内「ストラテジーボード」(パッチ7.4実装・ゲーム内版 raidplan.io) は、
散会図などを **共有コード** (`[stgy:...]` 形式の数百文字の文字列) でやり取りする。

PS5 プレイヤーは、ネットで拾った共有コードをゲーム内の「図面一覧 → 新規作成 →
共有コードから作成」の **URL 貼り付け欄** に貼る必要がある。しかし:

- PS5 のテキスト入力は **1回あたりの文字数上限が小さい** (確認できた一次ソースでは
  「マクロ180文字」)。コードは数百文字あるため一度に貼れない。
- 従来の回避策は「コードを2〜3分割してマクロに入れ、図面読み込み欄でマクロを順番に
  貼り継ぐ (バケツリレー方式)」という苦行 (出典: ロドスト日記)。

本ツールは、**スマホで共有コードを貼ると安全な長さに自動分割し、上から順にコピー
できる** ようにして、PS Remote Play (スマホアプリ) 経由でゲーム内貼り付け欄に
少しずつ貼る作業を一発で楽にする。

### 確認済みの事実 (一次ソース)

- **バックグラウンドストリーミング = 最大5分**。設定(右上⚙)→ON。一度ONで次回も有効。
  (PlayStation 公式: https://remoteplay.dl.playstation.net/remoteplay/lang/en/1100003.html)
  → アプリ切替後も接続維持され、LoPo↔PS App を往復してコピペできる。
- **PS5 の貼り付け上限**: 確認できたソースは「マクロ180文字」。ただしこれはマクロ欄の
  上限であって、リモプでスマホのクリップボードから直接「URL貼り付け欄」に貼る場合の
  上限を明記した一次ソースは見つからなかった。**環境差の可能性があるため上限値は
  ハードコードせず、ユーザー調整可能とする** ([[feedback_no_hardcoding]])。
- 貼り先は同じ1つの「URL貼り付け欄」に **継ぎ足し** (ユーザー実体験で確認)。
  → 分割は機械的スライスで安全 (繋げれば必ず元コードに一致)。

## 2. スコープ

### やること
- 独立ページ (`/stgy`)。**ログイン不要・サーバー不要・完全クライアント側**。
- 共有コードを貼る → 安全マージンで N 分割 → 上から順にコピー (進捗状態管理)。
- スマホ専用 1 画面フロー型 UI。
- 入口: 軽減アプリの「⋯」メニュー、LP の「Tools」セクション。

### やらないこと (YAGNI / 将来)
- コードの **配布** (刻んだ断片を仲間に共有する URL 発行等) → サーバーが必要なので対象外。
- コード中身の解釈・検証 (図面プレビュー等)。本ツールは中身を一切読まない。
- PC 向け専用レイアウトの作り込み (PC で開いても破綻しない程度に留める)。

## 3. ルーティング

- `src/App.tsx` に新規ルート `<Route path="/stgy" element={...} />` を追加 (既存の独立ページ
  /support /privacy 等と同列)。重い依存は無いので遅延ロードは必須でないが、既存に倣い
  `lazy()` + `Suspense` でも可。
- ページは認証ガード対象外 (誰でもアクセス可)。
- `useCanonicalUrl('/stgy')` 等、既存ページに倣って canonical / title / description を設定。

## 4. ページ UI (スマホ専用・1画面フロー型)

縦 1 画面で上から下へ「やること」が流れる。状態に応じて下のステップが出現する。

### ① 準備 (初回だけ・折りたたみ)
- 「PS Remote Play 設定(右上⚙) → バックグラウンドストリーミングを ON」の手順を案内。
- 一度開いて閉じた / 「OK・理解した」を押すと **localStorage に記録し次回から畳む**。
  (キー例: `stgy_bgstream_ack`)
- 「最大5分つながったまま」である旨を1行添える。

### ② コードを貼る
- 1つの textarea。プレースホルダに `[stgy:...]` の例。
- 入力 (onChange) で即座に分割を再計算 → ③ を表示。
- 空白除去後が空なら ③ 非表示。

### ③ 上から順にコピー
- 分割された断片を縦リストで表示。各行:
  - チェックボックス風アイコン (未コピー = ⬜ / コピー済 = ✅)
  - 「①コピー」「②コピー」… のラベル付きボタン
  - 断片プレビュー (先頭〜末尾を省略表示。全文は出さなくてよい)
- ボタン押下で `navigator.clipboard.writeText(chunk)` → そのボタンを **グレーアウト＋✅**、
  進捗カウント +1、トースト (既存 `showToast` 流用可)。
- 進捗バー / カウンタ「N / M コピー済」。全部コピーで「完了 🎉」表示。
- 「やり直す」ボタン: コピー済み状態をリセット (textarea は保持)。

### 詳細設定 (折りたたみ・既定で閉じる)
- 「1回の文字数」スライダー (range 80〜180、既定 **170**)。
- 変更すると分割を再計算 (コピー済み状態はリセット)。
- 普段は触らせない。メイン動線を「貼る → 順にコピー」の2手に保つ。

## 5. 分割ロジック (純関数・テスト対象)

`splitStrategyCode(raw: string, chunkSize: number): string[]`

1. **正規化**: `raw` から全空白文字 (スペース/改行/タブ等、`/\s/g`) を除去。
   - 理由: フォーラム等からのコピペで改行が紛れ込むため。stgy コードは内部に空白を
     含まない連続トークンである前提。**実装時にこの前提を実コード仕様で1回確認する**
     (例の `[stgy:...]` に空白が無いことは確認済み。`+ - _ =` 等は含む可能性あり=除去しない)。
2. **スライス**: 正規化後の文字列を `chunkSize` 文字ごとに機械的に分割。最後は余り。
   - `for (i=0; i<s.length; i+=chunkSize) chunks.push(s.slice(i, i+chunkSize))`
3. **保証**: `chunks.join('') === normalized`。区切り位置の特別配慮は不要。
4. 空入力 → `[]`。

別ファイル `src/lib/strategyCode.ts` (純関数) に置き、UI から分離してテスト可能にする。

### テスト (TDD)
- 空文字 → `[]`
- 改行・空白混入 → 除去されて連結が元の非空白列に一致
- ちょうど割り切れる長さ / 余りが出る長さ
- `join('')` が正規化後文字列に完全一致 (ラウンドトリップ不変条件)
- chunkSize 境界 (80 / 170 / 180)

## 6. 状態管理

ページローカル state のみ (グローバルストア不要):
- `raw: string` (textarea 内容)
- `chunkSize: number` (既定 170)
- `copiedSet: Set<number>` (コピー済み断片の index)
- 分割結果 `chunks` は `raw` / `chunkSize` から派生 (`useMemo`)。
- `chunkSize` 変更時・`raw` 変更時に `copiedSet` をクリア。

## 7. 入口 (エントリポイント)

### 7.1 軽減アプリ「⋯」メニュー
- `src/components/HeaderToolsMenu.tsx` に項目を1つ追加:
  - ラベル: i18n キー (例 `ui.ps5_strategy_paste`)
  - アイコン: lucide から適切なもの (例 `ClipboardPaste` / `Copy`)
  - 動作: `/stgy` を **別タブで開く** (`window.open('/stgy', '_blank')`)。
    軽減の作業状態を壊さないため新タブ。
- readOnly 状態でも有効 (ツール自体は軽減データ非依存)。

### 7.2 LP「Tools」セクション
- `src/components/landing/LandingPage.tsx` の Projects セクションに `ProjectCard` を追加:
  - `number="03"`、`title` = i18n、`desc` = 簡単な説明、`badge="UTILITY"`
  - `onClick={() => navigate('/stgy')}`
- 01 軽減・02 ハウジングと同じ ProjectCard 形式で自然に並べる。

## 8. デザイン

- LoPo 本体トンマナ準拠 (`.claude/rules/ui-design.md` / `DESIGN.md`): **白黒＋機能色のみ**、
  色・フォントサイズは **トークン経由** (`--color-*` / `--font-size-*` / `text-app-*`)、
  glass-tier 利用。Inter 禁止・AIグラデ禁止。ハウジングルールは非適用。
- スマホ幅前提。PC で開いた場合は中央に最大幅 (例 `max-w-[480px] mx-auto`) を設けて破綻回避。
- ボタン: `active:scale-95`、`transition-all duration-200` 等の既存規約。
- コピー済みボタンのグレーアウトは機能色でなくミュート (`text-app-text-muted` / `opacity`)。
  完了 ✅ は OK 系=緑、で既存に倣う。

## 9. 多言語

- UI テキストは全て i18n キー経由 (`.claude/rules/i18n.md`)。
- **ja / en を完備**。`ko` / `zh` はキーを追加し暫定訳 (または ja/en 相当) を入れて
  4言語 parity を維持 ([[feedback_locale_json_textual_edit]] に従い該当ブロックのみ textual 編集)。
- 英語モードで崩れないか確認。

## 11. 受け入れ基準

1. `/stgy` にアクセスでき、ログイン不要で表示される。
2. 実際の `[stgy:...]` コードを貼ると 170 文字ごとに分割され、断片数が正しい。
3. 各断片を順にコピーでき、押すとグレーアウト＋✅、進捗が進む。全コピーで完了表示。
4. 分割断片を順に連結すると元コード (空白除去後) に完全一致する。
5. スライダーで文字数を変えると分割が再計算され、コピー済み状態がリセットされる。
6. 「⋯」メニュー / LP カードから `/stgy` に到達できる。
7. ja / en で表示が崩れない。`npm run build` と `vitest run` が通る
   ([[feedback_vercel_tsc_strict]])。
8. **実機 (スマホ + PS Remote Play)**: 貼り付けアシストで実際にゲーム内ストラテジーボード
   読み込みが成功する (ユーザー確認)。
