## 確定した設計方針（2026-03-24）

### サイドバーの操作フロー
- コンテンツクリック = 即編集開始。「テンプレート」という概念はユーザーに一切見せない
- テンプレートがあるコンテンツ → 裏で読み込み → 自動でプランとして保存 → 即編集
- テンプレートがないコンテンツ → 空プランを自動作成 → 即編集
- 同コンテンツに既存プランがある場合 → 即オープン（重複作成しない）
- 同コンテンツ1件以上 → サブアイテム展開。末尾に「+ 追加」行
- プラン名はサブアイテムのペンシルボタンでインライン編集（ヘッダーではダブルクリック）
- コンテンツ未紐付きプラン（自由作成）はレベル別「カスタム」セクションに表示
- 「新規作成」ボタンはレベルタブの上に控えめ配置（極・滅・ヴァリアントダンジョン等で使用）

### 保存の仕組み（2層構造・確定方針）
- **1層：localStorage（常時・無音・無料）**
  - 30秒間隔 + プラン切替時 + ページ離脱時に自動保存
  - ユーザーは「保存」を一切意識しない。トースト通知なし
  - ネットもデータベースも使わない
- **2層：Firestore（間引き・ログインユーザーのみ）**
  - 保存タイミング：タブ切替 / ページ離脱 / プラン切替 / 3分以上編集時に1回
  - 前回から変更があった場合のみ書き込む（dirty flag方式）
  - 1セッションあたり5〜10回に抑え、DAU 2,000〜3,000人で無料枠内に収める
  - 未ログインユーザーはlocalStorageのみ（Firestore書き込みゼロ）

### 共有の仕組み
- ヘッダーのリンクコピーまたはX共有ボタン
- POST /api/share → Firestoreに保存 → nanoid 8文字の短縮URL
- /share/:shareId で閲覧 + 「自分の軽減表にコピー」ボタン
- 軽減内容が変わったら共有URLキャッシュをクリア（新しいURLを発行）

### デザイン方針
- まず全体を白黒ベースに統一 → あとでアクセントカラーを入れる
- ヘッダーのヒーロータイトル: コンテンツ名は省略なし、ユーザーが付けたプラン名の方を省略

### プラン管理方針
- プラン最大件数は5件
- 古いプラン（30日以上未編集、もっと短くてもいい）はFirestoreに退避して圧縮 → 次回オープン時に復元（遅くなるがデータ節約）
