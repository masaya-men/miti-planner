## 1. ゴールと非ゴール

### ゴール
- 表の「共有」から **「コピーを配る(既存)」と「一緒に編集(新規)」を最初に2択**で選ばせる(B案)。
- 「一緒に編集」で **ルーム(roomToken)を発行**し、オーナーの表を共同編集モードで接続(`startCollabSession(roomToken)`)。
- **オーナーパネル**でリンク表示・コピー・**入れる人数設定**・**失効**・**再発行**。
- 表のツールバーに **共同編集状態を示す常設チップ**を置き、そこからいつでもパネル→失効へ到達できる(業界水準: Figma/Google ドキュメントはツールバー常設の Share/presence から失効する)。
- 既存の**コピー配布共有(ShareModal の従来機能)・1人モードは一切壊さない**。

### 非ゴール (後続に送る)
- **ジョイナーの入室・一時ビュー**(⑤-3b)。⑤-3a 時点では他人は実際には入れない(土台の roomToken 経路を通すまで)。
- **注意の初回フルモーダル・部屋内の常時赤バナー・未ログイン=閲覧のみのログインゲート**(⑤-3c)。⑤-3a のオーナーパネル内警告文は出すが、ジョイナー向け UI とサーバ側編集認証は ⑤-3c 以降。
- **編集席8/閲覧席20 の席種別カウント**(presence=段取り④)。⑤-3a は総参加数の単一上限のみ(§3)。
- **実データ往復の2ブラウザ E2E**(⑤-3d)。⑤-3a の実機確認はオーナー1人での発行→チップ→人数→失効まで。

## 2. 人数モデル (v1 の確定事項)

ブレストで確定。**v1 は編集席/閲覧席を別々に数えない**(席種別は presence=④と一緒に作る)。v1 にあるのは「部屋に入れる**総人数の単一上限**」だけ。

- **既定 8 人**(零式/絶のフルパーティ1組)。オーナーが **1〜20 人**で設定可。
- **システム上限 = 20**(従来コードは 28 = 8編集+20閲覧 だった。v1 は総上限 20 に統一する → `SYSTEM_MAX_PARTICIPANTS` を 28 から **20** へ変更)。
- **編集/閲覧の区別はログインで決まる**(v1): 「ログインした人が編集 / していない人は閲覧だけ」。ただし**閲覧専用の別枠カウント(編集8とは別に閲覧20)は持たない**(presence=④で追加)。注意文もこの v1 の実態どおりに書く(まだ無い席種別を約束しない)。

> ⚠ `SYSTEM_MAX_PARTICIPANTS` は worker `workers/collab/src/collabCapacity.ts` と api `api/collab/_roomLogic.ts` の**両方**にある(別ランタイムで複製)。両方を 20 に変更する。`DEFAULT_MAX_PARTICIPANTS = 8` は不変。

## 3. オーナー体験 (UX 確定)

### 3.1 入口 = 共有の2択 (B案)
- 表の「共有」を押すと、まず **「コピーを配る」「一緒に編集する」の2枚カード**。「コピー＝相手が自分用に持つ」「一緒に編集＝同じ表をいじる」は性質が全く違い、流出時に本物が書き換わるリスクもあるため、**最初に意図をはっきり選ばせて事故を防ぐ**。
- 「コピーを配る」= 既存の共有フロー(OGPプレビュー + 共有URLコピー)へ。**無改修**。
- 「一緒に編集」= 未ログインなら既存のログイン導線(LoginModal)へ。ログイン済なら下記オーナーパネルへ。

### 3.2 「一緒に編集」選択時の挙動
- `/api/collab/room`(action=create・冪等)を呼び roomToken を取得(既にあれば再利用)。
- `startCollabSession(roomToken)` で自分の表を部屋に接続(ライブ化)。クライアント Firestore 自動保存は抑制(既存 `_collabActive` 経路・保存は DO が代表)。
- **オーナーパネル**を表示。

### 3.3 オーナーパネル (確定レイアウト)
モック準拠(`.superpowers/brainstorm/.../revised-owner.html`)。上から:
- **ヘッダ**: 「共同編集」+ 右に**実際の参加人数**「1人(あなただけ)」(誤解を招く"ライブ編集中"表記は使わない・実数を出す)。
- **警告ブロック(赤=危険の機能色)**: 確定文言 ——
  > このリンクを知っている人は誰でもあなたの表を直接書き換えられます(元に戻せません)。固定メンバー内だけで使い、SNS など外部に公開・転載しないでください。
- **情報ブロック(中立)**: 確定文言 ——
  > 最大 20人まで一緒に入れます(ログインした人が編集 / していない人は閲覧だけ)。この表はあなたのアカウントに残ります(共同編集はあなたの表をそのまま一緒に編集しているだけ)。いつでも下の「リンクを失効」で共同編集を無効にできます。
- **共同編集リンク** + コピーボタン。
- **入れる人数**: ステッパー(− N ＋)、既定 8、1〜20。変更時 `/api/collab/room`(action=set-max)。
- **アクション**: 「リンクを作り直す」(reissue) / 「リンクを失効」(revoke・赤)。

> **「表はアカウントに残る」は事実確認済**: [api/collab/_saveHandler.ts:37-46](../../../api/collab/_saveHandler.ts#L37-L46) は共同編集の保存を `plans/{planId}`(=オーナー本人の表)の `data.timelineMitigations` 部分更新としてのみ行う。別物へコピー・移動しない。

### 3.4 表ツールバーの常設チップ (業界水準)
- **通常時**: いつもの「↗ 共有」ボタン(押すと 3.1 の2択)。
- **共同編集オン時**: **発行後ずっと**、緑チップ「● 共同編集中 · N人」+ 参加者アバターをツールバーに表示。一目で「この表は今ライブ」と分かる。**クリックでオーナーパネル**(→そこで失効)。人が増えると人数とアバターが増える(在室数は worker の `/count` から取得・presence の本格表示は④)。
- 失効が「奥のボタン」でなく常設の入口から手軽に到達できる(ユーザー指摘 + Figma/Google ドキュメント準拠)。

## 5. データの流れ (サーバは実装済・⑤-3a は結線のみ)

- 発行/人数/失効/再発行 = `/api/collab/room`(⑤-2a 実装済・ID トークン認証→`plans.ownerId===uid`照合)。
- 接続 = roomToken で worker(`lopo-collab`)に WebSocket(⑤-1 解決・⑤-2b 満員拒否・本番稼働済)。
- 保存 = DO の onSave が `/api/collab/save`(roomToken)経由で `plans/{planId}` へ(③ 実装済)。
- → ⑤-3a で新規に書くサーバコードは無い(`SYSTEM_MAX` 値変更を除く)。
