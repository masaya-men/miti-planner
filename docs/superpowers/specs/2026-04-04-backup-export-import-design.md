# 軽減表バックアップ機能（エクスポート/インポート）設計書

**日付**: 2026-04-04
**ステータス**: レビュー待ち

---

## 概要

ユーザーが全軽減表をJSONとしてバックアップ（エクスポート）し、復元（インポート）できる機能。

## UIレイアウト

### サイドバー下部

```
[プラン一覧スクロールエリア]
────────────────────────────
  バックアップ / 復元 ボタン
────────────────────────────
```

- プラン一覧スクロールエリアとの間を罫線で区切る
- 既存のglass-tierスタイルに合わせる

### 「縦の…」メニュー廃止

- 既存の3点メニューをゴミ箱アイコンに置き換え
- 削除の2段階確認は維持（タップ→確認→削除）
- CSVダウンロード機能は削除

---

## エクスポート（バックアップ）フロー

### 手順

1. ユーザーが「すべての軽減表をバックアップする」ボタンをタップ
2. **編集中のプランをフラッシュ**: 現在開いているプランの編集状態（`useMitigationStore.getSnapshot()`）を`updatePlan`でプランデータに保存する。これをしないと直近の編集がバックアップに含まれない。
3. **ログイン中の場合**: `manualSync`（PUSH→PULL）を実行して全デバイスのデータを最新化
4. **未ログインの場合**: localStorageのデータをそのまま使用
5. モーダルが開く

### モーダル内容

- **説明文**: バックアップの目的と保管方法を明示（「このテキストをメモ帳やスプレッドシートに貼り付けて保管してください」等）
- **JSONテキストエリア**: 読み取り専用、全データのJSON文字列を表示
- **コピーボタン**: クリップボードにJSONをコピー（成功トースト表示）
- **ダウンロードボタン**: `.json`ファイルとしてダウンロード

### エクスポートデータ形式

```json
{
  "version": 1,
  "exportedAt": "2026-04-04T12:00:00.000Z",
  "planCount": 5,
  "plans": [ ...SavedPlan[] ]
}
```

- `version`フィールドで将来のフォーマット変更に対応
- `planCount`は復元時の確認表示用

## インポート（復元）フロー

### 手順

1. ユーザーが「バックアップから復元する」ボタンをタップ
2. モーダルが開く

### モーダル内容

- **説明文**: 復元方法の説明（「バックアップしたテキストを貼り付けるか、ファイルを選択してください」）
- **テキストエリア**: JSONを貼り付け
- **ファイル選択ボタン**: `.json`ファイルを選択（選択時にテキストエリアに内容を反映）
- **「復元する」ボタン**

### 確認フロー（しつこく確認する）

1. **バリデーション**: JSONをパースし、`version`フィールドと`plans`配列の存在を検証。不正な場合はエラー表示。
2. **確認ダイアログ1**: 「現在の軽減表はすべて上書きされます。この操作は取り消せません。」→ 続行 / キャンセル
3. **確認ダイアログ2**: 「本当に復元しますか？ 現在の○件の軽減表が、バックアップの○件に置き換わります。」→ 復元する / やめる
4. 復元実行
5. **ログイン中の場合**: 即時PUSH（`forceSyncAll`）でFirestoreに反映
6. 完了トースト表示

### 復元ロジック

- **マージ方式**: バックアップのプランを既存データに上書きマージする（既存プランを削除しない）
  - 同一IDが存在 → バックアップ版で上書き
  - 同一IDが存在しない → 新規追加
  - バックアップに含まれない既存プラン → そのまま残す
- **ownerId書き換え**: バックアップ内の全プランのownerIdを現在のログインユーザーのuidに更新する（未ログインなら`'local'`）。別アカウントや別端末で復元する場合にFirestoreの権限エラーを防ぐため。
- Zustand storeを更新（`setPlans`）
- ログイン中は即時Firestore同期（`forceSyncAll`）
- 現在開いているプランがバックアップで上書きされた場合、表示を再読み込み
