# スマホUX改善 設計書

> 作成日: 2026-04-07
> ステータス: 承認待ち

---

## 概要

スマホ表示の「小さくて見にくい・操作しにくい」問題を5つの改善で解決する。
PC表示には一切影響を与えない設計とする。

---

## 改善項目と優先順位

| 順序 | 項目 | 概要 |
|------|------|------|
| A | MobileTimelineRow（2行カード） | 新規コンポーネント。行高80px、Apple風デザイン |
| B | FABメニュー | ヘッダーボタン6つを右下FABに集約 |
| D | 同期トースト通知 | 手動同期・タブ復帰時にトースト表示 |
| C | PC同期テキスト | SyncButtonにテキストラベル追加（PC専用） |
| E | iOSズーム防止 | モバイルinputのfont-size 16px保証 |

## A. MobileTimelineRow（2行カードレイアウト）

### PC影響ゼロの実装方針: 完全分離

**TimelineRow.tsx は一行も変更しない。** 新規ファイル `MobileTimelineRow.tsx` を作成し、Timeline.tsx で条件分岐してレンダリングする。

```tsx
// Timeline.tsx
const rowHeight = isMobile ? 80 : 50;
const pixelsPerSecond = rowHeight;

// レンダリング
{isMobile ? (
  <MobileTimelineRow time={time} top={currentY} ... />
) : (
  <TimelineRow time={time} top={currentY} ... />  // 既存ファイル（完全に無変更）
)}
```

**Timeline.tsx の変更範囲:**
- `pixelsPerSecond` を `isMobile ? 80 : 50` に分岐（PCは従来通り50）
- レンダリング部分の条件分岐追加
- 既存のPC側ロジック・値は一切変更しない

**安全性の根拠:**
- TimelineRow.tsx の diff = 0行
- PC幅（≥768px）では `isMobile = false` → 従来と完全に同じコードパス
- 問題発生時は MobileTimelineRow.tsx を削除するだけで即復旧

### 2行カードレイアウト

行高: **80px**（現在50px → 1.6倍）

```
┌──────┬──────────────────────────────────────────┐
│      │ 1:23  ■ ダークホーリー              AoE  │  上段: 時間+種別+攻撃名+対象
│ ラベル│ 180k → 92k ▼49%          [SH][WT][AL]  │  下段: ダメージ+軽減%+スキルアイコン
├──────┼──────────────────────────────────────────┤
│      │ 1:30  ■ シャドウアイ                 MT  │
│      │ 250k → 198k ▼21%                  致死  │
└──────┴──────────────────────────────────────────┘
```

**フェーズ/ラベル列:** 左端24pxで現状維持。フェーズがある場合はフェーズ表示、ない場合はラベル表示（既存ロジック踏襲）。

### 2イベント行の分離

現在: 1つの50px行に2イベントを詰め込み
改善: **同じ時間の2イベントを各80pxの独立カードに分離**

```
┌──────┬──────────────────────────────────────────┐
│      │ 1:35  ■ ホーリーブレス              AoE  │  1つ目のイベント（通常カード）
│      │ 140k → 85k ▼39%               [SH]      │
├──────┼──────────────────────────────────────────┤
│      │ 〃    ■ テイルスラップ               MT  │  2つ目のイベント（時間は「〃」）
│      │ 120k → 67k ▼44%                         │  背景を微妙に変えて連続を示唆
└──────┴──────────────────────────────────────────┘
```

- 2つ目の時間欄は「〃」で同時刻を示す
- 背景色を `rgba(255,255,255,0.015)` で微妙に変える
- 各カードを個別タップ → 個別にモーダルが開く
- **Timeline.tsx への影響:** 2イベント時の行高は `rowHeight * 2`（160px）。timeToYMapの計算で対応。

### Apple風デザイン要素

すべての色はデザイントークン（CSS変数）経由とし、将来のカラーパレット変更に対応可能にする。

- **種別アイコン:** 円 → 角丸正方形（squircle、border-radius: 4px）
- **対象バッジ:** pill型（大きめ border-radius: 6px）
- **ボーダー:** 極薄（border-app-border、opacity控えめ）
- **色階層:** `text-app-text` のopacity違いで奥行き表現
  - 主テキスト: opacity 1.0
  - 時間: opacity 0.5
  - 素ダメージ: opacity 0.3
  - 軽減%: opacity 0.25

### アニメーション

- **タップフィードバック:** `scale(0.98)` + 背景ハイライト（150ms ease-out）
- **軽減追加時:** 新アイコンが右からスライドイン（framer-motion spring: stiffness 300, damping 25）
- **致死→安全:** ダメージ色が赤→緑に `transition-colors duration-300`
- **初回表示:** staggered fade-in（各行50msずらし）

### デザイントークンの使用

ハードコーディングされた色を一切使わず、すべて既存のCSSカスタムプロパティ経由にする。

| モックアップでの色 | 実装で使うトークン |
|---|---|
| `#000` | `bg-app-bg` (`--color-bg-primary`) |
| `#fff` | `text-app-text` (`--color-text-primary`) |
| `rgba(255,255,255,0.5)` | `text-app-text-sec` (`--color-text-secondary`) |
| `rgba(255,255,255,0.3)` | `text-app-text-muted` (`--color-text-muted`) |
| `rgba(255,255,255,0.06)` | `border-app-border` (`--color-border`) |
| `#34D399` / `#4ade80` | `text-green-400`（既存Tailwind） |
| `#F87171` / `#ef4444` | `text-red-400`（既存Tailwind） |

将来 `--color-bg-primary` を `#000` → `#0D1117` 等に変更すれば、モバイル版も含め全体が一括で切り替わる。

## B. FABメニュー

### 概要

MobileHeaderの右側3ボタン（テーマ・同期・言語）を削除し、ナビゲーション機能3つと合わせて右下FABに集約する。

### FAB構成（6項目）

展開時、上方向に表示。区切り線でナビゲーション系と設定系を分離。

**ナビゲーション（上段）:**
| 項目 | アイコン | タップ時の動作 |
|------|---------|--------------|
| フェーズ | リストアイコン | ボトムシートでフェーズ一覧 → タップでジャンプ |
| ラベル | タグアイコン | ボトムシートでラベル一覧 → タップでジャンプ |
| 攻撃名検索 | 検索アイコン | ボトムシートで検索input → マッチした攻撃へジャンプ |

**設定（下段）:**
| 項目 | アイコン | タップ時の動作 |
|------|---------|--------------|
| 同期 | クラウドアイコン（青） | 即時同期実行（SyncButton相当） |
| 言語 | 地球アイコン | 言語選択リスト表示 |
| テーマ | 太陽/月アイコン | ダーク/ライト即時切替 |

### 動作

- **閉じた状態:** 右下に52pxボタン（「⋯」アイコン）
- **タップで展開:** 上方向にspring アニメーションで6項目 + ラベル出現
- **背景オーバーレイ:** 半透明黒、FAB外タップで閉じる
- **ジャンプ:** 既存の `handleNavJump(time)` をそのまま利用（PC側影響なし）

### MobileHeader変更

右側の3ボタン（テーマ・同期・言語）を削除。ヘッダーは:
- 左: LoPoロゴ
- 中央: コンテンツ名/プラン名（表示領域が広がる）

## C. PC同期テキスト

### 概要

PC版の SyncButton にテキストラベルを追加し、アイコンだけでは伝わりにくい状態を明示する。

### ステータス表示

| 状態 | アイコン | テキスト | テキスト色 |
|------|---------|---------|----------|
| saved | チェックアイコン（緑） | 保存済み | `text-app-text-muted`（灰色） |
| pending | 更新アイコン | 同期する | `text-app-text-muted`（灰色） |
| syncing | クラウドアイコン（パルス） | 同期中... | `text-app-text-muted`（灰色） |
| error | 警告アイコン（赤） | エラー | `text-red-400` |

- PC専用の変更（SyncButton.tsx のテキスト追加のみ）
- スマホでは FAB 内に同期ボタンがあるためテキスト不要

## D. 同期フィードバックのトースト通知

### 概要

同期の結果をトースト通知でユーザーに伝える。既存の `Toast.tsx` / `showToast()` をそのまま活用。

### 通知タイミング

| イベント | 通知する？ | メッセージ | タイプ |
|---------|----------|----------|--------|
| 手動同期 PUSH 成功 | する | クラウドに保存しました | success |
| 手動同期 PUSH 失敗 | する | 同期できませんでした | error |
| タブ復帰 PULL 成功 | する | 最新データを取得しました | success（青寄り） |
| タブ復帰 PULL 失敗 | する | データ取得に失敗しました | error |
| 自動同期（5分クールダウン） | しない | — | — |

### Toast.tsx の拡張

- 現在のタイプ: `'success' | 'error'`
- 追加: `'info'`（PULL成功用、青アイコン）
- 表示位置: 画面下中央（現行通り）。FABと被らないよう `bottom` を調整（FABの上に表示）
- 自動消滅: 3秒（現行通り）
- PC/スマホ両方で表示

## E. iOSズーム防止

### 概要

iOS Safari は font-size 16px 未満の input にフォーカスすると自動ズームする。モバイルの input/select 要素を 16px 以上に保証する。

### 対応方針

- MobileTimelineRow 自体に input はないため直接影響なし
- FABの攻撃名検索 input は最初から 16px で作成
- A（MobileTimelineRow）完了後にモバイル全体の input を監査して一括対応
- 対象: 軽減追加モーダル内の input/select、プラン名入力、等

## 実装順序

```
A. MobileTimelineRow  ←  最もインパクト大、独立して実装可能
    ↓
B. FABメニュー        ←  Aの後にヘッダー整理する流れで自然
    ↓
D. 同期トースト        ←  A・Bと独立、既存Toast.tsx拡張のみ
    ↓
C. PC同期テキスト      ←  PC専用、SyncButton.tsx変更のみ
    ↓
E. iOSズーム防止       ←  A完了後にinput監査
```
