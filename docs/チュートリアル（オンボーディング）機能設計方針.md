# チュートリアル（オンボーディング）機能設計方針

## 1. 目的
「軽減表ツール」を初めて利用するユーザーに対し、操作の基本導線（コンテンツ選択 → パーティ編成 → 軽減配置）を実際に操作させながら学習させる強制チュートリアルを実装する。

## 2. 発火条件と前提システム
* **Google認証・ユーザーDBとの連携**:
  認証したユーザーのデータベース（Firestore等を想定）に `hasCompletedMitigationTutorial: boolean` というフラグを持たせる。
* **トリガー**:
  トップページ（ポータル）から「軽減表ツール」を選択し、新規の空の軽減表が開かれた際、上記フラグが `false` の場合にのみ自動発火する。
  ※ゲストユーザー（未ログイン）の場合は、ローカルストレージ（`localStorage`）でフラグを管理する。

## 3. UI/UXの仕様（ハイライト＆強制操作）
画面全体に半透明の黒いマスク（オーバーレイ）をかけ、**「いま操作してほしい箇所」だけを明るくくり抜き（スポットライト）、それ以外の操作をブロックする**仕様とする。

### 推奨技術スタック
フルスクラッチで作ることも可能だが、座標計算が複雑になるため、Reactのチュートリアル定番ライブラリである `react-joyride` をベースにカスタマイズする、もしくは Zustand で `TutorialStore` を作成し独自のハイライトコンポーネントを実装する方針をとる。

## 4. チュートリアルのステップ定義

チュートリアルは以下のステップで状態遷移（State Machine）を管理し、ユーザーが正しい操作を行わない限り次のステップへ進めない「強制進行型」とする。

* **Step 1: コンテンツ（タイムライン）の用意**
  * **ハイライト対象**: 「FFLogs Import」ボタン、または「コンテンツ選択」ボタン。
  * **説明テキスト**: 「まずは戦うボスを選びましょう。FFLogsのURLを入力するか、プリセットから選択してください。」
  * **完了条件**: タイムラインにイベント（敵の攻撃）が1つ以上読み込まれること。

* **Step 2: パーティ編成**
  * **ハイライト対象**: 「パーティ編成（Party Comp）」ボタン。
  * **説明テキスト**: 「次に、一緒に戦う仲間（ジョブ）を設定しましょう。クリックして編成を開いてください。」
  * **完了条件**: パーティ編成モーダルが開き、ジョブが変更・確定されること。

* **Step 3: 軽減の追加**
  * **ハイライト対象**: タイムライン上の任意の空きセル（例: 最初の全体攻撃の行のヒーラーの列）。
  * **説明テキスト**: 「敵の攻撃に合わせて軽減を追加しましょう！明るくなっているマスをクリックしてください。」
  * **完了条件**: セルがクリックされ、軽減選択ポップアップから何らかの軽減が追加されること。

* **Step 4: 完了と祝福**
  * **表示**: 画面中央に完了ダイアログ。
  * **説明テキスト**: 「完璧です！これで基本的な使い方はマスターしました。オートプランなどの便利機能もぜひ試してみてください！」
  * **完了処理**: ユーザーDB（またはローカルストレージ）の `hasCompletedMitigationTutorial` を `true` に更新し、チュートリアル状態を解除する。

## 5. 状態管理（Zustand）の設計
`useTutorialStore` を新設し、現在のステップと進行状態を管理する。

```typescript
interface TutorialState {
  isActive: boolean;
  currentStep: number;
  startTutorial: () => void;
  nextStep: () => void;
  completeTutorial: () => void;
}
```
※既存の `useMitigationStore` のアクション（イベント追加や軽減追加など）の中で、チュートリアル進行条件を満たしたか監視し、自動で `nextStep` を発火させる仕組みを構築する。
