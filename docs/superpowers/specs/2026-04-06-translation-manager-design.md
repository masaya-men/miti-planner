# 翻訳管理画面 設計書

## 概要

管理画面に「翻訳」タブを追加し、ゲーム内データの多言語翻訳（ja/en/zh/ko）を一元管理する。既存の管理画面（スキル追加モーダル等）には一切触れない。

## 背景

- 現在zh/koが不足しているゲームデータが約700項目ある
- スキル名、コンテンツ名、攻撃名、フェーズ名等が散在しており、翻訳作業の全体像が見えない
- 外部の翻訳者にデータを渡して返してもらう仕組みがない

## 対象データ

| カテゴリ | 件数 | Firestoreパス | 現状 |
|---------|------|--------------|------|
| スキル名 | ~90 | `master/skills` | ja/enのみ |
| コンテンツ名 | 63 | `master/contents` | ja/enのみ |
| シリーズ名 | 13+ | `master/contents`（新規） | ja/enハードコード → Firestore化 |
| カテゴリ名 | 5 | `master/config` | ja/enのみ |
| レベル名 | 4 | `master/config` | ja/enのみ |
| 攻撃名 | 500+ | `templates/{contentId}` | ja/enのみ |
| フェーズ名 | ~10 | `templates/{contentId}` | 英語のみ（LocalizedString化が必要） |
| ギミックグループ名 | 可変 | `templates/{contentId}` | ja/enのみ |

**対象外:** ジョブ名（i18n経由で4言語対応済み）、ロール名（英語のまま）、ダメージ種別（i18n対応済み）、UIテキスト全般（localeファイルで対応済み）

## 設計方針

### 管理画面の思想

- 非エンジニアが、管理の知識ゼロ・勉強なしで、間違わず安全に操作できること
- 既存の管理画面には一切触れない（スキル追加モーダル等はそのまま）
- 翻訳作業は「翻訳タブ」に集約し、「翻訳はここ」と明確にする

### フォールバック

中韓が未入力の場合の表示優先順位:
1. Firestoreにzh/koがあればそれを表示
2. なければ英語（en）を表示
3. 英語もなければ日本語（ja）を表示

## 画面構成

### 翻訳タブのレイアウト

```
┌─────────────────────────────────────────────────┐
│  カテゴリ切り替え:                                │
│  [スキル] [コンテンツ] [攻撃名] [フェーズ] [その他] │
│                                                 │
│  フィルタバー:                                    │
│  [未翻訳のみ ☐]  [ジョブ: 全て ▼]               │
│  ※フィルタの選択肢はカテゴリによって変わる         │
│                                                 │
│  ツールバー:                                     │
│  [CSVエクスポート]  [CSVインポート]               │
│                                                 │
│  進捗バー:  zh: 12/90 (13%)  ko: 0/90 (0%)      │
│                                                 │
│  テーブル:                                       │
│  項目ID    │ 日本語    │ 英語     │ 中国語 │ 韓国語│
│  reprisal  │ リプライザル│ Reprisal │ [入力] │ [入力]│
│  bulwark   │ ブルワーク  │ Bulwark  │ [入力] │ [入力]│
│  ...                                            │
│                                                 │
│  [保存]                                         │
└─────────────────────────────────────────────────┘
```

### カテゴリ別のフィルタ

- **スキル**: ジョブ別フィルタ（ナイト、白魔等）
- **コンテンツ**: カテゴリ別（零式、絶等）
- **攻撃名**: コンテンツ別（M1S、FRU等）
- **フェーズ**: コンテンツ別
- **その他**: カテゴリ名、レベル名、シリーズ名をまとめて表示（件数が少ないので）

### テーブル操作

- 全列（ja/en/zh/ko）が編集可能
- セルをクリックすると入力欄になる
- ja/enを変更すると黄色い枠で「変更あり」を視覚的に警告
- 未入力セルは背景色で視覚的に区別
- 変更があると「保存」ボタンが有効化

### 保存時の安全策

- ja/enの変更がある場合: 確認ダイアログ「日本語/英語の変更が○件あります。既存データに影響します。よろしいですか？」
- zh/koの追加のみ: 確認なしで即保存（新規追加なので既存への影響がない）

## CSV連携

### エクスポート（管理画面 → CSV）

- 「CSVエクスポート」ボタンで現在のカテゴリの全データをCSVダウンロード
- CSV形式:

```csv
ID,ja,en,zh,ko
reprisal,リプライザル,Reprisal,,
bulwark,ブルワーク,Bulwark,,
```

- 翻訳者に渡す際の手順: CSVをGoogleスプレッドシートにインポート → URLを共有
- ja/en列は翻訳者向けの参照用。CSVレベルでのロックはできないが、ヘッダーに「※編集しないでください」と記載

### インポート（CSV → 管理画面）

- 「CSVインポート」ボタンでファイルアップロード、またはCSVテキストをテキストエリアに貼り付け
- ID列で既存データと紐付け（名前ではなくIDで照合）
- インポート前に差分プレビューを表示:
  - 「追加: zh 45件、ko 45件」
  - 「変更: en 2件」（ja/en変更がある場合は黄色警告）
  - 「不明なID: xxx」（存在しないIDがあれば警告）
- 空欄はスキップ（既存の翻訳を消さない）
- 確認して保存

## データ構造の変更

### フェーズ名のLocalizedString化

現在:
```json
{ "phaseNames": { "1": "Fatebreaker" } }
```

変更後:
```json
{ "phaseNames": { "1": { "ja": "フェイトブレイカー", "en": "Fatebreaker", "zh": "", "ko": "" } } }
```

### シリーズ名のFirestore化

現在: `contentRegistry.ts`にハードコード
変更後: `master/contents.series`にLocalizedString付きで保存。コード内の値はフォールバック用に残す。

## 保存先

翻訳タブで保存すると、既存のFirestoreマスターデータ構造をそのまま使って更新する:

- スキル名 → `master/skills`のmitigations[].name
- コンテンツ名 → `master/contents`のitems[].name
- シリーズ名 → `master/contents`のseries[].name
- カテゴリ/レベル名 → `master/config`のcategoryLabels/levelLabels
- 攻撃名/フェーズ名/ギミックグループ → `templates/{contentId}`

保存時にはdataVersionをインクリメントし、アプリのキャッシュを無効化する（既存の仕組み）。

## スコープ外

- 既存の管理画面の変更（スキル追加モーダル等）
- UIテキスト（i18n localeファイル）の管理 — 開発者がコードで管理
- ジョブ名の翻訳 — i18n経由で対応済み
- ジョブ名ツールチップが翻訳に追従しない問題 — 表示側のバグとして別対応
