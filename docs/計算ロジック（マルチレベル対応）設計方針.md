# 計算ロジック（マルチレベル対応）設計方針

## 1. 目的
現在の `src/utils/calculator.ts` はパッチ7.0（Lv100）の定数（`LEVEL_MOD_MAIN = 440`等）がハードコードされている。これを改修し、任意のレベル（Lv70, 80, 90, 100等）の設定値に応じて、動的にヒール量・バリア量を計算できるスケーラブルな構造にする。

## 2. 現状の課題と解決策
1. **定数のハードコード:** レベルごとの補正値（Main, Sub, Div等）を定数テーブル（レベルキャップマスタ）として分離する。
2. **スキルデータの重複:** `calculator.ts` 内にある `SKILL_DATA` は、多言語化・一元管理の観点から完全に廃止し、`mockData.ts` の `Mitigation` 型（`healingIncrease`, `value` 等）に統合する。
3. **計算式の純粋化:** 計算関数は外部から「ステータス」「レベル」「スキル威力」を受け取り、純粋な計算結果のみを返す Pure Function に徹する。

## 3. 実装方針（アーキテクチャ）

### A. レベルキャップマスタの作成
`src/data/levelModifiers.ts` （または `mockData.ts` 内）を作成し、以下のようなテーブルを定義する。

```typescript
export interface LevelModifier {
    level: number;
    main: number; // LEVEL_MOD_MAIN
    sub: number;  // LEVEL_MOD_SUB
    div: number;  // LEVEL_MOD_DIV
}

export const LEVEL_MODIFIERS: Record<number, LevelModifier> = {
    70: { level: 70, main: 292, sub: 364, div: 900 },
    80: { level: 80, main: 340, sub: 380, div: 1300 },
    90: { level: 90, main: 390, sub: 400, div: 1900 },
    100: { level: 100, main: 440, sub: 420, div: 2780 },
};
```

### B. ストア（状態管理）への「現在のレベル」の追加
`src/store/useMitigationStore.ts` に以下の状態を追加する。
* `currentLevel: number` (デフォルトは100)
* `setCurrentLevel(level: number)` (レベル変更時のアクション)
* ※レベルが変更された際は、全メンバーの `computedValues`（計算結果）を再計算して上書きするロジックを走らせる。

### C. calculator.ts のリファクタリング
* `calculatePotencyValue` などの関数に、引数として `levelMod: LevelModifier` を追加する。
* `calculateMemberValues(member, currentLevel)` に修正し、`SKILL_DATA` を参照するのではなく、`MITIGATIONS` 配列をループして、ヒールやバリア効果を持つスキル（`potency` や `isShield` を持つもの）を自動で計算するロジックに変更する。

## 4. 今後のタスク
1. （前提タスク）`types.ts` および `mockData.ts` のデータベース統合を完了させる（`SKILL_DATA`の移植）。
2. `LEVEL_MODIFIERS` マスタの作成とFF14の正確な定数の入力。
3. `calculator.ts` をレベル引数を受け取るようにリファクタリングし、`SKILL_DATA` を削除。
4. `useMitigationStore.ts` にレベル管理機能を追加し、`calculateMemberValues` の呼び出し箇所（初期化、ジョブ変更、ステータス変更）にレベルを渡すように修正。
5. UIにレベル切り替え用のセレクトボックス等を追加。
