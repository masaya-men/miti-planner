# テンプレートエディター機能強化 設計書

> 作成日: 2026-04-02

## 背景

管理画面のテンプレートエディターで本番テストを行った結果、以下の問題が判明した：

1. スプレッドシートのギミックグループ情報がインポートで失われる
2. フェーズを編集する手段がない（読み取り専用）
3. 翻訳名を1つ入力しても同じ技名の他の行に反映されない
4. ダメージ値（カンマ付き数値）がパースされない

## スプレッドシートのデータ形式

実際のスプシからTSVコピペすると以下の形式になる：

```
開幕	00:00	00:00	戦闘開始！
	00:07	00:07	AA	Physical		50,000
	00:13	00:13	インビジブル・アスカロンメルシー
蒼天の陣：雷槍	00:29	00:29	蒼天の陣：雷槍
	00:42	00:42	百雷	Magic		41,000
```

- 1列目: ギミックグループ名（グループの最初の行のみ、他は空）
- 2列目: 時間
- 3列目: 時間（重複、スキップ対象）
- 4列目: 技名
- 5列目: ダメージ種別（Physical/Magic、ない場合もある）
- 6列目: （空列）
- 7列目: ダメージ量（カンマ区切り、ない場合もある）

列構成はスプシによってまちまちだが、上記が基本形。

## データモデルの階層

```
Phase (P1, P2, P3...)          ← 戦闘の大きな区切り
  └── MechanicGroup (開幕、蒼天の陣：雷槍...)  ← ギミックの塊
        └── TimelineEvent (AA、百雷...)         ← 個別の技
```

- フェーズとギミックグループは別の概念
- 1つのフェーズに複数のギミックグループが属する
- ギミックグループはスプシからインポート、フェーズはエディターで設定

## 機能1: ギミックグループのCSVインポート

### カラムタイプ追加

`ColumnType` に `'mechanic'` を追加する。

### 自動推測 (`guessColumnType`)

- ヘッダーに「ギミック」「mechanic」「group」を含む → `'mechanic'`
- ヘッダーがない場合: 1列目の値が大半空で、たまにテキストが入るパターンを検出

### `convertCsvToEvents` の変更

- `mechanic` 列の処理:
  - 値がある行 → 新しいギミックグループ開始。その値が `mechanicGroup` 名
  - 値が空の行 → 直前のギミックグループに属する
- 結果: 各 `TimelineEvent` に `mechanicGroup?: string` フィールドが付与される

### TimelineEvent 型の拡張

```typescript
export interface TimelineEvent {
  // 既存フィールド...
  mechanicGroup?: string;  // 追加
}
```

### TemplateData

変更なし。`mechanicGroup` は各イベントのフィールドとして保持し、表示時に動的にグルーピングする。`TemplateData` にグループ一覧を別途持つ必要はない。

## 機能2: エディター上のフェーズ範囲編集

### UI

- ギミックグループのセパレーター行にフェーズ選択ドロップダウンを配置
- ドロップダウンの選択肢: 既存フェーズ一覧 + 「+ 新規フェーズ」
- フェーズの切り替わりはギミックグループ境界でのみ発生

### 表示イメージ

```
━━━ P1 ━━━━━━━━━━━━━━━━━━━━━━━━
[P1 ▼] 開幕
  00:00  戦闘開始！
  00:07  AA              Physical  50,000
  00:13  インビジブル...
[P1 ▼] 蒼天の陣：雷槍
  00:29  蒼天の陣：雷槍
  00:42  百雷             Magic     41,000
[P2 ▼] 蒼天の陣：風槍     ← ここでフェーズ切り替え
━━━ P2 ━━━━━━━━━━━━━━━━━━━━━━━━
  ...
```

### 動作

- ドロップダウンでフェーズを変更すると、そのギミックグループの開始時間が新フェーズの `startTimeSec` になる
- 後続のギミックグループは、明示的に別フェーズが設定されるまで同じフェーズに属する
- 「+ 新規フェーズ」選択時: フェーズ名を入力するインラインフィールドが表示される

### ギミックグループがない場合

グループ列なしでインポートした場合（従来のCSVやプラン昇格）は、フェーズセパレーターのみ表示。フェーズ編集は従来通りの動作を維持。

## 機能3: 翻訳名の自動伝播

### 動作

- JA名またはEN名を手入力して確定したとき、同じペアを持つ他の行に自動伝播する
- 伝播方向: JA→EN、EN→JA の双方向

### 伝播ルール

| 操作 | 伝播先の条件 | 動作 |
|------|-------------|------|
| EN名を入力 | 同じJA名で、EN名が空 | EN名を自動入力 |
| EN名を修正 | 同じJA名で、EN名が修正前の値と同じ | EN名を更新 |
| JA名を入力 | 同じEN名で、JA名が空 | JA名を自動入力 |
| JA名を修正 | 同じEN名で、JA名が修正前の値と同じ | JA名を更新 |
| いずれの場合も | 対象行に全く別の値が手入力済み | 上書きしない |

### トグル

- ツールバーに「翻訳自動伝播」トグルボタンを配置
- デフォルト: ON
- OFF時: 編集した行のみ更新、他の行への伝播なし

### ハイライト

- 自動伝播された行は `autoFilled` ハイライト（青背景）で表示

## 機能4: ダメージ値パース修正

### 問題

スプシのダメージ値が `50,000` のようにカンマ区切りだが、`parseInt` ではパースできず `NaN` になる。

### 修正

- `convertCsvToEvents` のダメージパース処理でカンマを除去してからパース
- `"50,000"` → `"50000"` → `50000`
- `guessColumnType` のダメージ列推測も改善（列の値が数値パターンかどうかで検出）

## スコープ外

- 軽減表（ユーザー向け画面）の変更は行わない
- ギミックグループはテンプレートエディター内のみの概念（軽減表への影響は将来検討）
- 行の追加・ドラッグ&ドロップは対象外
