# プロジェクト概要: FF14 高速軽減シミュレーター「Miti Planner (みてぃぷらんなー)」
スプレッドシートの重い動作を解消し、スマートフォンでもPCでも「60fpsでサクサク動く」軽量かつリッチなFF14専用の軽減構築ウェブアプリケーションを開発します。

## 1. 技術スタック・アーキテクチャ
- **フロントエンド:** [React / Vue / Svelte などから指定]
- **スタイリング:** [Tailwind CSS などから指定]
- **動作の前提:** シングルページアプリケーション（SPA）として構築し、画面遷移や軽減追加時のフルリロード（画面のチラつき）を徹底的に排除する。
- **データ保存:** 初期段階から `Firebase (Firestore)` を導入し、クラウド同期による端末間の共有を可能にする。
- **大規模アクセス（バズ）対策と負荷分散:**
  - **課題:** W1stチームや有名配信者が「閲覧専用URL」をSNS等で公開し、数万人が同時に1つのデータにアクセスした場合、Firestore（データベース）の同時読み込み制限や従量課金コストのスパイクが発生するリスクがある。
  - **解決策（エッジキャッシュの導入）:** スプレッドシートのようにクラッシュさせないため、閲覧専用URLからのアクセスに対してはデータベースへ直接アクセスさせず、CDN（VercelやCloudflare等）のエッジサーバーに一時保存された「キャッシュ（静的データ）」を配信するアーキテクチャを採用する。
  - **効果:** これにより、どれだけ突発的な膨大なアクセスが来てもサーバー負荷とコストを完全に抑え込み、ユーザーに「重い」と感じさせない快適な閲覧体験を維持する。


## 2. 徹底したパフォーマンス最適化（体感遅延ゼロの実現）
※現在は機能の実装を優先しますが、後日スプレッドシート以上の軽快さを出すためのパフォーマンスチューニングを行う予定です。そのため、以下の設計方針を最初から遵守してください。
1. **素直なコンポーネント設計:** 後からメモ化（React.memoやuseMemo等）がしやすいように、コンポーネントの分割と状態管理（State）はシンプルに保つ。
2. **ホバー処理のCSS化:** テーブルの行やセルのハイライト（hover効果）は、JavaScriptのイベント（onMouseEnter等）を使用せず、Tailwind CSSの `hover:bg-gray-800` のようにすべてCSSの疑似クラスで完結させる。
3. **再描画のブロック:** 状態（State）が変更された行や、特定ターゲットの計算結果が変わった箇所「のみ」が再レンダリングされるようにし、表全体の再描画を絶対に防ぐ。エフェクトには原則 `transform` や `opacity` 等のGPU処理されるCSSプロパティを使用する。

## 3. データ構造（JSON）の要件
コンテンツごとに軽減表をグループ化できるよう、以下のような階層構造でデータを管理します。
- **Series** (例: AAC Light Heavyweight Savage / ライトヘビー級零式)
  - **Floor** (例: 1層, 2層...)
    - **Plans** (ユーザーが作成した軽減プランの配列)

## 4. 画面レイアウトとレスポンシブ要件
### ナビゲーションとメニューの挙動
- **PC画面:** 左側に固定配置し、1クリックで層を切り替えられるようにする。ハンバーガーメニュー（またはトグルボタン）を設置し、クリックでサイドバーを折りたたんでメイン画面を最大化できるようにする。
- **スマホ画面 (モバイルファースト):** ハンバーガーメニューやドロワーは使用せず、画面下部に「ボトムナビゲーション」を固定配置する。タイムラインの入力操作を阻害しないよう、「📊 タイムライン」「👥 パーティ編成」「⚙️ 設定」などの大項目の画面切り替えのみに限定して使用し、片手での直感的な操作を実現する。


### メイン画面（右側エリア）の構成
- **上部:** 「Party Stats (ステータス設定)」と「Shield Preview (シールド値概算)」の入力・表示エリア。
- **下部 (PC版):** 軽減表のメインテーブル。
  - ヘッダー構成: `Ph` / `Time` / `Mechanic` / `Raw` / `Taken`
  - 「誰から誰へ」は `Boss on MT` のように `on` で繋ぎ、斜体・グレーでスタイリング。
  - Takenの下に `(-31%) | +41,791` のように軽減とバリア量をシンプルに表示。

### スマホ版のメインテーブルUI (モバイルファースト)
- 横長の巨大な表は表示せず、`Ph`, `Time`, `Mechanic`, `Raw`, `Taken` の情報のみが縦にスクロールできるリストにする。
- **軽減の追加アクション:**
  1. メカニック（行）をタップする。
  2. 現在のパーティメンバーの「ジョブアイコン」が表示される。
  3. ジョブをタップすると、そのジョブが持つ「軽減スキルアイコン」が並ぶので、使いたいものをタップして追加。
  4. 追加された軽減アイコンは、各行のメカニック付近にコンパクトに並べて表示する。

## 5. 詳細な機能・UI/UX仕様
### 1. テーブルレイアウトの制約と入力UX
- **ジョブロールごとの軽減表示枠の固定:** 軽減アイコンが増減するたびに列の幅がガタガタ動くのを防ぐため、各行に並ぶ軽減アイコンの最大枠数をロールごとに固定し、均一な幅を確保する。（Tank/Healerは最大5枠分、DPSは最大2枠分の固定スペース）
- **数値入力の即時性:** ステータスやダメージ値などの「数字入力セル」をクリック（フォーカス）した際は、即座に中の数字が「全選択状態」になり、バックスペースキーを押さなくてもそのまま上書き入力できるUXにする。

### 2. 単体軽減の投げ先選択（タンク間）
- ブラックナイトやコランダム等、誰かに投げる軽減については、追加時に「誰に投げるか（対象ジョブ）」を選択可能にする。
- 選ばれた場合、タイムライン上の軽減アイコンの右下に「対象者のジョブアイコン」を小さくバッジ表示する。
- 計算ロジックは即座に対象者（投げられた側）のステータス（HPや被ダメ）を参照して再計算されること。

### 3. 「自ジョブ（My Job）」の選択と強調表示機能
- **選択UI:** 「Party Stats」エリアでパーティメンバーのジョブを設定する際、どのキャラクターが「自分（My Job）」であるかを選択できるラジオボタンを設置する。
- **強調エフェクト:** 自ジョブが設定されている状態でタイムラインを見る際、自分のジョブの軽減アイコンだけが視覚的に際立つようにする（他者のアイコンは `opacity-40` や `grayscale` で目立たなくし、自ジョブはフルカラー＋ハイライト）。トグルスイッチでON/OFF可能にする。

### 4. 早見表（Cheat Sheet）ビューの作成
- プレイ中に横目で見ながら戦うための、視認性に特化した「早見表モード」を作成する。
- **レイアウト:** 左半分を「MT Group」、右半分を「ST Group」に分割。
- **表示項目:** `Time`、`Mechanic`、そして「設定された軽減アイコン」のみをシンプルに並べる。マクロ出力ではなく、UIビューとして実装する。

### 5. 表示の最適化とテーマ（UIデザイン）
- **空行の圧縮:** タイムライン上で「軽減もメカニックも入力されていない空白の行」を非表示に圧縮し、見やすくするトグル機能（Hide Empty Rows）を実装する。
- **グローバルテーマの同期:** 総合ポータルとして、Housing Tour Planner（ハウジングツール）とテーマ設定（ライト / ダーク）を共有・同期し、アプリ間移動時の視覚的ストレスを完全に排除する。
- **背景画像のデフォルト固定（設定の排除）:**
  - **背景画像のデフォルト固定とテーマ設計（モダン・アブストラクト）:**
  - ユーザーによる背景設定は排除し、Housing Tour Plannerと共通の「モダンで抽象的なテクスチャ」を背景に固定する。FF14プレイヤーに合わせた「ダークモード」をデフォルトとする。
  - **ライトモード:** 完全な純白（#FFFFFF）に微かな水滴とプリズムの光が滲む抽象背景の上に、半透明の白パネル（すりガラス）を適用。
  - **ダークモード:** 完全な漆黒（#000000）に微かな水滴と一筋の鋭い光が反射する抽象背景の上に、半透明の黒パネル（ブラック・グラス）を適用。
暗所でのプレイに優しく、高級感のあるプロツールを演出する。
- **パフォーマンス最適化の絶対ルール（60fps維持）:**
  - タイムライン（表）のセル一つ一つに透過効果をかけるとブラウザが重くなるため、必ず「表全体を囲う1つの大きなコンテナ（箱）」に対してのみ `backdrop-filter: blur()` を適用し、その上の文字や線は透過させずクッキリと描画する設計とする。
  - これにより、グラスモーフィズムの美しさと爆速のスクロールを両立する（※実装後のテストで動作が重いと判断した場合は、速やかに透過なしの最軽量フラットデザインに切り替える柔軟性を持たせる）。

### 6. 外部データ連携と自動化（スマート機能）
- **タイムラインのCSVインポート機能:**
  - 既存の外部ツールやスプレッドシートで作成された正確なタイムライン（`Time`, `Mechanic`, `Raw`ダメージ、攻撃属性など）を、CSV形式で一括インポートできる機能を実装する。
  - これにより、PCユーザーがタイムラインの土台を一瞬で構築してURLをシェアし、PS5やスマホユーザーが「手入力の手間なく自分専用のUIで軽減構築に参加できる」という、クロスプラットフォームな体験を実現する。

### 7. アルゴリズムによる軽減自動組み立て（オートプランナー）機能
- 外部のAI（LLM API等）はコストと速度の観点から使用せず、フロントエンドの純粋なJavaScriptアルゴリズムで実装する。
- **処理ロジック:** タイムライン上の攻撃属性タグ（「全体攻撃」「タンク強攻撃」など）と「致死量のRawダメージ」を検知し、パーティに編成されたジョブの軽減スキル（リプライザルや陣など）を、それぞれの「リキャストタイム（60秒、90秒など）」を計算しながら自動でタイムラインに割り当てる。
- **UXの目的:** ユーザーがゼロから組むのではなく、「とりあえず死なないベースの軽減表」を1クリックで瞬時に生成し、そこから自分好みに微調整を行うという革新的なUXを提供する。

## 8. 公開にあたっての法的要件とフッター記載事項
FF14の公式画像やアセットを使用する非公式ツールとして、スクウェア・エニックスの「ファイナルファンタジーXIV 著作物利用条件」を厳守し、以下の要素をアプリ内の全ページ（または常時表示されるフッター）に必ず実装すること。

- **権利表記（コピーライト）:**
  `(C) SQUARE ENIX CO., LTD. All Rights Reserved.` を視認可能な形で明記する。
- **免責事項（ディスクレーマー）:**
  「当サイト（Miti Planner）は非公式のファンツールであり、株式会社スクウェア・エニックスとは一切関係ありません。」といった文言を記載し、公式サービスではないことを明示する。
- **マネタイズの制限事項の遵守:**
  ツール内での直接的な課金（有料プランの提供やツールの販売等）は利用規約違反となるためシステムとして実装しない。マネタイズの導線を作る場合は、公式に許可された外部プラットフォーム（YouTube動画へのリンクや埋め込み等）への誘導に留めるUI設計とする。
