# カンペ スマホ UX 改善（編集モーダル + 副次調整）設計書

作成日: 2026-05-03
ベース: [2026-05-01-pip-revival-design.md](./2026-05-01-pip-revival-design.md)

---

## 1. 概要

スマホ全画面カンペ（`mode === 'fullscreen'`）の操作性を改善する。

戦闘中の即時参照ではなく **戦闘前の自分用カスタマイズ** が主用途のため、スマホでも確実に操作できる UI に組み替える。同時にスマホで動作しない/不要な要素を整理し、視覚的なリッチさも向上させる。

**スコープ厳守:** 全変更は `mode === 'fullscreen'` 分岐内のみ。PC 別ウィンドウ PiP（`mode === 'pip'`）は完全に現状維持。

---

## 2. 背景・課題

| 項目 | 現状 | 課題 |
|------|------|------|
| 攻撃名の編集 | ダブルタップで inline 編集 input | スマホでダブルタップは打ちづらく、編集導線が見えない |
| 同時刻イベント切替 | 行右端の `+N` バッジタップで切替 | クイック切替としては OK だが、編集と組合せると操作系が分散 |
| 背景カラーピッカー | `<input type="color">` を hidden にしてボタン経由で `.click()` | iOS Safari ではほぼ開かない（hidden + サイズ 0 + JS click の制約） |
| デフォルト色リセット | カラーピッカー隣のスウォッチ | カラーピッカー本体が動かないので無意味 |
| リスト本体のスクロール示唆 | なし | 上下に隠れているコンテンツの存在に気づきにくい |

---

## 3. ユースケース

- **戦闘前の準備**: スマホで PiP カンペを開き、同時刻に来る複数攻撃のうち「自分が見たい方」を選んで表示。さらに攻撃名に自分用メモを付ける（例: 「2 連目」「死ぬ」「タンク散開」）。
- **戦闘中の確認**: 既存の挙動どおり、`+N` バッジで素早く表示切替。

---

## 4. スコープ

### 含む
- 技名タップ → 編集モーダル（攻撃切替 + 名前編集、攻撃数に関わらず常に同じ UI）
- ダブルタップ編集をスマホ時のみ廃止
- カラーピッカー & デフォルト色スウォッチをスマホ時のみ非表示
- リスト本体の上下に弱めの mask-image フェード追加
- vitest 追加（モーダル開閉ロジック / 切替 / 編集 / ダブルタップ非発火）

### 含まない
- PC 別ウィンドウ PiP の挙動変更（一切触らない）
- メモ機能の Firestore 同期（端末ローカル localStorage 維持）
- 行内のメモリセット × ボタン廃止（残す。メモ付きアフォーダンス兼）
- カラーピッカー機能自体の修復（別 issue 候補：プリセット数色のスウォッチ列に置き換える等）

---

## 5. 仕様

### 5.1 編集モーダル

**トリガー**: スマホ全画面で技名（または `+N` バッジ以外の行領域）をタップ。

**表示形式**: 画面中央の固定モーダル。背景は半透明黒（`bg-black/50`）。

**モーダル内 DOM**:
```
┌──────────────────────────────────┐
│                              [×] │
│ ┌──────────────────────────────┐ │
│ │ [攻撃名 1]            [✎]    │ │ ← 同時刻の各イベント
│ ├──────────────────────────────┤ │
│ │ [攻撃名 2]            [✎]    │ │ ← 複数あれば
│ └──────────────────────────────┘ │
└──────────────────────────────────┘
```

**動作**:
| 操作 | 結果 |
|------|------|
| `[攻撃名]`（編集ボタン以外）タップ | そのイベントを「現在表示」に切替（`eventIndexByTime` 更新）→ モーダル閉じる |
| `[✎ 編集]` タップ | その行の攻撃名ラベル → input に切替（モーダル内完結） |
| 編集中 Enter / blur | 確定 → モーダル閉じる |
| 編集中 Esc | キャンセル → 攻撃名ラベルに戻る（モーダルは開いたまま） |
| `[×]` ボタン | モーダル閉じる |
| 背景タップ（編集中以外） | モーダル閉じる |
| 背景タップ（編集中） | input の blur で確定 → モーダル閉じる（Enter / blur と同じ）|
| Esc キー（編集中以外） | モーダル閉じる |

**サイズ**:
- パネル幅: `min(360px, calc(100vw - 32px))`
- 各行の高さ: 約 48px（攻撃名 17px + padding）
- 編集ボタン: 36×36（指で押しやすい）
- × 閉じるボタン: 32×32

**スタイル**:
- パネル背景: 既存の `glass-tier3` 相当
- 角丸: `rounded-xl`
- 各行は薄い区切り線で区切る
- 攻撃名: 17px font-bold（カンペ本体と同じ）
- 攻撃名にメモがあれば `notes[event.id]`、なければ `event.name[lang] || event.name.ja || event.name.en` の優先順（カンペ本体と同じロジック）

**新規 state**:
```typescript
const [menuTime, setMenuTime] = useState<number | null>(null);
// 既存: editingEventId はモーダル内編集に流用
```

`menuTime` で「どの時刻のメニューが開いているか」を管理。`null` なら閉じている。

### 5.2 ダブルタップ編集の廃止（スマホ時のみ）

`onDoubleClick={() => handleDoubleClick(event.id)}` を `mode === 'pip'` のときだけ付与する条件付きハンドラに変更。スマホ時は付与しない。

### 5.3 `+N` バッジ（変更なし）

`+N` バッジは現状の動作（タップで `eventIndexByTime` を循環更新）を維持。スマホ・PC 両方で残す。モーダル内の切替と機能重複するが、用途が違う（クイック切替 vs リストから選ぶ）ので両立。

### 5.4 行内メモリセット × ボタン（変更なし）

[PipView.tsx:314-323](../../../src/components/PipView.tsx#L314-L323) の現状実装を維持。メモ付きアフォーダンスとしての役割もあるため。

### 5.5 カラーピッカー & デフォルト色スウォッチ（スマホ時のみ非表示）

[PipView.tsx:225-248](../../../src/components/PipView.tsx#L225-L248) のカラーピッカーボタン・hidden input・デフォルト色スウォッチを `!isFs` 条件でラップし、スマホ時は描画しない。閉じるボタンの間隔調整は不要（既存の `flex-1 min-w-0` スペーサーで自然に詰まる）。

### 5.6 リスト本体の上下フェード（スマホ時のみ）

リスト本体（[PipView.tsx:262-266](../../../src/components/PipView.tsx#L262-L266) の `flex-1 overflow-y-auto` div）に上下方向の mask-image を追加：

```css
mask-image: linear-gradient(
    to bottom,
    transparent 0,
    black 24px,
    black calc(100% - 24px),
    transparent
);
```

ジョブアイコン横並びの右端 fade（50px）より弱め（24px）。スクロール可アフォーダンス + 視覚的リッチ。

`isFs` のときだけ inline style に追加。`-webkit-mask-image` も同時指定。

## 7. テスト計画

### 7.1 vitest（自動）
- モーダル: `menuTime` が `null` → 非表示、`time` 設定 → 表示
- 切替: 攻撃名タップで `eventIndexByTime[time]` が更新され、モーダルが閉じる
- 編集: `[✎]` タップ → 該当行が input になる、Enter で `updateNote` 呼ばれてモーダル閉じる、Esc で input が閉じてモーダル開いたまま
- ダブルタップ: `mode === 'fullscreen'` のとき `handleDoubleClick` が呼ばれない（ハンドラ未付与）
- ダブルタップ: `mode === 'pip'` のとき `handleDoubleClick` が呼ばれる（既存挙動維持）

### 7.2 実機（手動・スマホ）
1. スマホ全画面でカンペを開く
2. 攻撃 1 つだけの行をタップ → モーダルにその攻撃 1 件 + 編集ボタン
3. 編集ボタン → input に切替 → 「テスト」入力 → Enter → モーダル閉じる、行に「テスト」表示、行内 × ボタン出現
4. 攻撃 2 つの行をタップ → モーダルに 2 件並ぶ
5. 2 件目の攻撃名タップ → モーダル閉じる、行の表示が 2 件目に切替
6. 行右端 `+N` バッジで素早く切替（モーダル経由せず）
7. 行内 × ボタンでメモリセット
8. ツールバー上のジョブアイコンを横スクロール（既存）
9. ツールバーにカラーピッカー・デフォルト色スウォッチが**ない**こと
10. リスト本体の上下に弱いフェード
11. モーダル開いた状態で背景タップ → 閉じる
12. モーダル開いた状態で × ボタン → 閉じる
13. 編集中 Esc → input 閉じてモーダル維持、もう一度タップ可能

### 7.3 実機（手動・PC）
- PC 別ウィンドウ PiP で全機能（ダブルタップ編集 / カラーピッカー / `+N` バッジ）が**変わっていない**こと
- ツールバーのレイアウトが変わっていないこと

## 8. リスク・既知の制約

- **モーダル開いた状態で別の行をタップできない**（モーダルが背景全部覆うので）。閉じてから再タップ。これは仕様。
- **モーダル背景タップで閉じる仕様** は既存 LoPo パターンと同じ。誤タップで閉じても再タップで開けるので問題小。
- **`<input type="color">` は DOM に残る**（Hidden input は PC でのみ意味があるため、スマホ時は描画しないが PC 時は残す）。条件分岐の都合上、ボタンと input をセットで `!isFs` ラップする。
- **編集 input の autoFocus** は既存 [PipView.tsx:97-102](../../../src/components/PipView.tsx#L97-L102) の `useEffect` をそのまま流用（モーダル内でも同じく `editInputRef.current?.focus()` `select()` が走る）。
