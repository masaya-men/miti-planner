   ※ Claude は Discord OAuth 不可のため、この検証はユーザー操作が必須。

2. 🚨 **マップ問題の決着** (→ 詳細は [housing-map-todo.md](./housing-map-todo.md))
   現状デフォルトビューが「マップ」だが、マップは `sampleWardLayout` の**モック配置のまま** (実物件に地図座標が無い)。
   このまま公開すると、ランディングでユーザーに**偽の物件配置**を見せてしまう。
   → **方針 (2026-05-21 確定)**: マップ用データのオーサリングは masaya 本人作業。完成までは時間がかかるため、
     **公開時はデフォルトビューを list にして先行リリース**し、マップは後追いで有効化する。

3. 🚨 **コールドスタート回避 (事前データ登録)**
   現状 `housing_listings` は実物件 1 件のみ。公開前に自分で実 UI から複数物件を登録し、一覧が寂しくない状態にする。

4. 🚨 **通報→運営対応が /admin で完結するか動作確認**
   通報された物件を運営が /admin から非表示/対応できるフローを実機で確認 (Discord/Firestore 直叩きに頼らない)。

5. 🚨 **スマホ最適化の最低ライン確認**
   主要導線 (一覧→詳細→お気に入り→登録) がモバイルで破綻しないか。

6. 🚨 **英語(en) 翻訳の完成** (公開言語 = 日本語 + 英語)
   ハウジング UI の en 翻訳が現状 ja 値コピーのため、英語の実値が必要 (`housing.gallery.loading/error` 含む)。
