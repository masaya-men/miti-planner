# ランディングページ 超クオリティ化 指示書

## セッション開始時の必須作業
**まず `CLAUDE.md` と `docs/TODO.md` を読むこと。**

## 目標
LoPoのランディングページを、以下の参考サイトと同等レベルのインタラクティブな没入体験に作り替える。

### 参考サイト（必ず全部見ること）
- https://www.igloo.inc/ — フルWebGL、プロシージャルビジュアル、3D空間カメラ移動
- https://activetheory.net/ — 没入型3D環境、SF的フォント、AIチャットナビ
- https://buttermax.net/ — スタガードタイポグラフィ、セクション背景色遷移、カスタムカーソル
- https://www.neoculturalcouture.com/ — グリッドトレイルエフェクト、ディスプレイスメントマップ、流体タイポグラフィ
- https://labs.noomoagency.com/ — プリローダー演出、clip-pathリビール、スクロール速度連動
- https://www.denofwolves.com/en — ゲーム世界観のUIメタファー化

### これらのサイトの共通する本質（現在のLPに欠けているもの）
1. **ページ全体が1つの連続した体験** — 「セクション1、セクション2」とブログ的に並べるのではなく、スクロールが物語を進めるエンジンになっている
2. **スクロール＝アニメーションのトリガー** — スクロール量に完全に同期して要素が動く（scrub）。フェードインが終わったら止まる、ではない
3. **背景とコンテンツが一体** — WebGLや3Dが「背景」ではなく「主役」。テキストはその世界の中に存在する
4. **予測不可能な動き** — 次に何が起こるかわからないワクワク感。同じパターンの繰り返しがない

### 問題点（なぜ参考サイトと全然違うのか）
- セクションが独立していて**ブログ記事のように並んでいるだけ**
- スクロールしても**テキストがフェードインするだけ**で驚きがない
- 背景WebGLとコンテンツが**完全に分離**していて一体感がない
- **同じアニメーションパターンの繰り返し**（fadeIn, slideIn, scale）
- タイポグラフィが**まだ大人しい**
- **インタラクティブ要素が少ない**（カーソル以外に触って楽しいものがない）

## やるべきこと

### アプローチ: 全面作り直し
現在のセクション分割構造を捨てて、**1つの連続的なスクロール体験**として再構築する。

### 具体的な方向性

#### 1. スクロール駆動の連続体験
- GSAP ScrollTrigger の `pin` + `scrub` を多用
- セクション間の明確な境界をなくす
- スクロール位置に完全同期したアニメーション（toggleActionsではなくscrub）
- 要素が画面に固定（pin）されて、スクロールで変形していく演出

#### 2. WebGLを主役にする
- ParticleBackgroundを拡張するか、セクションごとに異なるシェーダー演出
- テキストがWebGL空間の中に存在する感覚
- スクロールでシェーダーのパラメータが変化

#### 3. 予測不可能な動き
- セクションごとに異なるアニメーション手法
  - Hero: 文字が3D空間で回転しながら組み上がる
  - 軽減紹介: アプリ画面がスクロールで「飛び出してくる」（perspective + z軸移動）
  - 機能: カードが物理シミュレーション風に落下してバウンス
  - ハウジング: 画像が液体のように変形して出現
- 同じパターンを2回使わない

#### 4. インタラクティブ要素
- マウス位置でテキストが微妙に3D傾斜
- ホバーで要素が「磁石的に」吸い付く
- クリック時のリップルエフェクト
- パーティクルがマウスから逃げる/集まる演出

### 動画素材について
- 現在は動画素材なし。プレースホルダーで実装
- 軽減プランナーのモック画面は**実際のアプリUIを簡略化したもの**でリッチに見せる
- ハウジングは🏠アイコン+グリッドのコンセプトモックで良い
- 動画が撮れたら後から差し替え可能にしておく

## ルール
- **白黒のみ**（CLAUDE.md準拠）。アクセントカラー禁止
- **i18nキー経由**でテキスト表示（ハードコーディング禁止）
- **既存のi18nキーをそのまま使う**（portal.hero.*, portal.miti.*, etc）
- **既存のルーティング構造を維持**（/ → LandingPage, /miti → MitiPlannerPage）
- 文言は仮置き。後でユーザーと調整する
- **モバイル対応**も忘れずに（カスタムカーソルはモバイル無効化等）

## 成功基準
ユーザーが参考サイトを見た後にLoPoのランディングページを見て、「おお、これはすごい」と思えるクオリティ。ユーザーは非エンジニアなので、コードではなく**見た目と体験**で判断する。
