# 住所登録なし「一時ツアー」 設計書

> 2026-07-10 brainstorming でユーザー承認済み。公開前必須の大物3つの3つ目
> (原要望 = `docs/.private/2026-07-08-housing-release-feature-braindump.md` #6)。
> 実装は本設計書 → writing-plans の計画書を経てから。

## 1. 目的

**「登録はしたくないけど、ツアーには組み込みたい」**を叶える。
SNS で見つけた家や知っている住所を、Firestore に保存せずその場でツアーに混ぜて回れる。

## 2. 確定済みの設計判断 (2026-07-10 ユーザー確定・変更不可)

1. 本命ペルソナ = **B: 既存ユーザーが未登録の家をツアーに組み込む**。
   → **登録済みの家と一時の家の混在ツアーが核**。単独の一時ツアーも自然に成立する。
2. 入口 = 3箇所: **探すのツアートレイ / お気に入りのツアートレイ / ツアータブの空状態**に「+ 住所から追加」。
3. 入力 = 2方式を1パネルで: **SNS URL 貼り付け** (住所+画像を自動取得) と **住所テキスト入力**
   (「ミスト 3区 15番地」→ 自動解釈)。どちらも**読み取れた住所をチップで確認表示**してから追加。
   不足分だけ選択肢で補完 (登録フォームの自動入力と同じ流儀)。
4. **保存しない・ログイン不要**。ツアー自体が揮発 (リロードで消える) なので、それに素直に乗る。
   追加パネルに「リロードで消えます」を1行注記。
5. 一時の家には**「一時」バッジ** (トレイ・ツアー画面)。
6. ショーケースに**「この家を登録する」の控えめなリンク** — 押すと登録フォームに住所と URL が
   入った状態で開く (ツアー体験→登録への橋)。
7. **探すの一覧には絶対に混ぜない** — 一時の家は専用プールに分離 (公開一覧の store を汚染しない)。

## 3. データ設計

### 3.1 一時 listing (EphemeralListing)

- 形は `MockListing` の**最小サブセット互換** (ツアー描画が読むフィールドのみ実在させる):
  `id` / `area` / `ward` / (`plot`+`size?` または `buildingType:'apartment'`+`apartmentBuilding?`+`roomNumber?`) /
  `title?` (無ければ住所表示にフォールバック) / 画像系 (`imageMode`+`ogImageUrl` 等、SNS 取得時のみ) /
  `postUrl?` (登録リンクの引き継ぎ用)。
- `id` は `ephemeral-` prefix + 連番/時刻で衝突回避。**この prefix が「一時」判定の唯一の根拠**
  (表示バッジ・詳細遷移の抑止・登録リンクの分岐は全て id prefix 判定に閉じる)。
- 検証: `area` は 5 住宅街 enum / `ward` 1-30 / `plot` 1-60 / アパート部屋 1-90。範囲外は追加時に弾く。

### 3.2 一時プール (専用 store)

- 新設 zustand store (`persist なし` = リロードで消える)。`ephemeralListings: EphemeralListing[]` +
  `add / remove / clear`。
- **合流点はツアー解決の pool 計算のみ**: TourNavPage (および TourTray の解決) が使う
  `mergeListingsForViewer(listings, myListings, uid, now)` の結果に一時プールを**追加でマージ**する。
  探す一覧 (`BrowsePage` の `merged`) には合流させない (§2-7)。
- ツアー開始フロー (`orderTourStopIds` → `setListings` → `start`) は無変更で、一時 id が
  混じった ids がそのまま流れる (id→pool 解決が前提の既存構造に乗る)。

### 3.3 経路・地図

- 道案内 (`getPlotDirections(area, plot)`) と地図配置は**登録済みと同一品質で動く**ことを確認済み
  (経路データは 5 住宅街 × 番地 1-60 の静的マスタで ward 非依存・全件生成済み。
  アパートも建物 1 点+幾何解決の既存機構で動く)。一時ツアーのための経路側変更は**ゼロ**。

## 4. 画面仕様

### 4.1 追加パネル (「+ 住所から追加」)

- トレイ内ボタン → housing トンマナのモーダル/パネルを開く。
- 上段: **URL 欄** — 貼ると既存の取得機構 (OGP / ツイート) で住所テキスト抽出 + 画像取得。
  取得中はスピナー相当の静かな表示。
- 下段: **住所テキスト欄** — `parseHousingFromText` で即時解釈。
- 解釈結果は**チップ表示** (住宅街 / 区 / 番地 or アパート部屋)。欠けている項目だけ選択肢
  (住宅街 5 択・区 1-30・番地 1-60 / アパート切替+部屋番号) が現れて補完できる。
- 曖昧・読取不能のとき: 「住所を読み取れませんでした」を表示して手入力に誘導。
  **推測で埋めない** (住所誤爆と同じ方針 [[feedback_no_speculative_alias_data]] の精神)。
- 全項目が揃ったら [ツアーに追加] 活性化 → 一時プールに追加 + トレイに積む → パネルは連続追加できるよう
  入力だけクリアして開いたまま (閉じるは明示操作)。
- 注記 1 行: 「一時の家は保存されません (ページを離れると消えます)」。

### 4.2 トレイ・ツアー画面での表示

- トレイ項目・ツアーのステップ/ショーケースに小さな「一時」バッジ (装飾ピルにしない・既存の
  `badge_private` 系の質感)。
- 画像なしの一時の家は住所のみのシンプル表示 (プレースホルダ画像は既存の PLACEHOLDER を流用)。
- 一時の家はクリックしても詳細ページに**飛ばない** (登録がないため)。カーソル/role も リンク扱いにしない。

### 4.3 「この家を登録する」リンク

- ショーケースパネルの一時の家にのみ表示する控えめなテキストリンク。
- 押すと `/housing/register` へ遷移し、**一回限りの受け渡し領域** (sessionStorage の専用キー) 経由で
  住所と SNS URL をフォームに初期投入 (RegisterPage 側はマウント時に読み取り→即クリア)。
- 未ログインでも登録ページ自体の既存ログイン導線に乗るだけ (特別扱いしない)。

## 5. 安全・エラー

- SNS 取得は**既存のログイン不要エンドポイント** (`/api/og-fetch`, `/api/tweet-meta`) をそのまま使う。
  新しい API は増やさない (乱用面の検討も既存と同一枠)。
- 存在しない番地・範囲外は追加時に拒否 (§3.1 検証)。
- URL 取得失敗 / 住所抽出失敗はパネル内のエラーテキストのみ (トースト乱発しない)。
- 一時プールは最大 30 件で頭打ち (トレイの実用上限。超過時は注記表示)。

## 6. トンマナ・i18n

- `.claude/rules/housing-design.md` 準拠。新規キーは `housing.ephemeral.*`。4言語 parity。「ハウジング」統一。

## 7. テスト方針

1. パーサ連携 (テキスト→チップ→不足補完→追加可否) — コンポーネント
2. プール分離 (一時追加後も BrowsePage の一覧に現れない / ツアー pool には現れる) — ユニット+コンポーネント
3. 混在ツアー (登録済み id + 一時 id の混在で resolveTourSteps が両方解決) — ユニット
4. バッジ/詳細遷移抑止/登録リンク (id prefix 判定) — コンポーネント
5. 検証境界 (ward 31 / plot 61 / 部屋 91 拒否) — ユニット
6. 実機: SNS URL→画像つき一時の家→混在ツアー 1 周 (ユーザー確認)

## 8. スコープ外 (YAGNI・明示的に作らない)

- 一時ツアーの保存・共有 URL 化 (保存が必要 → 共有ツアー同期の別PJ `docs/.private/2026-07-08-synced-shared-tour-vision.md` へ)
- 一時の家のお気に入り登録 (お気に入りは Firestore の listingId 前提のため対象外。ボタン自体を出さない)
- 一時の家の編集 (削除して入れ直す)
- OCR (画像内の住所読み取り)

## 9. 実装順の目安 (計画書化の入力)

1. EphemeralListing 型 + 検証 + 一時プール store (テスト先行)
2. ツアー解決 pool への合流 (TourNavPage / TourTray) + 混在ツアーのユニットテスト
3. 追加パネル (URL 取得 / テキスト解釈 / チップ+補完 / 上限・注記)
4. バッジ・詳細遷移抑止・画像なし表示
5. 「この家を登録する」リンク + RegisterPage の一回限り初期投入
6. i18n 4言語 + CSS + 全体検証
