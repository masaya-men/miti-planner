# セッションワークフロー

## 開始時
1. `docs/TODO.md` を読む — これだけで現在の状態・タスク・注意事項がわかる (100 行以内のはず)
2. 「次セッション最優先」 セクションの 1 番上のタスクから着手
3. 詳細議論ファイル (`docs/.private/`) は該当タスクに着手するときに読む
4. 他のドキュメント (DESIGN_DECISIONS.md / ADMIN_SETUP.md 等) は必要時のみ読む

## 作業中
- **会話で決まったアイデア・方針・設計判断は即座に記録する** (指示なしでも自動的に):
  - 公開可能な技術的 TODO → `docs/TODO.md` の該当セクション
  - 大きな設計議論 / 詳細未確定の議題集約 → `docs/.private/YYYY-MM-DD-{topic}.md` (gitignore 済)
  - 短いルール・判断 → memory (`feedback_*.md`)
- **完了したタスクはその場で `docs/TODO_COMPLETED.md` へ移動** (確認してから)
- TODO.md は常に **100 行以内**を維持 (旧目標 50 → 現実的に 100、 余裕あれば 50 を目指す)

## 終了時 (必須チェックリスト)
1. **TODO.md の行数確認** — `wc -l docs/TODO.md` で 100 行以内か確認、 超過なら整理
2. TODO.md 先頭の「現在の状態」 セクションを更新 (ブランチ / 直近デプロイ / 進行中タスク)
3. 完了タスクが残っていないか確認 → 残っていれば TODO_COMPLETED.md へ移動
4. コミット → push → デプロイまでセットで完了
5. ユーザーに引き継ぎメッセージを出力 (次セッション最初にコピペできる形式):
   - 変更したファイル一覧
   - 次の最優先タスク
   - 「docs/TODO.md 読め」 指示

## コンパクション時の保持指示
When compacting, always preserve: 現在のタスク、変更中のファイルパス、TODO.md の「現在の状態」 セクション内容
