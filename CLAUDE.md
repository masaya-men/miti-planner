# CLAUDE.md

## セッション開始時の必須作業
**毎回のセッション開始時に、`docs/` フォルダ内の全計画書を必ず読むこと。**
これらはユーザーが蓄積してきたプロジェクトの設計方針・仕様書であり、作業の前提となる。

読むべきファイル一覧:
- `docs/TODO.md` — タスク管理（進行中・未着手・完了・設計方針）
- `docs/GRAPL_PROJECT_PLAN.md` — プロジェクト全体ロードマップ
- `docs/Firebase設計書.md` — Firestore構造・セキュリティ・スケーラビリティ
- `docs/CORE_UPGRADE_PLAN.md` — SA法オートプランナー + OGP + チームロゴ
- `docs/高精度オートプラン・アルゴリズム仕様書.md` — オートプランの詳細アルゴリズム
- `docs/計算ロジック（マルチレベル対応）設計方針.md` — レベル別計算式
- `docs/マスタデータ（データベース）設計方針.md` — データ構造設計
- `docs/多言語対応（i18n）アーキテクチャ設計方針.md` — i18n設計
- `docs/チュートリアル（オンボーディング）機能設計方針.md` — チュートリアル設計
- `docs/miti-planner-requirements.md` — 軽減プランナー初期要件定義
- `docs/FUTURE_EVENT_DND.md` — 将来実装候補
- `docs/housing-tour-planner-requirements.md` — ハウジングツール要件
- `docs/TECH_NOTES.md` — 技術的な落とし穴と解決策（必読）

## 会話中の方針決定の記録
ユーザーとの会話で決まったアイデア・方針・設計判断は、必ず `docs/TODO.md` に追記すること。
- **アイデア・要望** → 「アイデア・やりたいこと」セクションに追記
- **確定した設計方針** → 「確定した設計方針」セクションに追記
- **新しいタスク** → 適切なセクション（進行中・未着手・バグ等）に追記
- **完了したタスク** → 「完了」セクションに日付付きで移動

これらは指示されなくても自動的にやること。会話で決まったことが `docs/TODO.md` に反映されていなければ、次のセッションで失われる。

## 言語設定
- 常に日本語で会話する
- コメントも日本語で記述する
- エラーメッセージの説明も日本語で行う
- ドキュメントも日本語で生成する

## 色のルール
UIの全体的なデザイン整えを行うまでは、すべてのUI要素に白と黒のみを使用すること。
アクセントカラー、赤、青、緑などの色は使わない。
既存のテーマ変数（app-text, app-bg等）の白黒はOK。

## 多言語対応の徹底
UIテキストは必ず i18n キー経由で表示すること。ハードコーディング禁止。
英語モードで表示が崩れないか常に確認すること。

## CSS記述ルール
- **`backdrop-filter: blur(...)` を直接書くな。** Tailwind v4のLightning CSSがビルド時に削除する。必ず `--tw-backdrop-blur` 変数パターンを使うこと。詳細は `docs/TECH_NOTES.md` を参照。

## UIデザイン「やるな」リスト
AIっぽいデフォルトデザインを避けるためのルール。指示がなければデフォルトに戻るので、明示的に禁止する。
- AIグラデーション禁止（青→紫のやつ）
- Interフォント禁止
- Lucideアイコンのみの使用禁止（他のアイコンセットも検討すること）
- 「shadcnデフォルトそのまま」禁止（カスタマイズして使うこと）
