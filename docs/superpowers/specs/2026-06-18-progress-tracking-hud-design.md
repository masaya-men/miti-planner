## 目的 / 背景

タイムライン（軽減表）で攻略を「何日もかけて進める」体験を、**日ごとの最高到達点の波（非単調な軌跡）**として可視化し、踏破時にお祝いする。FF14 攻略の「3 歩進んで 2 歩下がる」感覚を独自のビジュアルにすることが核。

- 試作ブランチ `feat/progress-celebration-proto`（throwaway・main 未マージ）に試作あり：
  - `src/dev/progressProto/ProgressProtoPanel.tsx` / `src/dev/progressProto/useProgressProtoStore.ts` / `src/dev/progressProtoMode.ts`
  - `npm run dev:progress` で `/miti` ヘッダー下段中央に浮かせて確認できる。
  - `canvas-confetti` は試作ブランチに導入済み。**main には未導入**＝本実装で依存追加が必要。
- 本実装は **main で新規作成**し、試作の見た目資産を参照する。最終的に `src/dev/progressProto/` は撤去。

## スコープ（やること / やらないこと）

### やる
- 攻略の軌跡 HUD（ヘッダー下段中央）を **PlanData 駆動の本実装**にする。
- 到達点の記録 UX（HUD クリック → 記録パネル → タイムラインの時間クリックで打点）。
- クリアボタン＋お祝い演出（canvas-confetti ＋ スキルアイコン降り）。
- 表示制御（端末グローバル・デフォルト ON・ワンクリックでオフ・⋯メニューで復帰）。
- 空状態（記録ゼロ）の誘導表示。
- 任意オプションの活動日数・活動時間（手入力・デフォルト非表示）。

### やらない（今回スコープ外）
- **活動○×（直近 N 日の活動有無）**。「誰がいつ来れたか」はスケジュール調整機能と地続きのため、そちらと一緒に後で設計する。データ（`dailyBest.day`）は持つので後で再利用可能。
- スケジュール調整機能そのもの（別 spec）。
- collab 限定ゲート。本機能は **collab/ソロ問わず**動く（下記参照）。

## 表示制御

- **対象**: 全プラン（collab/ソロ問わず）。進捗データは**表（プラン）ごと**に持つ。
- **置き場所**: ヘッダー下段中央（左ツール群と右リスト群の間）。試作が dev-gate で組み込まれている場所と同じ。
  - 試作の `import.meta.env.DEV && isProgressProto()` ゲートを、本実装では**「進捗バー表示フラグ」**に差し替える。
- **表示フラグ**: 端末グローバル（localStorage）・**デフォルト ON**。
- **オフ**: HUD 上のワンクリック操作で即非表示（× ボタンを想定するが、最終的な具体形は実機を見て決める）。状態は localStorage 保存で**全プランに効く**。
- **復帰**: ヘッダー「⋯ その他」メニュー（`HeaderToolsMenu.tsx`）に「**進捗バーを表示**」項目を追加。
- **空状態（記録ゼロ）**: 誘導型。「クリックして攻略の軌跡を記録」的な案内を帯に出し、押すと記録パネルへ。1 点でも記録されたら自動で軌跡表示に切り替わる。

## データモデル

`src/types/index.ts` の `PlanData` に `progress?` を追加（既存プランは `undefined`＝`memos?` と同じ後方互換パターン。マイグレ不要）。

```ts
export interface PlanProgress {
  /** 軌跡の本体。day = カレンダー日付(JST, 'YYYY-MM-DD')、reachedPos = その日の最高到達点(タイムライン上の秒位置) */
  dailyBest: { day: string; reachedPos: number }[];
  /** クリアボタンで true */
  cleared: boolean;
  /** 任意・手入力。デフォルト非表示 */
  activeDays?: number;
  /** 任意・手入力。デフォルト非表示 */
  activeHours?: number;
}
```

- **軌跡（棒グラフ）**: 横軸＝日付順、縦軸＝`reachedPos`。1 日 1 バー。同じ日に再記録したら**最高到達点に更新（max）**。日をまたいで上下する非単調の波。
- **進捗％**: `最高reachedPos ÷ タイムライン全長`。クリアで 100%。**デフォルト表示**。
- **活動日数 / 時間**: `±` カウンターで手入力。入れた人だけ HUD 左に「○日 ○時間」表示。

### 永続化
- `progress` は `PlanData` の一部なので、**既存のプラン保存（Yjs 同期＋Firestore 永続）に相乗り**する。進捗専用の Firestore 書き込み・読み込みは増やさない（コストは塵）。
- collab の **seed-on-open** にも `progress` を乗せる → クリア済みの表を開いた瞬間にお祝いが出る。
- データ量は 1 打点が十数バイト。100 日記録しても数 KB（メモ機能と同程度以下）。

## 記録 UX

HUD をクリックすると**記録パネル**を開く（スマホは `MobileBottomSheet` 流用）。パネルの中身：

1. **到達点を記録**: 「今日どこまで進んだ？」→ タイムライン上の時間をクリックして打点。
   - 長い表でも探さずに済むよう **フェーズジャンプ（既存機能）** を併設。
   - タイムラインの時間列は編集不可だが、**記録モード中だけクリックを横取り**して時間値だけ取得（編集には入らない）。**既存のクリック挙動＝軽減配置・編集を壊さないこと**（実装時に実機総点検）。
2. **クリアボタン**: 踏破（時間切れ/エンレイジ到達）はボタンで記録 → `cleared = true` → お祝い演出。
3. **（任意）活動日数・時間**: `±` カウンター。入れると HUD 左に表示される。
4. **誤記録の修正**: その日の点を消す/直す手段を記録パネルに小さく用意。

打点ルール: 同じ日の再記録は最高到達点に更新（max）。

## ビジュアル

**HUD レイアウト**: 左＝進捗％（＋オプションで「○日 ○時間」）／中央＝軌跡（主役）／右＝なし。

**軌跡の光**:
- 棒グラフ（幅あり）。光は**滑らかな線の尾**（SVG `stroke-dasharray` 方式。canvas はボツ済）が棒の輪郭を辿って左→右へゆっくり進み、右端で消えて左から再開（ループ）。
- **常時の道（軌跡全体の線）は描かない**。通った後の余韻だけ。尾はやや長め。
- **クリア時のみ全線点灯＋パルス**。
- 尾の長さ・速度・グロー強さの最終チューニングは `npm run dev:progress` で実機調整。

**お祝い演出**:
- **canvas-confetti** ではじけ（中央大砲＋左右）＋**スキルアイコンが降り注ぐ**（設定パーティのジョブが持つスキルアイコン全部）＋中央に「おめでとう！」（多言語）。
- **クリックするまでループ**。
- 発火: **クリアボタン押下時** ＆ **クリア済みの表を開いた時** の両方。

## リスク / 注意点

- **タイムラインのクリック横取り**は記録モード中だけに限定し、既存の軽減配置・編集挙動を壊さない（memory `feedback_structural_refactor_runtime_audit`：共有 DOM/座標基準を触る改修は実機総点検）。
- **collab 同時編集の競合**: 複数人が同時に `dailyBest` を触る場合は Yjs の自動マージに乗せる。配列のマージ挙動（同日 max・重複日の扱い）は plan 段階で詰める。
- **canvas-confetti / スキルアイコン降り**は本番ゲート（表示フラグ）連動。dev でないときも出るので、パフォーマンス（大量パーティクル）に注意。
