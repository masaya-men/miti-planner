# マスタデータ（データベース）設計方針

## 1. 目的
`mockData.ts`等で管理されているスキル（軽減・バリア・回復）やジョブ、タイムラインのイベントデータを、多言語対応と将来の拡張（オートプランの精度向上や新スキルの追加）に耐えうる堅牢なデータ構造に再設計する。

## 2. データ構造の基本ルール

### A. 多言語対応の内包 (LocalizedString型)
データ内の名称や説明文は、単一の文字列や `nameEn` のようなフラットなプロパティではなく、言語キーをプロパティに持つオブジェクト構造とする。

```typescript
type LocalizedString = {
    ja: string;
    en: string;
    // 将来的に fr, de など追加可能
};
```

### B. スキル（Mitigation）のデータスキーマ定義
各種軽減やバフのデータは、計算ロジック（オートプラン）が解釈しやすいように、効果量を明確に型定義する。

```typescript
interface MitigationDef {
    id: string;           // 一意のID (例: 'reprisal')
    jobId?: string;       // 紐づくジョブ (ロール共通アクションの場合は undefined)
    role?: string;        // 紐づくロール (例: 'tank')
    name: LocalizedString; // 多言語化されたスキル名
    icon: string;         // 画像パス
    recast: number;       // リキャストタイム（秒）
    duration: number;     // 効果時間（秒）
    
    // --- 効果定義（オートプランやダメージ計算の頭脳となる部分） ---
    isShield: boolean;    // バリアかどうか
    isInvincible?: boolean; // 無敵かどうか
    scope: 'party' | 'target' | 'self' | 'enemy'; // 効果範囲
    
    // 軽減率（％）
    value?: number;          // 全属性軽減
    valuePhysical?: number;  // 物理軽減
    valueMagical?: number;   // 魔法軽減
    
    // 回復・バリア系
    healingIncrease?: number; // 回復魔法効果アップ（％）
    // バリア量などの係数は、キャラクターステータスと連動させるための計算式を別途考慮する
}
```

## 3. 今後のタスク
1. 現在の `MITIGATIONS` および `JOBS` の配列を、上記の `LocalizedString` を含む構造にリファクタリングする。
2. 各スキルの `value`（軽減率）や `duration` などの数値がFF14の最新パッチに準拠しているか確認し、不足しているスキルがあれば追加する。
3. `CheatSheetView.tsx` や `Timeline.tsx` など、データを参照している側のコンポーネントを、新しいデータ構造（例: `def.name.ja`）に合わせて修正する。
