## 確定した設計方針（2026-03-28 第27セッション追記）

### 管理基盤・Firestore移行の設計方針（確定）
- 全ゲームデータ（コンテンツ・テンプレート・スキル・ステータス・アイコン・DC/サーバー）をFirestoreに段階的移行
- 操作中のFirestoreアクセスゼロ（起動時1回フェッチ→localStorage+メモリキャッシュ→同期的に使用）
- バージョン番号でキャッシュ無効化（変更時のみ再取得）
- マスターデータは種類ごとに1ドキュメント（読み取り回数の最適化）
- 管理画面は `/admin` ルート（同一アプリ内、Custom Claims認証）
- テンプレート自動生成: FFLogsインポートから発見フェーズ（14日）→ロック → 人気プラン昇格（管理者承認制）
- アイコンはFirebase Storage + Vercelエッジキャッシュで配信
- Firebase App Check + APIレート制限 + 監査ログ + BAN機能
- ローカル開発は本番Firestoreに読み取り専用接続（静的ファイルはフォールバック用）
- 詳細設計書: `docs/superpowers/specs/2026-03-28-admin-master-data-platform-design.md`

### プラン複製機能の設計方針（確定）

## 確定した設計方針（2026-03-27 第24セッション追記）

### ガラス表現の視認性向上方針
- タイムライン枠: ボーダー光沢 + 影（ダーク:グレー線+にじみ、ライト:黒パキッと線）。blurは不要（中のテーブルが読めなくなる）
- ヘッダー区切り線: 今より明るくするだけ。影やglowは不要（小さすぎて効果がない）
- サイドバー選択中: 控えめなボーダー追加。枠として認識できる程度
- モーダル: 既にglass-tier3でそこそこ。ボーダーだけ合わせてもいい
- **全部に同じスタイルではなく、場所に合った強さにする**（ごちゃごちゃ防止）
- テーブル横罫線: ユーザーがトグルで選択可能（デフォルト非表示）
- 光の周回アニメーションは軽減表ページではやらない（作業中ずっと見る画面なので集中の邪魔になる）

### サイドバーのハイライト階層ルール（確定）
- 開いているプラン: 左インジケーター（border-l-2）で「今ここ」を明示
- アクティブなプランを含む大カテゴリ: 全周ボーダーで強調
- ただ展開しているだけの大カテゴリ: グレーハイライトのみ
- ヘッダー区切り線: ダーク=白25%、ライト=純黒

### パフォーマンス最適化の順序（確定）
- **視覚的な変更を全部終えてから最後にやる**
- 理由: UIの見た目を変えるとコンポーネント構造やpropsが変わる。先にメモ化すると見た目変更のたびにメモ化を壊したり調整し直す手間が増える
- React.memo / useMemo で対応予定

## 確定した設計方針（2026-03-24）

### サイドバーの操作フロー
- コンテンツクリック = 即編集開始。「テンプレート」という概念はユーザーに一切見せない
- テンプレートがあるコンテンツ → 裏で読み込み → 自動でプランとして保存 → 即編集
- テンプレートがないコンテンツ → 空プランを自動作成 → 即編集
- 同コンテンツに既存プランがある場合 → 即オープン（重複作成しない）
- 同コンテンツ1件以上 → サブアイテム展開。末尾に「+ 追加」行
- プラン名はサブアイテムのペンシルボタンでインライン編集（ヘッダーではダブルクリック）
- コンテンツ未紐付きプラン（自由作成）はレベル別「カスタム」セクションに表示
- 「新規作成」ボタンはレベルタブの上に控えめ配置

### 保存の仕組み（2層構造・確定方針 2026-03-30 第51セッション改定）
- **1層：localStorage（リアクティブ・無音・コストゼロ）**
  - 変更検知→500msデバウンス→即保存
  - 保存インジケーター（「保存中...」→「保存済み ✓」）はlocalStorage保存完了に連動
- **2層：Firestore（イベント駆動 + 定期バックアップ・ログインユーザーのみ）**
  - 保存タイミング：ページ離脱 / タブ非表示 / プラン切替 / ログアウト / 5分ごとの定期同期
  - 3分クールダウンで過剰書き込みを防止、forceSyncAllは10秒タイムアウト付き
  - 編集のたびにFirestoreに送るのはNG（試行錯誤中の無駄な同期を避ける）
  - ログイン時にカウンター修復（repairPlanCounts）+ ローカルが新しいプランはFirestoreに書き戻し
  - ログアウト時は全プラン強制同期（forceSyncAll）

### 共有の仕組み
- POST /api/share → Firestore → nanoid 8文字短縮URL → /share/:shareId
- PUT /api/share → 既存shareIdのロゴを上書き更新（第39セッション追加）

## 管理者ログイン手順（初回セットアップ）

### 準備
1. **Vercelに環境変数`ADMIN_SECRET`を追加**
   - Vercelダッシュボード → Settings → Environment Variables
   - 名前: `ADMIN_SECRET`
   - 値: 長いランダム文字列（例: 32文字以上の英数字）
   - 対象: Production, Preview, Development すべてにチェック
   - 追加後、再デプロイが必要（Settings → Deployments → 最新を Redeploy）

2. **自分のFirebase UIDを確認**
   - Firebase Console → Authentication → Users → 自分のメールアドレスの行のUID列をコピー

### 管理者ロール設定
3. **ターミナルから以下のコマンドを実行**（UIDとsecretを自分のものに置き換える）
```bash
curl -X POST "https://lopoly.app/api/admin?resource=role" \
  -H "Content-Type: application/json" \
  -d '{"uid":"ここにFirebase UID","role":"admin","secret":"ここにADMIN_SECRET"}'
```
   - 成功: `{"success":true,"uid":"...","role":"admin"}`
   - 失敗: `{"error":"Unauthorized"}` → secretが間違っている

4. **ブラウザでログインし直す**（Custom Claimsの反映にはトークン更新が必要）
   - ログアウト → 再ログイン
   - または: 1時間待つ（トークンの自動更新）

5. **管理画面にアクセス**
   - https://lopoly.app/admin にアクセス → ダッシュボードが表示されれば成功

### トラブルシューティング
- `/admin`にアクセスしてもトップページにリダイレクトされる → ログアウト→再ログインを試す
- curlで403が返る → `ADMIN_SECRET`の値がVercelの環境変数と一致しているか確認
- curlで500が返る → Vercelのログ（Functions タブ）でエラー内容を確認
