# ハウジング詳細ページ(PC) 紹介文ホバー全文表示 設計書 (2026-07-20)

## 背景

PC版ハウジング詳細ページの右カラム(幅320px固定)は、タイトル/住所/元URL/登録者/タグ/紹介文/他の登録 を1つのスクロール域に詰め込んでいる。紹介文が長いとタグ等の下に埋もれ、読むのにスクロールが要る。ユーザーからは「紹介文をさっと読めて、UXを損なわずおしゃれにしたい」との要望。

ブレスト過程で「右カラム全体のスクロールを無くす」案も出たが、タグ数や重複登録件数が多い物件でそれらが画面からはみ出るリスクがあるため不採用。**紹介文の扱いだけを変え、他要素の並び・スクロールは現状維持**で決定。

## スコープ

- 対象: PC版詳細ページ (`HousingDetailContent.tsx` の `.housing-detail-description`) のみ。位置・順序は変更しない。
- スマホ版・タグ・住所・他の登録セクションは無変更。

## 決定した挙動

1. 紹介文は現在の位置のまま、**3行でクランプ**(`-webkit-line-clamp:3`)。3行に収まる短い紹介文はそのまま全文表示(クランプの見た目にならない)。
2. クランプが実際に効いている(=省略されている)ときだけ、紹介文のすぐ下に小さく薄いグレー文字で「ホバーで全文を確認」の案内を出す。省略されていない場合はこの案内も出さない。
3. 紹介文のブロックに **マウスホバー、またはキーボードフォーカス** すると、全文を表示するカードがふわっと(opacity+わずかなtranslateYのフェード)浮き出る。マウスアウト/フォーカス外れで閉じる。
4. カードの幅は右カラムの320pxに収めず、写真エリア側に少しかぶるくらい広め(目安420px、画面が狭い場合は`min(420px, viewport - 余白32px)`)にして実際に読みやすくする。
5. カードは housing.css の既存トークン(`--housing-panel-bg-solid` / `--housing-panel-border` 等)を使った静かなガラスパネルの見た目にする。honey色の装飾ピルや色付きalert箱にはしない([[feedback_housing_no_ai_pills]]準拠)。

## 技術的な注意点

- `.housing-detail-textscroll`(紹介文を含む右カラムのスクロール域)は `overflow-y:auto` により暗黙的に `overflow-x` も `auto` 化されるため、素朴に `position:absolute` で浮かせるとカードが横方向にクリップされる。
- 対策: モーダルと同じ `createPortal(..., document.body)` パターンで body 直下に描画し、対象要素の `getBoundingClientRect()` から座標を計算して配置する([[reference_fixed_inside_backdrop_filter]] と同型の対策)。
- ホバー判定は「クランプで実際に省略が発生しているか」を判定する必要がある(`scrollHeight > clientHeight` 等)。判定タイミングは要素マウント時 + リサイズ時([[reference_perf_forced_reflow_resizeobserver]] に倣い ResizeObserver で計測、hover 中の同期 reflow は避ける)。
- キーボードフォーカスでも同じ表示を発火させるため、紹介文ブロックに `tabIndex=0` を付与し `onFocus`/`onBlur` を `onMouseEnter`/`onMouseLeave` と同じハンドラに揃える。
- マウスアウト時は即座に閉じず、わずかな遅延(150ms程度)を挟んでちらつきを防ぐ。

## スコープ外

- 紹介文以外の要素(タグ/住所/他の登録)の並び替え・スクロール仕様変更
- 右カラムの列幅自体の変更
