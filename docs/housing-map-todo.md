## 現状 (2026-05-21 時点)

- マップは **PNG 画像** 1 枚 (`public/housing/maps/sample-ward.png`)。家の位置は `src/data/housing/sampleWardLayout.ts` に 0〜1 割合座標で 30 区画ぶん手書き。**これはサンプル(モック)**。
- マップ素材は masaya が**実機マップを自分でなぞって抽象化した自作物**。ゲーム素材は使わない (権利クリーンで広告 OK・世界観統一のため)。→ ベクター(SVG)化に最適。

## ナビの本当の仕組み (行き方スプレッドシートで判明)

FF14 ハウジングの移動は「ワード全体の最短経路計算」ではなく、
**① 目的の家の最寄りエーテネットにテレポ → ② そこから短い徒歩**。
masaya の行き方スプレッドシートはこれを全 60 区画ぶん言葉で記録済み:

| 番地 | 最寄りエーテライト(テレポ先) | 行き方補足(言葉ナビ) |
|------|------------------------------|----------------------|
| 1 | ミストゲート・スクエア | 西の階段をまっすぐ降りたとこ |
| … | … | … |
| 60 | [拡張街]ミスト・ヴィレッジ南西 | 西の突き当りのＬハウス |

→ **マップに必要な座標データは「エーテネットの位置」と「家の箱の位置」だけ**。
道の中央線・全交差点ノード・経路計算アルゴリズムは**MVP には不要**。言葉ナビが本体、線アニメは将来の飾り。

---

## 区画番号 (1〜60)

- **表(本区画) = 1〜30 / 裏(拡張街) = 31〜60**。スプレッドシートと同じ番号でユーザーに見せる。
- 拡張街は本区画と同じレイアウト形状だが、最寄りエーテネット名が `[拡張街]…` で別、行き方の向きも違う (シートに両方記録済)。
- 対象 5 エリア: ミスト / ラベンダーベッド / ゴブレット / シロガネ / エンピレアム (シートにエリア別タブがある想定)

## A. masaya 本人の作業 (SVG を描いて名前を付ける) 🎨

ベクター編集ソフト (Illustrator / Figma / Inkscape など) で、エリアごとに SVG を作り、要素に**名前(id)** を付ける。名前がそのままデータになる。

### A-1. 背景の絵
- [ ] 自作の抽象化ワード俯瞰図を **SVG** で書き出す (既存のなぞり絵を SVG 化)
- [ ] 5 エリア分。縦横比を揃える

### A-2. 家の箱 (区画) — ★MVP 必須
- [ ] 各区画を図形で描き、**番地で名前を付ける** (`plot-1` 〜 `plot-60`)
  - 表 1〜30 / 裏(拡張街) 31〜60。表裏で形が同じなら 1 枚を使い回し、コードで +30 表示でも可
- [ ] (任意) サイズ S/M/L を名前か属性で区別

### A-3. エーテネット(最寄りエーテライト) — ★MVP 必須
- [ ] 各エーテネットを点で打ち、**スプレッドシートの「最寄りエーテライト」列と完全一致する名前**を付ける
  - 例 `ミストゲート・スクエア` `[拡張街]ミスト・ヴィレッジ南西`
  - これがナビの起点。名前が一致していないと家と紐付かない

### A-4. 道の中央線・交差点 (将来の飾り・後回し可)
- [ ] 道幅の真ん中に 1 本線を引き、交差点で繋げ、レイヤー名 `roads`
- [ ] 交差点に点を打ち `node-1` `node-2` …
  - ※ 言葉ナビがあるので MVP では不要。線でルートを描きたくなったら作る
