# 管理画面ウィザードファースト刷新 設計書

> 作成日: 2026-03-31 第60セッション

---

## 目的

管理画面を「記憶ゼロ・知識ゼロでも確実に操作できる」ウィザードファーストUIに全面刷新する。サービス登録のオンボーディングのように、1画面1質問で迷いようがないレベルを目指す。

## 方針

- **UIのみ刷新。API・データ構造は変更しない**
- 既存機能を制限しない（任意項目を必須にしない、特殊フラグを設定できる等）
- 管理者は1人（権限分けは不要）
- frontend-designスキルでデザインを仕上げる

---

## 1. ダッシュボード（ホーム画面）

### 構成

**メイン: 「何をしますか？」カード一覧**

| カード | 説明 | 遷移先 |
|--------|------|--------|
| 新しいコンテンツを追加 | 零式・絶などの新コンテンツ登録 | コンテンツ追加ウィザード |
| テンプレートを登録 | タイムラインテンプレートの登録 | テンプレート登録ウィザード |
| スキルを追加・編集 | ジョブ・軽減スキルの管理 | スキル編集ウィザード |
| ステータスを更新 | パッチごとのHP・メインステ等 | ステータス更新ウィザード |
| サーバー情報を編集 | DC・ワールド・ハウジングエリア | サーバー編集画面（直接編集） |
| 設定 | 昇格しきい値など | 設定画面（直接編集） |

**補助セクション:**

- **最近の変更**（監査ログから直近5件）— 日本語で表示（例:「3時間前：スキル『リプライザル』のリキャスト時間を変更」）。「すべて見る」リンクで監査ログ画面へ
- **バックアップから復元** — 復元画面へのリンク

**サイドバーナビゲーションも残す**（慣れたら直接アクセス可能）

## 2. ウィザード共通仕様

全ウィザードに共通するUI仕様。

### 画面構成

```
┌─────────────────────────────────┐
│  ← 戻る          ステップ 2/5   │
│  ━━━━━━━━━━░░░░░░░░░░░░        │  ← プログレスバー
│                                 │
│  質問テキスト                    │
│                                 │
│  [入力フィールド / 選択肢]       │
│                                 │
│           [次へ →]              │
└─────────────────────────────────┘
```

### 共通ルール

| 要素 | 仕様 |
|------|------|
| 進行バー | 上部に「ステップ N/M」＋プログレスバー |
| 戻るボタン | 全ステップに配置。前の入力は保持される |
| 次へボタン | 必須項目が入力されるまで無効（グレーアウト） |
| 任意項目 | 「スキップ」ボタンを別に表示。「次へ」と混同しない |
| 確認画面 | 最終ステップで入力内容の一覧を表示。各項目の横に「編集」ボタン（そのステップに戻れる） |
| 成功画面 | 「登録しました」＋「ダッシュボードに戻る」 |
| エラー時 | 何が間違っているか具体的に表示（「コンテンツ名（日本語）を入力してください」等） |
| 入力例 | プレースホルダーで例を表示（薄いグレー文字） |

### バリデーション方針

- 必須 vs 任意は既存APIのバリデーションと一致させる
- 数値フィールドは範囲制限（負の数や非現実的な値を防ぐ）
- コンテンツIDは入力中にリアルタイムで重複チェック（「このIDは使えます」「既に使われています」）
- IDの重複はAPI側でも409エラーで二重チェック済み

## 3. コンテンツ追加ウィザード

### ステップ一覧

| ステップ | 質問 | 入力方式 | 必須/任意 |
|---------|------|---------|----------|
| 1 | カテゴリを選んでください | 大きなボタン5択（零式/絶/ダンジョン/レイド/カスタム） | 必須 |
| 2 | レベルを選んでください | 大きなボタン4択（Lv70/80/90/100） | 必須 |
| 3 | コンテンツIDを入力してください | テキスト入力。例:「m5s」。リアルタイム重複チェック表示 | 必須 |
| 4 | コンテンツ名（日本語）を入力してください | テキスト入力。例:「次元の狭間オメガ零式：アルファ編4」 | 必須 |
| 5 | コンテンツ名（英語）を入力してください | テキスト入力。例:「Alphascape V4.0 (Savage)」 | 必須 |
| 6 | どのシリーズに属しますか？ | 既存シリーズのリスト＋「新しいシリーズを作る」ボタン。新規時はシリーズ名入力欄がその場に展開 | 必須 |
| 7 | パッチ番号を入力してください | テキスト入力。例:「7.2」。数値形式チェック | 任意（スキップ可） |
| 8 | FFLogs用のIDはありますか？ | テキスト入力。例:「67」。分からなければスキップ | 任意（スキップ可） |
| 確認 | 入力内容の一覧表示 | 全項目を1画面に表示。各項目の横に「編集」ボタン | — |

### 補足
- 短縮名（shortName）はコンテンツIDから自動生成（手動入力不要）
- ステップ3にヒント:「パッチ番号を含めると将来の重複を避けやすいです（例: 7.2-m5s）」（必須ではなくガイダンス）

## 4. テンプレート登録ウィザード

### 最初の分岐（ステップ1）

| 選択肢 | 説明 | 用途 |
|--------|------|------|
| FFLogsからインポート | URLを貼ってボス行動を取得 | 新コンテンツの初期テンプレート |
| 自分のプランをテンプレートにする | 編集済みプランを選択 | 調整済みプランの公式化。共有URLからコピーしたプランも含む |
| JSONファイルをアップロード | 従来方式 | バックアップからの復元等 |

### A. FFLogsインポートの流れ

| ステップ | 質問 | 入力方式 |
|---------|------|---------|
| 1 | どの方法で作りますか？ | 大きなボタン3択 |
| 2 | どのコンテンツのテンプレートですか？ | 登録済みコンテンツのリストから選択 |
| 3 | FFLogsのURLを貼ってください | テキスト入力。URL形式チェック。例を薄く表示 |
| 4 | インポート結果のプレビュー | イベント数・フェーズ数を表示。「○件のボス行動を取得しました」 |
| 確認 | 内容確認 → 「テンプレートとして登録する」 |

### B. プランからテンプレート化の流れ

| ステップ | 質問 | 入力方式 |
|---------|------|---------|
| 1 | どの方法で作りますか？ | 大きなボタン3択 |
| 2 | どのプランをテンプレートにしますか？ | 自分のプラン一覧から選択。コンテンツ名・最終編集日つき |
| 3 | プレビュー | イベント数・フェーズ数を表示。軽減配置は除外されテンプレート（ボス行動のみ）になることを説明 |
| 確認 | 内容確認 → 「テンプレートとして登録する」 |

### C. JSONアップロードの流れ

| ステップ | 質問 | 入力方式 |
|---------|------|---------|
| 1 | どの方法で作りますか？ | 大きなボタン3択 |
| 2 | どのコンテンツのテンプレートですか？ | 登録済みコンテンツのリストから選択 |
| 3 | JSONファイルを選んでください | ファイル選択。形式チェック。エラーがあれば具体的に表示 |
| 確認 | 内容確認 → 「テンプレートとして登録する」 |

## 5. スキル編集ウィザード

### 最初の分岐

| 選択肢 | 説明 |
|--------|------|
| 新しいスキルを追加する | 新スキルの登録 |
| 既存のスキルを編集する | 値の調整（リキャスト変更等） |
| 新しいジョブを追加する | 新ジョブの登録（拡張パッケージ時） |

### A. 新しいスキル追加の流れ

| ステップ | 質問 | 入力方式 | 必須/任意 |
|---------|------|---------|----------|
| 1 | どのジョブのスキルですか？ | ジョブ一覧から選択（アイコン付き） | 必須 |
| 2 | スキル名（日本語） | テキスト入力。例:「リプライザル」 | 必須 |
| 3 | スキル名（英語） | テキスト入力。例:「Reprisal」 | 必須 |
| 4 | 軽減率は何%ですか？ | 数値入力。0〜100。例:「10」 | 必須 |
| 4a | 最初だけ追加軽減がありますか？ | はい/いいえ。「はい」→ 追加軽減率（%）と持続秒数の入力が出る | 任意（いいえでスキップ） |
| 5 | 効果時間は何秒ですか？ | 数値入力。0.1〜999。例:「15」 | 必須 |
| 6 | リキャスト時間は何秒ですか？ | 数値入力。0〜999。例:「60」 | 必須 |
| 7 | 軽減の種類は？ | ボタン3択（全体/魔法のみ/物理のみ）。「物理と魔法で軽減率が違う」チェックボックスあり → チェック時は個別数値入力 | 必須 |
| 8 | 効果範囲は？ | ボタン3択（自分のみ/パーティ全体/対象指定） | 必須 |
| 8a | 自分自身にも使えますか？ | はい/いいえ（ステップ8で「対象指定」を選んだ場合のみ表示） | 条件付き必須 |
| 9 | 使用可能レベルは？ | 数値入力。1〜100。例:「50」 | 必須 |
| 10 | アイコン画像を選んでください | ファイル選択（画像） | 任意（スキップ可） |
| 11 | 特殊な動作はありますか？ | チェックリスト形式。選んだものだけ次ステップで詳細入力 | 任意（なければスキップ） |
| 確認 | 入力内容一覧 → 「登録する」 | |

### ステップ11 特殊動作チェックリスト

選択すると、その項目に対応する追加入力ステップが動的に追加される。

| チェック項目 | 説明 | 追加入力 |
|-------------|------|---------|
| シールド | ダメージを吸収するバリアを張る | valueType（HP%/ポテンシー）、shieldPotency、stacks、reapplyOnAbsorption、onExpiryHealingPotency |
| 無敵になる | ダメージが0になる | なし（isInvincible=trueのみ） |
| 他のスキルを先に使う必要がある | 前提スキルが必要 | requires（スキルIDを選択） |
| フェアリーが必要 | フェアリーを召喚していないと使えない | なし（requiresFairy=trueのみ） |
| チャージ制 | 複数回分を溜められる | maxCharges（数値） |
| リソースを消費する | エーテルフロー等 | resourceCost.type（選択）、resourceCost.amount（数値） |
| 回復量もUPする | 回復ポテンシーが増加 | healingIncrease（%）、healingIncreaseSelfOnly（はい/いいえ） |

### B. 既存スキル編集の流れ

| ステップ | 質問 | 入力方式 |
|---------|------|---------|
| 1 | どのジョブですか？ | ジョブ一覧から選択 |
| 2 | どのスキルですか？ | 選んだジョブのスキル一覧から選択 |
| 3 | 編集画面 | 現在の値が全て表示され、変更したい項目だけ直接編集（1画面に全項目表示） |
| 確認 | 変更箇所のみハイライト表示 → 「保存する」 |

### C. 新ジョブ追加の流れ

| ステップ | 質問 | 入力方式 | 必須/任意 |
|---------|------|---------|----------|
| 1 | ジョブID | テキスト入力。例:「vpr」 | 必須 |
| 2 | ジョブ名（日本語） | テキスト入力。例:「ヴァイパー」 | 必須 |
| 3 | ジョブ名（英語） | テキスト入力。例:「Viper」 | 必須 |
| 4 | ロール | ボタン3択（TANK/HEALER/DPS） | 必須 |
| 5 | アイコン画像 | ファイル選択 | 任意 |
| 確認 | 入力内容一覧 → 「登録する」 |

## 7. サーバー情報・設定画面

ウィザード化しない。現在の直接編集画面を維持し、各フィールドの説明文を充実させる。

理由：更新頻度が極めて低く（新DC追加時、設定変更時のみ）、1問1答にする意味が薄い。

## 8. バックアップ復元画面

### アクセス
ダッシュボードの「バックアップから復元」カード

### 一覧表示
種類（スキル/ステータス/コンテンツ/テンプレート）× 日時のリスト

### フィルタ
種類で絞り込み可能

### 復元フロー
1. 復元したいバックアップを選択
2. 確認ダイアログ:「本当に復元しますか？現在のデータは自動でバックアップされます」
3. 実行 → 成功メッセージ

### 安全策
- 復元実行前に現在のデータも自動バックアップ（復元の復元が可能）
- Firestoreの既存バックアップコレクション（`master_backups/`、`template_backups/`）をそのまま使用

### API
既存の `/api/admin/templates` エンドポイントにGETパラメータ `?type=backups` と PUTパラメータ `?type=restore` を追加（Vercel関数枠を消費しない）

## 9. 監査ログ画面

### アクセス
ダッシュボードの「最近の変更」→「すべて見る」リンク

### 表示
日時・操作内容を日本語で表示（例:「スキル『リプライザル』のリキャスト時間を60→45に変更」）

### フィルタ
種類（スキル/コンテンツ/テンプレート等）で絞り込み

### API
既存の `/api/admin/templates` エンドポイントにGETパラメータ `?type=logs` を追加

## 10. 技術方針

### コンポーネント設計
- `AdminWizard` — 共通のウィザードフレームワーク（プログレスバー、戻る/次へ、確認画面）
- 各ウィザード（`ContentWizard`、`TemplateWizard`、`SkillWizard`、`StatsWizard`）はステップ定義を渡すだけ
- 既存の `AdminContentForm` 等は段階的に置き換え

### API変更
- **既存APIのロジックは変更しない**
- バックアップ一覧・復元・監査ログ閲覧の3つを既存エンドポイントに追加（Vercel関数枠を消費しない）

### デザイン
- frontend-designスキルで仕上げる
- 白黒ベース（CLAUDE.mdの色ルールに準拠）
- 直感的で迷いようがないUI
