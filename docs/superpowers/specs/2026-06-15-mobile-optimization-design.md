# スマホ最適化 設計メモ (2026-06-15)

> ユーザーとのブレストで確定したスマホ UX 改善の設計。実装は段階的・実機確認しながら。
> デプロイは全項目完了後にバッチ(push=本番自動デプロイ)。

## 背景 / 目的
スマホ・タブレットでの操作性改善。タブレットは画面が広く PC 版が出る(業界標準的に許容)が、
PC 前提の操作(キーボード/ホバー)にタッチ代替が無い箇所が問題だった。本セッションで
個別バグ修正 + ボトムナビ再設計を行う。

## 確定スコープ

### 済(本コミット前に実装・コミット済)
- **対象選択バグ**: スマホで対象指定スキル/鼓舞コピーの選択 UI が出ず即配置 → 修正(`resolveMitigationTap`)。
- **チュートリアル F キー詰み**: タブレットで F が押せず行き止まり → キーキャップをタップ可に。
- **C 共同編集参加ヘッダー**: 独自簡易ヘッダー → 本物 `MobileHeader`(viewer + 右スロット)に統一。二重余白解消。※本番確認はデプロイ後。
- **D FAB 背景**: 約2倍濃く(FAB 専用 `--color-fab-overlay` トークン)。実機 OK。

### A. ボトムナビ再設計(本メモの主対象)
**新ナビ(5タブ・順番固定)**: `メニュー / インポート / カンペ / 共有 / ログイン`

| タブ | 中身 | 現状からの変更 |
|---|---|---|
| **メニュー** | プラン一覧 / コンテンツ選択 / パーティ編成(内に MY JOB トグル) / 自動組み立て / ☕支援(埋もれない位置) | パーティ編成・自動組み立てを集約。MY JOB トグルをパーティ編成内へ。支援を上位に。 |
| **インポート** | FFLogs 取込(将来スプシ/URL) / みんなの軽減表 | 「ツール」を改名。Undo/Redo を撤去、自動組み立てをメニューへ逃がす。みんなの軽減表は「人の表を取り込む=インポート」として同居。 |
| **カンペ** | カンペビュー(`mobile:open-cue-sheet`) | FAB から昇格。 |
| **共有** | 配布フロー(コピー/共同編集 + 複数選択) | 新設。中身は **B で別途設計**(A では枠 + 入口まで)。 |
| **ログイン** | アカウント | 変更なし。 |

**廃止タブ**: パーティ(→メニュー)、MY JOB(→パーティ編成内トグル)。

**Undo/Redo**: ナビ枠を使わず **FAB 近くに常設**の小さい浮きボタン(親指圏)。ツールシートからは撤去。
業界標準(編集系は常設ツールバー、ナビ枠は消費しない)。

**支援の可視化(あ案)**: 現状サイドバー最下部([Sidebar.tsx:1574](../../../src/components/Sidebar.tsx#L1574))=スクロール必須で埋もれる。
メニューシート内で **スクロールなしで目に入る位置**(sticky 下部 or 上部)に出す。タイムライン本体やポップアップでは促さない(非催促)。

**文言の PC 一致 + 将来防止**:
- スマホのツール文言ベタ書き("Auto Plan" / "FFLogs Import")を廃止し **PC と同じ i18n キー参照**に統一。
  - 自動組み立て = `mitigation.auto_plan`(「軽減自動組み立て」)
  - FFLogs = `fflogs.tooltip_generate`(「からタイムラインを生成」)+ ブランド表記「FF Logs」
  - みんなの軽減表 = `popular.open_popular`(PC と同一)。**スマホの 2 行目 `popular.subtitle` を外し PC とラベル一致**。
- 単一ソース(共通キー)化により、将来スプシ対応(⑤⑥⑧)でキーを 1 箇所変えれば PC+スマホ両方追従=再ズレ防止。**「将来の名前」を今先取りで変えない**(再設計時に決める)。

### B. 共有配布フロー(A の後・別設計)
共有タブをタップ → スマホ限定で「コピー配布 / 共同編集配布」を選択 + 複数選択。
既存の共同編集入口(⑤ ShareModal の2択)とスマホ版で統合。**別途ブレスト/設計**。
