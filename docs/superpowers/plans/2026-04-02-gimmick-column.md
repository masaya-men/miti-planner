# ギミック区間列の追加 — 実装計画

## 概要
フェーズ列の右に「ギミック区間」列を追加。フェーズ内をさらにギミックグループで区切って表示する。

## 列幅（PC / md以上）

| 列 | 現在 | 変更後 | 備考 |
|---|------|-------|------|
| フェーズ | 100px | **60px** | 縦書き化、上位階層なので少し広め |
| **ギミック（新規）** | — | **50px** | 縦書き1行表示 |
| 時刻 | 70px | **60px** | 10px削減 |
| イベント | 200px | 200px | 変更なし |

**合計: 170px → 170px（変更なし）**

## コントロールバーとの整合性

Area A: `PHASE(100) + TIME(70) = 170px` → `PHASE(60) + GIMMICK(50) + TIME(60) = 170px`

**Area A 幅は170pxのまま。コントロールバー・区切り線すべて変更不要。**

## データソース

`TimelineEvent.mechanicGroup?: string` — 既存フィールド。テンプレートに設定済み。

ギミック区間の算出ロジック:
```
timelineEvents を時間順に走査
→ mechanicGroup が変わるたびに区間を区切る
→ フェーズと同様に startTime / endTime の縦バーで表示
```

## スクロール時の挙動

- **フェーズ列**: 現行と同じ sticky 表示（スクロールで上に残る）
- **ギミック列**: フェーズと同様に sticky 表示（スクロールで上に残る）
- 両列ともスクロールに追従して現在のフェーズ名・ギミック名が常に見える

## 変更対象ファイル

### 1. Timeline.tsx（ヘッダー + オーバーレイ）
- **ヘッダー**: フェーズ列の幅を100px→60px
- **ヘッダー**: フェーズ列の右にギミック列ヘッダー（50px）を挿入
- **時刻ヘッダー**: 幅70px→60px
- **フェーズオーバーレイ**: 幅100px→60px、left位置調整
- **ギミック区間オーバーレイ**: フェーズオーバーレイ（line ~1857）と同じ仕組みで追加。left=60px、width=50px
- **フェーズ名**: 縦書き（`writing-mode: vertical-rl`）に変更

### 2. TimelineRow.tsx（セル）
- フェーズ列セルの幅を100px→60px
- フェーズ列の右にギミック列セル（50px）を挿入（背景色のみ、テキストはオーバーレイ）
- 時刻列セルの幅を70px→60px

### 3. モバイル対応
- ギミック列: 非表示（`hidden md:flex` / `hidden md:block`）
- ギミックオーバーレイも `hidden md:block` で非表示
- フェーズ列: モバイル幅は現状維持（24px）、縦書きもモバイルでは不要（現行の短縮表示のまま）
- 時刻列: モバイル幅は現状維持（36px）

## 表示仕様

- フェーズと同じオーバーレイ方式（縦バー + 縦書きラベル）
- フェーズ・ギミックともに縦書き（`writing-mode: vertical-rl`）
- ギミックは1行で縦書き表示
- 色: フェーズとは異なる控えめなスタイル（要相談）
- 折りたたみ機能: 初回は無し。必要になったら追加

## リスク

- **低**: 合計幅170pxを維持するため、コントロールバー・他列への影響なし
- **低**: データは既存、表示ロジックはフェーズ列のコピー＆改変
- **確認事項**: フェーズ名が60px縦書きで読みやすいか（実装後に目視確認）
