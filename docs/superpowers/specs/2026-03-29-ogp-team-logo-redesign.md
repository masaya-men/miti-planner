# OGP画像 チームロゴ背景デザイン刷新

## 概要

OGP画像（共有時にSNSに表示される1200x630pxの画像）にユーザーのチームロゴ画像を背景として使用する。画像を50%暗くし、コンテンツ名・プラン名を大きく中央に配置する。ロゴ未設定時は現行デザインを維持する。

## 決定事項

- **暗さ**: 50%（`rgba(0,0,0,0.5)` オーバーレイ）
- **画像配置**: `object-fit: cover` + `object-position: center`（アスペクト比維持、余白なし、中央クロップ）
- **テキスト**: 中央配置、大きなフォント、テキストシャドウで視認性確保
- **左パネル**: 浸食不可。ファビコン（`favicon-512x512.png`）+ 縦書き "LoPo"
- **下帯なし**: テキストは画像上に直接重ねる
- **バンドル同シリーズの区切り**: 縦線（`｜`）

## レイアウト構成

### 共通構造（全パターン）

```
┌──────┬──────────────────────────────────────┐
│      │                                      │
│ fav  │   ユーザー画像（50%暗）               │
│      │   + テキスト中央配置                  │
│ L    │                                      │
│ o    │                                      │
│ P    │                                      │
│ o    │                                      │
│      │                                      │
└──────┴──────────────────────────────────────┘
 144px              1056px
```

### パターン1: ロゴなし（現行維持）

条件: `teamLogoUrl` がない場合
→ 現行の黒背景デザインをそのまま使用。変更なし。

### パターン2: 単体プラン共有（ロゴあり）

条件: 1プラン共有 + `teamLogoUrl` あり

テキスト構成:
- カテゴリタグ: `SAVAGE — Lv.100`（18px、letter-spacing 5px、白40%透過）
- コンテンツ名: `至天の座アルカディア零式：ヘビー級1`（52px、weight 900、白）
- プラン名: `M9S`（22px、白45%透過）— ユーザーがON/OFF可能（既存の `showTitle` パラメータ）

### パターン3: バンドル共有 — 同シリーズ（ロゴあり）

条件: 複数プラン共有 + 同一シリーズ + `teamLogoUrl` あり

テキスト構成:
- カテゴリタグ: `SAVAGE — Lv.100`（18px）
- シリーズ名: `至天の座アルカディア零式`（42px、weight 900）
- まとめ表記: `ヘビー級 1 ｜ 2 ｜ 3 ｜ 4`（30px、weight 700、白70%透過）

#### 同シリーズ判定ロジック

contentId からシリーズを判定する:
- `m9s`, `m10s`, `m11s`, `m12s_p1`, `m12s_p2` → 至天の座アルカディア零式：ヘビー級
- `_p1` 接尾辞 → 「前半」、`_p2` → 「後半」

まとめ表記のルール:
- 層番号を抽出して縦線区切りで並べる
- 前半・後半があればそのまま付ける
- 飛び番もそのまま表示（例: `1 ｜ 3`）

例:
- `m9s + m10s + m11s + m12s_p1 + m12s_p2` → `ヘビー級 1 ｜ 2 ｜ 3 ｜ 4前半 ｜ 4後半`
- `m9s + m11s` → `ヘビー級 1 ｜ 3`
- `m11s + m12s_p1 + m12s_p2` → `ヘビー級 3 ｜ 4前半 ｜ 4後半`

#### 同シリーズ判定条件

全プランの contentId が同じシリーズプレフィックスに属する場合に適用。
例: `m9s`〜`m12s_p2` はすべて `m` シリーズ（ヘビー級）。
異なるシリーズが混在する場合（例: `m9s` + `fru`）はパターン4（混在）を使用。

### パターン4: バンドル共有 — 混在コンテンツ（ロゴあり）

条件: 複数プラン共有 + 異なるコンテンツ混在 + `teamLogoUrl` あり

テキスト構成:
- ヘッダー: `N PLANS SHARED`（14px、letter-spacing 5px、白25%透過）
- コンテンツリスト: 各コンテンツ名（37px、weight 900、白）+ ID（17px、白25%透過）
- 件数に関わらずフォントサイズは固定（変えない）
- 各アイテムの間に `rgba(255,255,255,0.06)` の区切り線

## 左パネル仕様（浸食不可）

- 幅: 144px
- 背景: `#030303`
- 右ボーダー: `1px solid #111`
- 上部装飾ライン: 1px × 72px、`#1a1a1a`
- ファビコン: `favicon-512x512.png`、64x64px、丸形
- 縦書き "LoPo": 各文字を縦に配置、28px、weight 700
- 下部装飾ライン: 上部と同じ

**重要**: `grape.svg` ではなく `favicon-512x512.png` を使用する。

## テキスト視認性

全テキストに共通のテキストシャドウを適用:
```
text-shadow: 0 2px 16px rgba(0,0,0,0.9), 0 0 6px rgba(0,0,0,0.7);
```

## 既存機能との互換性

- `showTitle` パラメータ: 単体共有時のプラン名表示ON/OFFは維持
- `logoUrl` パラメータ: 既存のクエリパラメータをそのまま使用
- ロゴなし時: 現行デザインを完全に維持（回帰なし）

## 変更対象ファイル

- `api/og/index.ts` — OGP画像生成ロジックの刷新
  - `buildSingleLayout()` — ロゴあり時に新レイアウト、なし時は現行維持
  - `buildBundleLayout()` — 同シリーズ判定 + まとめ表記 or 混在リスト
  - `buildLeftPanel()` — `grape.svg` → `favicon-512x512.png` に変更
  - 同シリーズ判定ヘルパー関数の追加

## 変更しないもの

- ShareModal.tsx のUI（既に修正済み）
- logoUpload.ts（既に修正済み）
- storage.rules、firestore.rules
- ロゴアップロード/削除フロー
