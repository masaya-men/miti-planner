## 確定した設計方針（2026-03-24）

### サイドバーの操作フロー
- コンテンツクリック = 即編集開始。「テンプレート」という概念はユーザーに一切見せない
- テンプレートがあるコンテンツ → 裏で読み込み → 自動でプランとして保存 → 即編集
- テンプレートがないコンテンツ → 空プランを自動作成 → 即編集
- 同コンテンツに既存プランがある場合 → 即オープン（重複作成しない）
- 同コンテンツ1件以上 → サブアイテム展開。末尾に「+ 追加」行
- プラン名はサブアイテムのペンシルボタンでインライン編集（ヘッダーではダブルクリック）
- コンテンツ未紐付きプラン（自由作成）はレベル別「カスタム」セクションに表示
- 「新規作成」ボタンはレベルタブの上に控えめ配置（極・滅・ヴァリアントダンジョン等で使用）

### 保存の仕組み（2層構造・確定方針）
- **1層：localStorage（常時・無音・無料）**
  - 30秒間隔 + プラン切替時 + ページ離脱時に自動保存
  - ユーザーは「保存」を一切意識しない。トースト通知なし
  - ネットもデータベースも使わない
- **2層：Firestore（間引き・ログインユーザーのみ）**
  - 保存タイミング：タブ切替 / ページ離脱 / プラン切替 / 3分以上編集時に1回
  - 前回から変更があった場合のみ書き込む（dirty flag方式）
  - 1セッションあたり5〜10回に抑え、DAU 2,000〜3,000人で無料枠内に収める
  - 未ログインユーザーはlocalStorageのみ（Firestore書き込みゼロ）

### 共有の仕組み
- ヘッダーのリンクコピーまたはX共有ボタン
- POST /api/share → Firestoreに保存 → nanoid 8文字の短縮URL
- /share/:shareId で閲覧 + 「自分の軽減表にコピー」ボタン
- 軽減内容が変わったら共有URLキャッシュをクリア（新しいURLを発行）

### デザイン方針
- まず全体を白黒ベースに統一 → あとでアクセントカラーを入れる
- ヘッダーのヒーロータイトル: コンテンツ名は省略なし、ユーザーが付けたプラン名の方を省略

### モバイル設計方針（確定 2026-03-24）
- **全パネルはMobileBottomSheet方式** — メニュー・パーティ・ステータス・ツールすべてボトムナビの上にシートで表示。再タップで閉じる。排他制御。
- **ボトムナビは常に最前面** — z-[400]で全シートの上。シートはbottom:4remでナビの上に配置。
- **コントロールバーはモバイル非表示** — Compact/Undo/Redoはツールシートに集約。表の行数を稼ぐ。
- **軽減追加は5列フラットグリッド** — MT→D4の順で全メンバーのスキルを羅列。メンバーごとのグルーピングなし。同名スキル（ロールアクション）が被る場合のみジョブアイコンバッジ。配置後もシートを閉じない（連続配置可）。
- **ダメージ表示はモバイルで千単位省略** — 211,000→211k、1,500,000→1.5M
- **ヘッダーカラム名は英語略語** — Ph / Raw / Tkn（日英共通）
- **PC版には一切影響させない** — すべてモバイル判定（window.innerWidth < 768 / md: ブレークポイント）で分岐

### プラン管理方針
- 1コンテンツあたり最大5件 / 合計最大50件
- 30件超過でアーカイブ警告を表示（「現行零式・絶以外の古い軽減表は自動的に圧縮保存されます」）
- 50件でハードブロック（作成不可）
- 古いプラン（30日以上未編集 + 現行零式・絶以外）はFirestoreに圧縮退避 → 次回オープン時に復元（Cloud Functions必要・将来実装）
- 将来的に収益化で上限拡張も検討可能

### FFLogs API管理方針（2026-03-25確定）
- FFLogsインポートはログイン限定（API保護 + ログインメリット強化）
- クライアント側レート制限: 1ユーザー1時間15回まで
- FFLogs API制限: 120秒あたり240リクエスト（キーごと）
- 1回のインポートで約6〜10回のGraphQLリクエスト消費
- **APIキー5組でラウンドロビン運用中**（api/fflogs/token/index.tsで自動切り替え）
- 追加キーはVercel環境変数に FFLOGS_CLIENT_ID_2〜5 / FFLOGS_CLIENT_SECRET_2〜5 で登録
- 悪意あるユーザーはFirebase Auth無効化でBAN（当面は手動、将来自動化）
