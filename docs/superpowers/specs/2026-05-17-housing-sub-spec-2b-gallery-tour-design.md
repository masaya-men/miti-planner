## 1. 設計判断の根拠 (なぜそうしたか)

### 1.1 「1 ページ完結 (Adaptive Workspace)」 に変更した理由

親仕様はボトムナビ式 (探す → ツアー → 登録の 3 画面切替) だったが、 ユーザー使い勝手の検討で「**画面遷移を挟まずブラウジング → ツアー組立 → ツアー実行を一気に完結したい**」 が判明した。 業界の現代的なベストプラクティスもこの方向 (Figma / Notion / VS Code / Linear)。

| 採用パターン | 業界名 | 代表例 |
|---|---|---|
| 1 ページ + 開閉パネル | **Adaptive Workspace** | Figma, VS Code, Notion |
| 左フィルタ + 中央メイン + 右サブ | **Master / Detail / Aux** | Linear, GitHub Project, Slack |
| マップ + リスト同居 | **Map-List Hybrid** | Airbnb, Zillow, SUUMO |
| 地図ピン or ホバーカード | **Map-Based Card Overlay** | Google Maps Place card, Apple Maps Look Around |
| マルチ選択 + DnD でツアー組立 | **RTS-style Multi-select + DnD** | RTS ゲーム, Trello, Notion DnD |

### 1.2 各 UI 要素の「なぜここか」

| 要素 | 配置 | 根拠 |
|---|---|---|
| ロゴ | top bar 左 | F パターンの起点 (視線が最初に届く) |
| グローバル検索 | top bar 中央 | サイト全体のサーチは中央が定石 (GitHub, Linear) |
| 登録 CTA | top bar 右 + 左パネル末尾 | Dual entrypoint。 常時アクセス + 探索後の自然な「私もやってみよう」 動線 |
| お気に入り (♡) | top bar 右 | 「自分のもの」 系は右側のアイコン集約 (Twitter, Instagram) |
| アバター | top bar 右端 | アカウント系は最も右、 業界 100% |
| Faceted filter | 左パネル | F パターン 2 行目、 探索の起点。 Airbnb / Zillow と同じ |
| Result count | 左パネル ヘッダー | Airbnb / Zillow / Hotels.com の標準位置 |
| ビュー切替トグル (Map/Pinterest) | 中央エリア 右上 | Yelp / GitHub Issues / Project の慣習 |
| マップ吹き出しカード | 家マークの上 | Google Maps / Apple Maps の Place card 流 |
| 物件リスト (右パネル) | 中央右 | サブ情報帯。 ツアーモードで主役化 |
| 共有ボタン | 右パネル上部 | ツアー中の主要アクション、 サーフェスさせる |
| 設定 (テーマ等) | status bar | グローバル状態の永続表示、 mockup の硬派タクティカル感を継承 |

### 1.3 iterate-first の宣言

UI/UX は仕様だけでは精度が出ない。 **実装してブラウザで触り、 調整する**プロセスが必要。 本書は次を確定する:
- 画面構造 (パネル分割、 要素配置、 状態モード)
- 各 UI 要素の存在理由 (なぜここか)
- インタラクションフロー (何が起きるか)
- 状態管理 (Zustand store 設計)
- アクセシビリティ要件

以下は**確定しない** (実装で iterate する):
- 色の正確な値 (mockup の honey/glow/candle を初期値)
- 全 spacing 値 (mockup を初期値)
- アニメ秒数 / イージング
- リキッドグラス filter パラメータ (mockup の `applyLiquidGlass` を初期値)
- カードグリッドの列数閾値
- auto-scroll 速度

## 2. トンマナ (モックアップ準拠、 親仕様 §11.2 をオーバーライド)

### 2.1 採用するデザイン言語

**コンセプト**: 「綺麗な景色を窓越しに眺める」

ハウジングは LoPo 本体 (黒+白+機能色の硬派タクティカル路線) とは**別世界観**で進める。 居心地・お家紹介の情緒を出すため、 暖色 + 景色 + 物理的ガラス表現を採用。

### 2.2 背景動画 (テーマ切替)

| テーマ | 動画 | 制作プロセス |
|---|---|---|
| Light | 朝/昼の景色、 5 秒シームレスループ | 元素材 8 秒 → ffmpeg xfade 焼き込み (F=3 秒) |
| Dark | 夜景、 28 秒シームレスループ (0.25x スロー) | 元素材 8 秒 → `setpts*4` でスロー化 (32 秒) → xfade 焼き込み (F=4 秒) |

**ループ繋ぎ目の処理**: ピンポン (順 ⇄ 逆) は「雲ドカン戻り」 が出るため**不採用**。 代わりに ffmpeg の `blend=all_expr='A*(1-T/F)+B*(T/F)'` で末尾と冒頭を crossfade して焼き込む。 動きの少ない素材ほど自然 (動画自身がぼやけ気味なので crossfade が見えない)。

ファイル形式:
- H.264 mp4 (Safari/iOS フォールバック)
- VP9 webm (Chrome/Firefox 優先)
- ポスター WebP (中央フレーム抽出)

`prefers-reduced-motion: reduce` 時は動画停止 + ポスター静止画を表示。

### 2.3 ガラスエフェクト (Liquid Glass Precision Lens 流)

**採用ロジック** (Lucky Graphics 流、 kube.io 色収差方式は不採用):

| 要素 | 採用 / 不採用 |
|---|---|
| SVG `feImage` (displacement map データ) | ✅ |
| SVG `feDisplacementMap` (屈折) | ✅ |
| SVG `feColorMatrix` (R/G/B 分離) | ❌ 暗くなる副作用 |
| SVG `feOffset` (色収差オフセット) | ❌ |
| SVG `feComposite` (加算合成) | ❌ 同上 |
| SVG `feGaussianBlur` | ❌ ぼかし不採用 (動画自体がぼやけ気味) |
| CSS `backdrop-filter: blur()` | ❌ 同上 |
| CSS `saturate(...)` | ❌ ガラスに色載せない |

**Displacement map 生成ロジック**:
- Canvas に displacement vector を ImageData として描画 → toDataURL → `feImage` href
- 各ピクセルでの計算:
  1. **dEdge**: rounded rect SDF (パネル形状に沿う、 円形にならない)
  2. **magnitude**: `Math.pow(1 - t, 1.6)` で縁から内側に滑らかに減衰 (t = dEdge / edge)
  3. **vector**: 4 縁からの 1/距離重み付き smooth blending (Math.min() ハード選択は対角線で「角から中央への鋭い線」 が出る、 必須回避)
  4. **encode**: R=128+ux\*127\*magnitude, G=128+uy\*127\*magnitude, B=128

**確定パラメータ** (mockup 初期値、 実装で iterate):
```js
// .panel
{ edge: 160, radius: 18, scale: 49 }
// .top (top bar)
{ edge: 30, radius: 0, scale: 25 }
// .status (status bar)
{ edge: 24, radius: 0, scale: 20 }
```

**Safari フォールバック**: `-webkit-backdrop-filter: none` (素通し、 ガラス感落ちるが許容)

### 2.4 カラーパレット

| 用途 | 値 | 役割 |
|---|---|---|
| 蜂蜜色 (主アクセント) | `#ffc987` | 重要アクション、 強調 |
| グロー (副アクセント) | `#ffb35a` | ホバー、 アクティブ |
| ろうそく (補助アクセント) | `#ffe2b3` | 落ち着いた強調 |
| パネル背景 | `rgba(255, 255, 255, 0.04)` | 超薄ガラス (動画が透ける) |
| パネル枠 | `rgba(255, 255, 255, 0.22)` | 境界 |
| テキスト | `#ffffff` | 両テーマ共通の白固定 |
| テキスト (淡) | `rgba(255, 255, 255, 0.78)` | サブ情報 |
| テキスト (極淡) | `rgba(255, 255, 255, 0.55)` | ヒント、 disabled |

**テキスト視認性**: 動画背景上で読めるよう、 全テキストに `text-shadow: 0 1px 2px rgba(0,0,0,0.55), 0 0 10px rgba(0,0,0,0.32)` を適用。

### 2.5 タイポ

- Latin: "Inter", "Helvetica Neue", -apple-system, sans-serif
- 日本語: "Hiragino Kaku Gothic ProN", "Yu Gothic UI"
- ベース 14px、 行間 1.55
- `-webkit-font-smoothing: antialiased`

**LoPo 本体ルールの例外**: `.claude/rules/ui-design.md` で Inter は禁止だが、 ハウジングは**別世界観**として明示的に例外指定 (親仕様 §11.2 オーバーライドと同じ精神)。 LoPo 本体の硬派タクティカル路線とは違う、 居心地・景色を見せるための採用。

### 2.6 LoPo 全体のサイズ設計思想は維持

ハウジングだけ世界観が独自でも、 サイズ設計 (CLAUDE.md 思想ファイル参照) は LoPo 全体と共通:
- 全 text を px ベース (rem 使わない)
- UI 要素は `clamp(MIN_PX, N_VW, BASE_PX)` で proportional scaling
- アンカー viewport 1489
- container max-width 1489 で大画面の余白確保

### 3.2 状態モード

| モード | トリガー | 主な変化 |
|---|---|---|
| **閲覧モード** (default) | 初回ロード | 右パネルは「現在表示中の物件リスト」 を auto-scroll |
| **ツアーモード** | お気に入りモーダルから「全部回る」/ ツアー URL を開く | 右パネルがツアー進行画面に。 中央のマップ/Pinterest は次の物件をハイライト |
| **お気に入り編集モード** | top bar の ♡ / アバター → 「お気に入りを見る」 | 超でかいモーダルが開く (下透け)。 ベース画面は背景として残る |

**モード重複時の挙動**:
- ツアーモード中も**お気に入りモーダルは開ける** (ツアー実行を中断せず重ねる)。 モーダルを閉じればツアー進行に戻る
- ツアーモード中に「全部回る」 で新しいツアーを開始 → 確認ダイアログ「進行中のツアーを破棄して新しいツアーを開始しますか?」

### 3.3 パネル開閉

- **左パネル**: 閉じれて中央拡張。 閉じてもフィルタ条件は維持
- **右パネル**: ツアー外なら閉じれる、 ツアー中は固定 (進行表示のため)
- 両方閉じれば中央エリアが全幅 (動画 + マップ/Pinterest のみ)

**閉じる / 再オープン UI の配置**:
- **TopBar 左端 / 右端にパネルトグルボタン** (lucide `PanelLeftClose ⇄ PanelLeftOpen` / `PanelRightClose ⇄ PanelRightOpen`) を常時表示。 状態に応じてアイコンと aria-label (`panel.open_left` ↔ `panel.close_left`) を反転。 Figma / Notion 流
- パネル内側 (左パネル末尾の `PanelCloseButton`) からも直接閉じれる二重 UI
- 「両方閉じれば中央全幅」 仕様を守るため、 閉じた状態のサイドにレールやハンドルは残さない (再オープン経路は TopBar に一本化)
- ツアー中の右パネルトグルは `disabled` (進行固定)

### 3.4 モバイル (Phase 1 では設計しない、 PC 完成後 1 から)

ユーザー判断で、 モバイル UX は PC 実装が固まった後に**再設計**する。 本書には含めない。 親仕様 §10.3 のボトムナビ案は破棄、 1 ページ完結に整合する新設計を起こす。

## 4. 中央エリア (View Mode 切替)

### 4.1 マップビュー

- 1 ワード = 30 軒 (裏 = 別マップで各 30 軒)
- 家マークの上に**吹き出しサムネカード**が浮く
- 全 30 軒すべて表示、 密集対応はホバー拡大
- マップ画像は Figma 書き起こし (Phase 2 で本実装、 Sub-spec 2B 期間はモック画像を使用)

**初回訪問の振る舞い**:
- ワード単位のランダム選出
- 制約: 登録件数 **5 軒以上**のワードからのみ
- 結果は URL クエリには反映しない (リロードで違うワードが出る楽しみ)
- ユーザーが絞り込み or ワード手動切替したらランダム動作を解除

### 4.2 Pinterest ビュー (Masonry grid)

- 切替トグル: 中央エリア右上
- ソート: 新着順 (上から並ぶ)
- グリッド: 列数は画面幅で自動決定 (PC 大画面 4 列、 中 3 列、 狭め 2 列)
- 切替時: 左右パネル開きっぱなし
- アニメ: 縦の crossfade (300ms 程度、 実装で iterate)

### 4.3 物件カード (共通仕様)

- マップビューでは吹き出しカード、 Pinterest ビューでは Masonry セル
- 共通要素: サムネ画像、 住所 (短形式)、 サイズバッジ、 タグ最大 2 個
- **クリック挙動**: **inline expansion** (その場で展開、 ページ遷移なし、 モーダル開かない)
- 展開時の追加情報:
  - 画像大表示 (動画 OGP は埋め込み)
  - 紹介文 (200 文字まで)
  - 全タグ
  - 「↗ SNS で開く」 リンク (新タブ)
  - [♡ お気に入り] [+ ツアーに追加] アクションボタン

### 4.4 マップ吹き出しカード

- ベース: サムネ + 住所 + サイズ
- **ホバー時**: 拡大 + タグ表示 + アクションボタン (♡, +ツアー) 表示
- **連動なし**: 右パネルや他カードとは独立 (情報はカード内自己完結)

### 4.5 ビュー切替時の状態継承

- 絞り込み条件: 引き継ぐ
- 表示中のワード: マップビューでのアクティブワードがある場合、 Pinterest 切替時はそのワード内の物件を新着順で表示

## 6. 右パネル: 物件リスト + ツアー進行

### 6.1 閲覧モード

**内容**: 中央エリアの現在表示状態に追従:
- マップビュー → そのマップ内の 30 軒
- Pinterest ビュー → 表示中の全件 (絞り込み結果)

**挙動**:
- auto-scroll で流れる (速度は実装で iterate)
- ホバーで停止
- リスト 1 件クリック → 中央の対応物件に inline expansion を発動

### 6.2 ツアーモード

**遷移トリガー**:
- お気に入りモーダルから「全部回る」 ボタン
- ツアー共有 URL を開いた時

**内容**:
- ツアーに登録された物件が訪問順 (住所順 = DC → サーバー → エリア → 区 → 番地) で並ぶ
- 現在の進行位置をハイライト
- 進行: **Enter** または **Space** で次へ (キーボードナビ)
- マウス操作派には「↓ 次へ」 ボタンも併設
- 中央のマップ/Pinterest は次の物件にビュー追従

**共有ボタン**:
- 右パネル上部に大きく「📤 このツアーを共有する」 を表示
- 押下 → URL 発行 + クリップボードコピー (トースト確認)

### 6.3 共有 URL の振る舞い

**ツアー共有 URL**:
- 形式: `/housing/tour/{id}`
- 受信側: URL 開く → 通常画面の状態にツアーが復元 (右パネル進行モード + 中央ハイライト)
- マナー順守ポップアップ (親仕様 §8.5) はツアー開始時に**1 度だけ**表示、 「次回から表示しない」 を LocalStorage に保存

**物件単体共有 URL** (親仕様 §10.1 を維持):
- 形式: `/housing/p/{id}`
- 押下経路: inline expansion の中に「🔗 この物件の URL をコピー」 ボタン
- 受信側: URL 開く → 通常画面の状態でその物件が中央エリアに inline expansion で開いた状態で着地。 マップビューならその物件のあるワード、 Pinterest なら新着順表示中で該当カードまで auto-scroll してハイライト

## 7. お気に入りモーダル + ツアー組立

### 7.1 開き方 (multi-entrypoint)

| 入口 | 場所 | コピー |
|---|---|---|
| ♡ アイコン | top bar 右 | (アイコンのみ) |
| アバターメニュー | top bar 右端 | 「お気に入りを見る」 |
| (Phase 2 追加検討) | mock マップ上の状態バー等 | TBD |

### 7.2 構造

```
┌─ お気に入りモーダル (フルスクリーン、 下透け backdrop) ──────────┐
│ [全部回る (n軒)]  [📤 ツアーを共有する]            [× 閉じる]    │
├─ お気に入り一覧 ───────────┬─ ツアーエリア (DnD 受け) ─────────┤
│ ☐ [横長低背カード]         │ 1. [シロガネ 3-12 (M)]     [×]   │
│ ☐ [横長低背カード]         │ 2. [シロガネ 3-15 (S)]     [×]   │
│ ☐ [横長低背カード]         │ 3. [ラベンダー 5-7 (L)]   [×]   │
│ ☐ ...                      │   ⋮                              │
│                            │   ↑ 住所順 自動整列、 DnD で編集可│
│ ↑ クリック / Shift+クリック│                                  │
│   Ctrl+クリック / 矩形選択 │                                  │
│   → DnD でツアーエリアへ  │                                  │
└────────────────────────────┴───────────────────────────────────┘
```

### 7.3 横長低背カード仕様

- 高さ 64px 程度 (実装で iterate)
- 左: サムネ (正方形、 56x56 程度)
- 中: 住所 + サイズバッジ
- 右: 紹介文の最初の 50 文字 (省略付き)
- 左端: チェックボックス (選択状態)

### 7.4 選択操作 (RTS-style)

| 操作 | 結果 |
|---|---|
| クリック | 単一選択 (他の選択は解除) |
| **Shift + クリック** | 範囲選択 (前回選択と現在の間を全選択) |
| **Ctrl + クリック** (Mac: Cmd) | 個別追加選択 |
| **矩形ドラッグ** | 矩形範囲内の全カードを選択 |
| DnD でツアーエリアへ | 選択中の全カードを移動 |
| Esc | 選択解除 |

**矩形選択の発動条件**:
- お気に入り一覧パネルの**カード以外の領域 (背景余白)** でマウスダウン開始 → 矩形ドラッグ開始
- カード自身でマウスダウン開始 → そのカードのクリック / DnD として扱う (矩形選択にならない)
- これは Windows Explorer / Finder と同じ仕様、 ユーザー学習コストゼロ

### 7.5 ツアーエリアの自動整列

- 入れたら住所順で自動並び替え
- **入った瞬間に並び替えが動的にアニメーションで可視化される** (新規項目が滑り込んでくる感、 既存項目が押しのけられる感)。 ユーザーが「何が起きたか」 を視認できるよう FLIP / spring 系の物理ベース transition を使用 (実装で iterate)
- DnD で並び順編集可 (自動整列をユーザーが上書き)
- 「住所順に戻す」 ボタン (上部)
- 各項目に × ボタン (ツアーから除外、 お気に入り側には残る)

### 7.6 「全部回る」 ボタン

- お気に入り全件を 1 タップでツアー化、 そのままツアーモードで実行
- お気に入り 0 件のとき: disabled、 ツールチップ「気に入ったお家を ♡ で集めてから使えます」

## 9. 空状態 / ロード / エラー

### 9.1 絞り込み 0 件

- 中央エリア中央: 「該当ハウジングがありません」 + 「条件を変更してください」 メッセージ
- 左パネルの結果カウントは「0 / 300 軒」 と表示維持

### 9.2 お気に入り 0 件 (モーダル開いた時)

- モーダル開いた直後なので「自分で開いている」 → 違和感なし
- 内側に薄く「気に入ったお家を ♡ で集めて、 自分だけのツアーを作ろう」 のヒント表示

### 9.3 自分の登録 0 件

- Phase 1 では「マイページ」 概念を独立に持たない → お気に入りモーダル内でタブ切替で「自分の登録」 を見せる案あり (実装時に判断、 iterate-first)

### 9.4 ロード中

- スケルトンカード (シルエット + シマー)
- マップビューは「読込中…」 オーバーレイ
- 動画背景はポスター → 動画 fade in

### 9.5 ネットワークエラー

- 画面下部にトースト「接続できません、 再試行してください」
- 指数バックオフで自動リトライ (3 回まで)、 全部失敗で手動再試行ボタン

## 10. インタラクションフロー (主要 3 経路)

### 10.1 受動ブラウザフロー

```
初回訪問 → ランダムワードのマップが出る → 30 軒の家マーク + 吹き出しカード
        → 興味ある家にホバー → カード拡大 → クリック → inline expansion
        → 詳細を見て [♡ お気に入り] → お気に入りに追加
        → 別のお家も同じく追加 → top bar の ♡ をクリック → お気に入りモーダル
        → [全部回る] → ツアーモード突入
```

### 10.2 能動探索フロー

```
左パネルでフィルタ操作 (DC=Mana, エリア=シロガネ, タグ=和風)
  → 結果カウント 23/300 に更新
  → 中央エリアのマップ/Pinterest が絞り込み結果に追従
  → Pinterest に切替 → 新着順で見やすく → 気に入ったお家 ♡
  → ツアー組立 → 共有 URL 発行 → SNS でシェア
```

### 10.3 ツアー受信フロー

```
SNS で他人の共有 URL クリック → /housing/tour/{id} 開く
  → 通常画面に着地、 右パネルがツアー進行モード
  → 中央が最初の物件をハイライト → マナー順守ポップアップ
  → [はじめる] → Enter で次へ次へ → 全 7 件回って完了
```

## 11. 状態管理 (Zustand store 設計)

### 11.1 store 分割

| store | 責務 | 永続化 |
|---|---|---|
| `housingFilterStore` | 絞り込み条件、 結果カウント | URL クエリ |
| `housingViewStore` | マップ/Pinterest 切替、 パネル開閉、 アクティブワード | sessionStorage |
| `housingFavoritesStore` | お気に入りリスト | LocalStorage (ゲスト) + Firestore (ログイン) |
| `housingTourStore` | ツアー組立状態 + 実行進行位置 | URL ([/tour/{id}](./tour/{id}) パス) |
| `housingRandomStore` | 初回ランダム選出済フラグ | sessionStorage (リロードで違うワード) |

### 11.2 URL 状態

- 絞り込み条件: クエリパラメータ (`?dc=Mana&area=Shirogane&tags=和風,カフェ`)
- 物件単体共有: パス (`/housing/p/{id}`) — 着地時に中央エリアでその物件が inline expansion 状態
- ツアー実行中: パス (`/housing/tour/{id}`)
- ツアー進行位置: クエリ (`?step=3`)

### 11.3 同期戦略

- お気に入り: ゲスト = LocalStorage 単独、 ログイン = Firestore 双方向同期 (ログイン時にマージ)
- ツアー: 同上

## 12. アクセシビリティ

- 全 button / link に `aria-label`
- マップ吹き出しカードに `alt` (サムネ画像) + `aria-label` (住所読み上げ)
- キーボードナビ:
  - Tab で各 UI 要素を順に focus
  - Enter / Space で開く・実行
  - Esc でモーダル閉じる
  - 矢印キーでツアー進行 (←前へ / →次へ)
- `prefers-reduced-motion: reduce`:
  - 動画背景停止 → ポスター静止画
  - auto-scroll 停止
  - transition 無効
- カラーコントラスト: WCAG AA (text-shadow で 4.5:1 以上確保)

## 17. iterate-first 項目 (本書で確定しない)

実装中に感覚で調整するもの:

- 全 padding / margin / spacing 値
- 暖色アクセントの正確な色値 (mockup の honey/glow/candle を初期値)
- 動画背景の透明度や暗くなる勾配 (mockup の `body[data-theme="light"] .scenery::before/after` を初期値)
- auto-scroll の速度 (px/秒)
- ホバー拡大のアニメ秒数 + イージング
- カードグリッドの列数閾値 (画面幅 N px で M 列)
- リキッドグラスのパラメータ微調整 (mockup 値からスタート)
- スケルトンカードのシマー速度
- トーストの表示時間
- DnD のホバーエフェクト
- ツアー共有 URL 受信側のツアー復元: 永続化は Firestore (`housing_tours/{id}`) 必要、 Sub-spec 2B の mock では tour id だけ復元 + listings は LocalStorage 連動
